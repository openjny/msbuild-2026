## 概要

すべての Windows PC でローカル AI アプリを構築するための Microsoft Foundry on Windows スタックを包括的に解説するセッション。Windows AI APIs（ターンキー API）、Foundry Local（オープンソースモデルのローカル実行）、Windows ML（カスタムモデル推論フレームワーク）の 3 層構造を、「Unmetered Token Café」というデモシナリオを通じて実演した。ローカル AI のメリット（プライバシー、低レイテンシ、オフライン動作、コスト削減）を軸に、5 台のデバイスを使ったライブデモとパートナー事例（ClipChamp、VoiceMod）を紹介。

## キーポイント

1. **Windows AI APIs の CPU/GPU 拡張**: 従来 NPU（Copilot+ PC）限定だった多くの AI API が CPU・GPU にも対応。Speech Recognition API、Phi Silica on GPU、Video Super Resolution 等がより幅広いデバイスで利用可能に
2. **Foundry Local が GA**: オープンソースモデル（GPT OSS、DeepSeek、Mistral、Qwen、Phi、Whisper 等）を CPU・GPU・NPU でローカル実行。新たに Qwen 3.5 ビジョン言語モデルや音声モデルもカタログに追加
3. **Windows ML 2.0 出荷**: ONNX Runtime 統合と新しいハードウェアベンダープラグイン実行プロバイダーにより、スケール・パフォーマンス・デプロイの 3 軸で強化。Windows 10 1809 以降、Windows Server 2019 以降、Windows 365 をサポート
4. **Windows ML CLI（Preview）**: モデルの変換・最適化・量子化・ベンチマークを一貫して実行できる統合ツールチェーン。GitHub Copilot 向けエージェントスキルも提供し、AI ドリブンなワークフローに統合可能
5. **WebNN によるウェブアプリ対応**: Chromium ベースブラウザ（Edge、Chrome）で Windows ML ベースのネイティブハードウェアアクセラレーションをウェブアプリに提供。NPU で 30ms レイテンシ・毎秒 11 件以上の推論を実現
6. **Aion（Phi Silica 後継）**: Windows AI APIs 経由で利用可能になる次世代オンデバイス SLM。コンテキストウィンドウ拡大、トークン/秒の高速化、モデル品質の向上を実現
7. **Gen AI ワークロードで最大 2.6 倍のスループット向上**: Windows ML の最新版でジェネレーティブ AI ワークロードのパフォーマンスが大幅改善

## 詳細

### Microsoft Foundry on Windows の 3 層アーキテクチャ

Microsoft Foundry on Windows は、ローカル AI を段階的に活用するための 3 層スタックを提供する:

1. **Windows AI APIs**: 最も簡単な入門レイヤー。Windows のインボックスモデルを利用するターンキー API で、自然言語処理、ビジョン、音声認識などの一般的なタスクに対応。開発者はモデル管理や配信を意識する必要がない
2. **Foundry Local**: オープンソースモデルをローカル実行するミドルレイヤー。SDK がモデル管理・デバイス固有のハードウェアアクセラレーションを抽象化し、デバイス固有コードの記述を不要にする
3. **Windows ML**: カスタムモデルを実行するローレベルフレームワーク。自社学習モデルや Hugging Face のモデルを ONNX 形式で推論実行。スタック全体のバックボーンとしてクロスシリコン機能を実現

### Windows AI APIs の拡張

従来は Copilot+ PC の NPU 限定だった多くの API が CPU・GPU にも対応拡大:

- **タスク固有 API（GA）**: 会話要約、画像説明、リライト、テキスト要約、テキスト→テーブル
- **カスタマイズ API**: LoRA for Phi Silica、アプリコンテンツ検索（RAG）、セマンティック検索、構造化出力
- **メディア API**: Video Super Resolution、Speech Recognition
- **イメージング API**: 画像生成、OCR、画像超解像、オブジェクト消去、画像セグメンテーション

Phi Silica on GPU は既に Outlook Summarize 等のプロダクション環境で使用されている。構造化 JSON 出力 API により、言語モデルの出力をプログラマチックに利用可能。

### Foundry Local の GA と新機能

Foundry Local が GA となり、以下を提供:

- **モデルカタログ**: GPT OSS、DeepSeek、Mistral、Qwen（新: 3.5 VLM）、Phi、Whisper（音声）等
- **Foundry Local SDK**: モデルダウンロード・ロード・推論をシンプルな API で実行。OpenAI 互換インターフェース
- **Foundry Local CLI**: モデルの探索・テストをインタラクティブに実行

デモでは Qwen 3.5（9B パラメータ）ビジョン言語モデルを使用し、画像からのインベントリ分類を実演。SDK は `get_model` → `model.download` → `model.load` → `create_responses_client` → `create_streaming` の簡潔なフローで利用可能。

### Windows ML CLI によるモデル最適化

Windows ML CLI（Preview、aka.ms/winmlcli）は以下のコマンドを提供:

- **Inspect**: モデルの適合性を事前チェック
- **Export**: PyTorch/Hugging Face モデルを ONNX 形式に変換
- **Analyze**: モデルグラフの各オペレーターのハードウェアサポート状況を可視化（緑=完全対応、黄=部分対応、赤=非対応）
- **Optimize**: グラフの書き換え・オペレーター融合・非対応オペレーターの解決
- **Perf**: スループット・NPU/GPU 使用率等のベンチマーク

プリビルト設定による一括実行、GitHub Copilot エージェントスキルによる AI ドリブン最適化、VS Code の Foundry Toolkit 拡張によるUI ベース操作にも対応。

### WebNN によるウェブアプリのネイティブ加速

WebNN は Chromium ベースブラウザ上で Windows ML のネイティブハードウェアアクセラレーションを利用するための API:

- ONNX Runtime Web API + WebNN により、ブラウザ内で ONNX モデルをネイティブ加速で推論
- CPU 実行時: 約 300ms レイテンシ、毎秒約 3.5 件の推論
- NPU 実行時: 約 30ms レイテンシ、毎秒 11 件以上の推論（3 倍以上の性能差）
- 現在 Preview。Edge/Chrome で実験フラグを有効化して利用可能

### パートナー事例

**ClipChamp（Video Super Resolution）**: Windows 内蔵の無料動画エディタで VSR を統合。動画のアップスケーリングをローカルで実行し、ぼやけた映像を高精細化。CPU・NPU の両方で動作。

**VoiceMod（リアルタイム音声変換）**: Windows ML を使用してリアルタイム音声変換を実現。45ms 未満の超低レイテンシで音声 AI モデルを推論し、CPU から NPU/GPU にオフロード。Windows ML により一度構築すれば AMD・Intel・NVIDIA・Qualcomm の全シリコンパートナーに展開可能となり、開発速度 4 倍を実現。
