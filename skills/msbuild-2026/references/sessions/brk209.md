# Japan Wrap-up Session

Microsoft Build 2026 の主要アナウンスを日本語で 45 分に凝縮して解説する Japan Wrap-up セッション。キーノート会場のクロージングアクトとして開催され、インフラ（Maia 200・Cobalt 200・HorizonDB）、モデル・エージェント（Scout Autopilot・IQ・MXC・Foundry 評価最適化）、Windows 開発環境（Foundry Local・AI APIs・Surface RTX Spark・WSL Containers・Copilot SDK GA）、クラウドプラットフォーム（AI データセンター Fairwater・Azure Migrate Copilot メインフレーム対応・Rayfin）の 4 分野を日本マイクロソフトのエキスパート 4 名が紹介。

## 概要

日本から参加する開発者および日本からのライブ配信視聴者向けに、Microsoft Build 2026 の主要アナウンスを 45 分で日本語解説する Wrap-up セッション。Satya のキーノート会場でのクロージングセッション（Japan がクロージングアクト）として開催され、現地参加 2,500 名・オンライン 150,000 名超が参加。日本マイクロソフトの各分野エキスパート 4 名が登壇し、インフラ・モデル・エージェント・セキュリティ・Windows 開発環境・クラウドプラットフォームの主要発表をカバーした。「出張報告として持ち帰れるよう凝縮する」をコンセプトに構成。

## キーポイント

1. **インフラ革新**: Maia 200・Cobalt 200 の発表によりエッジ・クラウド双方で AI 処理の効率化を推進。「Performance = Tokens per Watt per Dollar」がキーメッセージ
2. **データ基盤**: Azure HorizonDB（PostgreSQL 互換マネージド OLTP）のパブリックプレビュー、Fabric GPU アクセラレーション（NVIDIA 連携）によるクエリ性能 7 倍向上
3. **Microsoft Scout**: M365 クラウド・ローカルデータを横断検索し、経費精算などのタスクを自律実行する Autopilot 機能のデモを日本語で実演。最終判断は人間が行う安全設計
4. **フロンティアモデルの進化**: 2026 年時点で 1 時間超の人間作業を要するタスクの約 80% に正答可能。単一モデルではなく複数モデルの競争・組み合わせが重要
5. **Windows ローカル AI**: Surface RTX Spark Dev Box（128GB 統合メモリ・約 120B モデル実行可能）、Foundry Local、Windows AI APIs（音声認識・構造化出力・GPU/CPU 対応）、Windows ML、WebNN ブラウザ対応
6. **開発環境**: Windows Dev Config（定義ファイルで一括セットアップ）、WSL Containers（軽量 Linux コンテナ）、Windows 365 Developer（32 vCPU/128GB/2TB+GPU）、Intelligent Terminal
7. **GitHub Copilot 進化**: Copilot App デスクトップ版（Windows）、Canvas 拡張（リポジトリ共有型）、Automation（定時コードレビュー自動化・スマホ通知）、Copilot SDK **GA**（6 言語・Java 新規追加・Sandbox 機能）
8. **Foundry 評価・最適化**: マルチターン評価・Rubric Evaluator（エージェント定義から自動生成）・Agent Optimizer（ループ型自動最適化）・ROI for Agents（コスト可視化）。NTT DATA がプライベートプレビューで検証済み
9. **MXC（Microsoft Execution Container）**: エージェントの安全な実行環境。OpenClaw Windows・Copilot SDK Sandbox の基盤技術。OS ネイティブバイナリ＋TypeScript SDK が GitHub 公開
10. **Rayfin**: AI + Microsoft Fabric でバックエンド（認証・運用・ガバナンス）込みのプロダクションレベルアプリを実現（**パブリックプレビュー**）

## 詳細

### キーアップデート概要（岡崎）

岡崎執行役員が Build 2026 全体像を整理。テーマは「フロンティア AI やエージェントを活用するエコシステムの構築」。開発者がインフラ層・モデルコンテキストツール層・エージェント開発者ツール層を自由に組み合わせ、自社データを安全に活用してエンタープライズシステムを実装することが主題。

#### インフラ

Satya のキーメッセージ「Performance = Tokens per Watt per Dollar」を引用し、AI 利用コストが増大する中でのコスト効率バランスの重要性を強調。

- **エッジ（Arm メータード・インテリジェンス）**: GPU＋NPU の最適組み合わせにより PC 側の AI 処理を最適化。Foundry Local と Windows 側技術を組み合わせてローカル実行を実現
- **サーバー**: Maia 200（AI 処理最適化チップセット）、Cobalt 200（Arm ベース・エージェントアプリケーション向け効率実行に最適化）
- **データベース**: Azure HorizonDB（PostgreSQL 互換オープンソース DB・大規模 OLTP をマネージド提供・**パブリックプレビュー**）。エージェントがデータベースに頻繁にアクセスする時代に対応するアーキテクチャ
- **Fabric Data Warehouse**: NVIDIA アクセラレーテッドコンピューティング連携でクエリ性能 7 倍向上。アプリケーション変更不要で高速化を実現

#### モデルとエージェント

Microsoft AI 推進オフィスの Mustafa Suleyman（元 DeepMind 創設者）が発表を主導。

- **Microsoft AI モデル（7 モデル発表）**: MAI Thinking One（Microsoft 内推論ベース標準モデル）、Image 2.5（画像最適化）、Transfer Live 1.5（高精度文字認識）など。オープンベンチマークでも高水準
- **フロンティアチューニング**: 自社ドメイン知識でモデルを最適化し、企業固有の最適処理を継続進化
- **エージェント最適化**: エージェントの作り方・動作を評価し、自社知識や経験に基づき継続的に改善するシステム構築が重要。単発ではなく継続進化のループが Build 全体のメッセージ

##### Microsoft IQ プラットフォーム

- Work IQ・Public IQ・Foundry IQ に加え **Web IQ** を新発表（エージェント処理に最適化された Web グラウンディングエンジン。低コスト・高速）
- **Work IQ API GA**: 自社アプリケーションからも IQ データウェア活用可能に

##### Microsoft Scout

日本語デモを実演。シナリオ:
1. M365 クラウドとローカルデータを同時検索し、出張経費精算に必要なファイルを特定
2. ファイルを分析し経費精算書に必要な内容を抽出
3. 通常使用する経費精算アプリケーションを特定
4. **最終実行前に人間の確認を求める安全設計**
5. ファイルアップロード・経費精算完了まで自律実行

3 年前に「Autopilot は作らない」と表明していた Microsoft が、モデルの進化により自律実行可能なレベルに到達したことを強調。2026 年時点でフロンティアモデルは 1 時間超の人間作業を要する長期プロセスの約 80% に正答可能。

##### エージェント安全実行環境

- **Microsoft Execution Container（MXC）**: OpenClaw のようにファイル削除等のリスクを伴うアクション実行時、安全・安心な環境を容易に構築。Windows 環境向けに強力なエージェントハンドリングを実現
- **Hosted Agents**: サーバーサイドでエージェントをスケーラブルに安全実行するための基盤。オープンなエージェントアーキテクチャをセキュアにスケール可能

#### セキュリティ

- **Security with AI**: OpenAI＋Anthropic の 4 モデルを組み合わせ、100 以上のエージェントで統合エコシステムを構築し脆弱性検出。Cyber Gym スコア **88**（単一モデル単体実行を上回る）。AI パワードの脅威に対する懸念が増加する中、利用可能な状態で提供
- **Agent 365**: エージェント数が急増する中での組織管理基盤。今回 **Agent 365 for Local Agent** を発表し、PC 上で動作するローカルエージェント（OpenClaw 等）を企業資産として検出・リスク低減
- **Agent 365 SDK**: 任意環境で構築されたエージェントをコントロールプレーンで適切に制御

### Windows・開発者ツール（大川）

「Microsoft はなぜこんなに頻繁に Windows をアップデートするのか？」への回答として、Windows はあらゆる開発者（Windows アプリ開発者だけでなくクラウド開発者含む）の開発環境であり、進化し続けるというメッセージを冒頭に提示。

#### ローカル AI 環境

- **Surface RTX Spark Dev Box**: RTX GPU＋Arm CPU＋128GB 統合メモリ。約 120B パラメータモデルのローカル実行が想定される。プライバシー・セキュリティ・低レイテンシ等の要件でローカルモデルを活用する企業が増加
- **Foundry Local**: オープンソースモデルをローカルにダウンロードし API 経由で呼び出し。Windows 上のローカル AI 実行基盤
- **Windows AI APIs**: 音声認識、Python 構造化出力が新規追加。GPU/CPU でも利用可能に拡張。テキスト要約、顔認識等のシナリオに対応
- **Windows ML**: ハードウェア差異（Intel/AMD/Qualcomm/NVIDIA）を吸収し、自社アプリケーションからプログラム経由でローカル ML 推論を実行する環境
- **WebNN**: ブラウザ上でモデル実行が可能に

Windows 上のローカル AI スタックは 3 層構成: Windows AI APIs（典型シナリオ向け）→ Foundry Local（オープンソースモデル共通実行）→ Windows ML（独自モデル実行、NPU＋GPU 対応）

#### 開発環境

- **Windows Dev Config**: 定義ファイルベースで開発ツールを一括インストール（Winget/WSL 対応）。マシン移行時の環境構築を効率化
- **Windows 365 Developer**: 32 vCPU・128GB メモリ・2TB ストレージ＋GPU のクラウド PC。開発機器を固定資産として購入・減価償却する代わりに変動費で利用可能。クラウド PC 上で言語モデルも利用可能。ローカルデバイスからクラウド PC の作業を監視・指示する連携も実現
- **WSL Containers**: 従来の WSL より軽量な Linux コンテナ実行環境。`wsl` / `wslc` コマンドで利用
- **Intelligent Terminal**: ターミナル内にエラー解析 AI を統合。エラー発生時にコピー＆ペーストで LLM に問い合わせる手間を省き、ターミナル上部から直接解決策を提示

#### GitHub Copilot

- **Copilot App デスクトップ版（Windows）**: エンタープライズ開発に特化したデスクトップアプリケーション。リポジトリベースで左にリスト表示、チャットセッションを開いて作業を進行。CLI・VS Code・ブラウザに加わる新しいインターフェース
- **Canvas 拡張**: Copilot に「Canvas アプリを作成して」と指示し、看板アプリ等を生成。作成されたアプリはリポジトリに保存され、同じリポジトリを共有するメンバーやエージェントとデータ共有が可能。エージェントと人間のチーム形成を促進する機能
- **Automation（自動化）**: 定時にコードレビュー等のタスクを自動実行する定義を設定。クラウド上で動作し、スマートフォンアプリに通知。結果確認後に Copilot へ再割り当て→自動マージまでのワークフローを構築可能
- **Copilot SDK **GA****: 6 言語対応（Java・Go が新規追加）。クロスプラットフォーム対応＋Sandbox 機能（セッション分離・アクセス範囲定義ファイル）。ローカルマシン・クラウド PC・ネットワーク切断環境いずれでも利用可能

コーディングエージェントが「人に聞かれて支援する」から「エージェントチームが人間と協働して 1 つのアウトカムを追求する」開発スタイルへの転換を強調。GitHub Copilot が Copilot Coding Agent → Copilot CLI → Copilot SDK と進化し、あらゆる環境で利用可能になった。

### クラウド AI・エージェント基盤（新田）

#### コンテキスト・ツール

- **IQ Serverless by Foundry**: AI Search のサーバーレス版がパブリックプレビュー。従来のキャパシティサイジング型からゼロスケール・従量課金に移行。開発者ティアとして試用可能
- **MXC（Microsoft Execution Container）**: キーノートで OpenClaw Windows や Copilot SDK の Sandbox として登場した基盤技術。OS レイヤーのネイティブバイナリを実装し、そのインターフェース SDK は TypeScript で実装され GitHub で公開中。プレビュー段階

#### Foundry 評価・最適化

NTT DATA が「Smart Air Agent」イニシアティブの中でプライベートプレビュー段階から Foundry のコントロールプレーンを検証し、「スマートエアエージェント推進を加速する強力なエンタープライズグレードシステム」と評価。

- **New Evaluation Capability（パブリックプレビュー）**: 単一インタラクションだけでなくマルチターン評価に対応。エージェント定義からマルチターン会話をシミュレーション。既存トレースデータからのデータセット自動生成。全量評価が困難な場合のサンプリング評価（継続評価）機能
- **Rubric Evaluator（プライベートプレビュー）**: 従来はカスタム評価者を手動作成していたが、エージェント定義から自動的にカスタム評価基準を生成。レビューベースで容易に評価実行可能に
- **Agent Optimizer（プライベートプレビュー→パブリックプレビュー予定）**: Rubric Evaluator を活用し、閾値を設定してモデル・ツール・スキル・プロンプトをループで自動実験・最適化。機械学習のハイパーパラメータチューニング（Grid Search 等）と同じイメージ
- **ROI for Agents（プライベートプレビュー→パブリックプレビュー予定）**: トークンコスト・評価コストをユーザーのビジネス価値と紐付けて費用対効果を可視化。投資対効果の説明を支援

### クラウドプラットフォーム・データ（山本）

「この 3 つだけ覚えて帰ってください」として AI データセンター・Azure Migrate Copilot・Rayfin を紹介。

#### AI データセンター

- 過去 1.5 年で Azure 最初の 10 年間に匹敵するデータセンター容量を拡張。Copilot 等の AI ツール利用時にセッション＋コンテナが次々に起動するため、GP 搭載の大容量インフラが不可欠
- AI 専用データセンター「**Fairwater**」の建設を開始（Satya がキーノートで言及）
- 電力料金非上昇・水消費ゼロ循環の原則を維持しつつサステナブルにグローバル展開
- Microsoft はデータセンター→チップ（Maia/Cobalt）→OS→開発者ツールのフルスタックで価値を提供

#### Azure Migrate Copilot（モダナイゼーション）

8 年間の Azure 支援経験から、モダナイゼーションが「開発チーム・インフラチーム・IT チームの連携の難しさ」により停滞している現実を指摘。Azure Migrate Copilot により開発・IT チームが同一基盤でモダナイゼーションを推進可能に。

- VS Code で Azure Migrate 拡張を起動しソースコード評価（規模に応じて約 20 分で結果出力）
- クラウド移行に必要な変更箇所を即座に可視化
- **Rulebook・Command Center** によるガイド付き移行
- **メインフレーム対応**を新規発表（最大の変更点）
- 日立・NTT DATA などの日本パートナーと連携してモダナイゼーション検証実績あり

#### Rayfin

AI 時代にアプリやモックアップを作ること自体は容易になったが、プロダクションレベルへの昇格（バックエンドの認証・運用・ガバナンス構築）が依然として困難。Rayfin は AI と Microsoft Fabric の力でバックエンド込みのプロダクションレベルアプリケーションを実現する。**パブリックプレビュー**として発表。

### フロンティア領域（岡崎）

#### Microsoft Discovery

2 年前に発表された研究探索型サービスの進化。創薬・化学物質等、日本の産業の中核に AI を適用。仮説生成→反復検証→解答発見のプロセスに最適化。量子チップとの連携も一部で実現。**GA** として利用可能。

#### Majorana 2（量子コンピューティング）

信頼性をさらに向上しエラー率最小化を実現。2029 年に量子コンピュータの実用化を見据えて Microsoft が注力。
