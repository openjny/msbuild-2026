## 概要

クラウドアプリケーションのレジリエンシーは、インフラ・データ・サイバーの 3 つの柱で構成される。本セッションでは Azure Infrastructure Resiliency Manager（Public Preview）を軸に、アプリケーションのレジリエンシーライフサイクル全体（Start Resilient → Get Resilient → Stay Resilient）を Agentic AI がどう変革するかを実演した。

Resiliency Agent は Azure Copilot 上の専門ドメインエージェントとして、自然言語による意図表明からゾーン冗長 IaC テンプレートを自動生成し、GitHub への PR 作成まで一気通貫で実行する。既存アプリケーションに対しては Service Group を通じたゾーン冗長目標の設定と自動修復、Azure MCP Server を活用した VS Code 上でのデータ・サイバーレジリエンシー強化を提供する。さらに Azure Advisor AI Powered が数百件の推奨事項を環境固有の優先度で 5 件に絞り込み、Chaos Studio Workspaces が実障害のリハーサルを可能にする。

## キーポイント

1. **3 つのレジリエンシーの柱**: インフラ（ゾーン冗長・リージョンペア）、データ（バックアップ・RPO/RTO）、サイバー（ランサムウェア対策・不変バックアップ）を統合管理
2. **Azure Infrastructure Resiliency Manager（Public Preview）**: Service Group でアプリケーション単位のレジリエンシー管理を実現。目標設定、推奨事項、ドリル、リカバリープランを統合
3. **Resiliency Agent**: Azure Copilot 上で自然言語からゾーン冗長 Bicep テンプレートを生成し、GitHub PR まで作成。ブラウンフィールドアプリのゾーン冗長化も自動実行
4. **Azure Advisor via Azure MCP Server**: VS Code・Cursor 等のコードエディタから単一プロンプトで IaC テンプレート（ARM/Terraform）を Advisor カタログに照合。リタイアリング SKU・レジリエンシーギャップを一括検出・自動修正
5. **Azure Advisor AI Powered（Limited Preview）**: AI がブラスト半径・緊急度・コスト影響を分析し、数百の推奨事項を環境固有の優先度付き Top 5 アクションプランに凝縮
6. **Azure Backup via Azure MCP Server**: VS Code から Azure MCP Server 経由で未保護リソースを検出し、Vaulted Backup を設定。データ・サイバーレジリエンシーを確保
7. **Chaos Studio Workspaces（Public Preview、6/11 開始）**: ゾーンダウン・AI ワークロード障害・実障害シナリオの再現による事前検証を提供

## 詳細

### レジリエンシーの 3 つの柱とライフサイクル

Azure のレジリエンシーは以下の 3 柱で構成される:

- **インフラレジリエンシー**: 可用性ゾーン、リージョンペア、SKU リタイアメント対応。ハードウェア障害やデータセンター障害への耐性
- **データレジリエンシー**: データ整合性、可用性、RPO/RTO の確保。Azure Backup による復旧
- **サイバーレジリエンシー**: ランサムウェア攻撃からの復旧。不変バックアップ、分離されたリカバリー環境

ライフサイクルは 3 フェーズで管理:

| フェーズ | 目的 | 主なツール |
|----------|------|-----------|
| Start Resilient | Day 1 からゾーン冗長にデプロイ | Resiliency Agent、Azure Advisor via MCP、GitHub Copilot |
| Get Resilient | 既存アプリのレジリエンシーギャップを検出・修復 | Resiliency Agent、Azure Advisor AI Powered、Azure Backup via MCP |
| Stay Resilient | 継続的な検証と監視 | Chaos Studio Workspaces、Service Health、Azure Advisor |

### Azure Infrastructure Resiliency Manager（Public Preview）

Ignite で発表された Azure Business Continuity Center の拡張版。リソース単位からアプリケーション単位のビジネス継続性計画への転換を実現:

- **Service Group**: アプリケーションを構成するリソースを論理グループ化。タグ・リソースグループ・サブスクリプション等の条件で自動検出
- **ゾーン冗長目標**: Service Group に対してゾーン冗長目標を設定し、各リソースのポスチャーを評価
- **レジリエンシードリル**: Chaos Studio ベースの障害シミュレーション。特定ゾーンのダウンを再現しポスチャーを検証
- **リカバリープラン**: リソース間の依存関係を考慮した復旧ワークフローの定義。DB → VM 等の実行順序を管理

管理はAzure Portal、PowerShell、CLI、Azure Copilot から実行可能。

### Start Resilient: Resiliency Agent による IaC 生成

Resiliency Agent は Azure Copilot 上の専門ドメインエージェント。デモでは Java Payroll アプリ（VMSS + PostgreSQL Flexible Server + Key Vault）のゾーン冗長テンプレート生成を実演:

1. 自然言語で構成意図を記述（リージョン、リソース構成、モジュール構造の要件）
2. エージェントがリソースごとのゾーン冗長サポート方法を提示（例: PostgreSQL は General Purpose 以上の Tier が必要）
3. リージョンの AZ サポート状況を確認
4. モジュール構造の Bicep テンプレート一式を自動生成
5. デプロイ手順を提示
6. GitHub リポジトリへの PR 作成まで Azure Copilot 内で完結

### Start Resilient: Azure Advisor via Azure MCP Server

既存 IaC テンプレートを持つ開発者向けのワークフロー:

- Azure MCP Server を VS Code・Cursor 等に設定
- 単一プロンプトで ARM/Terraform テンプレートを Advisor の推奨カタログに照合
- 検出される問題: ゾーン冗長未設定、リタイアリング SKU、モニタリングギャップ
- カテゴリ別に分割スキャン不要 — 全信頼性リスクを一括検出
- 修正コードをインラインで提示し、レビュー・コミットまでエディタ内で完結

デモでは Zava Commerce アプリの ARM テンプレートから、Basic SKU のコンテナレジストリ（→ Premium への昇格推奨）と F シリーズ VM のリタイアメント（→ 代替 SKU 推奨）を検出・自動修正。

### Get Resilient: ブラウンフィールドアプリのゾーン冗長化

既にデプロイ済みのアプリケーションに対して、Resiliency Agent が段階的にゾーン冗長化を実行:

1. Azure Copilot で Service Group を作成（タグ条件で対象リソースを検出）
2. ゾーン冗長目標を Service Group に割り当て
3. ポスチャーチェック — 各リソースの現在の冗長性状態を確認
4. 前提条件チェック — ゾーン冗長化の実行可否とブロッカーを確認
5. Resiliency Agent が自動でゾーン冗長化を適用（例: PostgreSQL Flexible Server の HA 有効化）

### Get Resilient: Azure Backup via Azure MCP Server

データ・サイバーレジリエンシーの確保:

- VS Code から Azure MCP Server 経由で未保護の PostgreSQL Flexible Server を ARG クエリで検出
- 同一リージョンの既存 Backup Vault とポリシーを自動検出
- Vaulted Backup の Protected Item を自動作成
- ランサムウェアからの復旧とコンプライアンス（最大 10 年の長期保持）に対応

### Get Resilient: Azure Advisor AI Powered（Limited Preview）

数百の推奨事項から環境固有の優先度付きアクションプランを自動生成:

- **AI 優先度付け**: ブラスト半径、緊急度（リタイアメント期限等）、コスト影響、HA 目標、プロダクション/非プロダクションの区別をシグナルとして活用
- **Top N アクションプラン**: 数百件 → 環境固有の Top 5 に凝縮。各推奨事項に「なぜ今重要か」の理由を表示
- **完了進捗トラッキング**: 推奨事項ごとの対応済み/未対応を可視化
- **カスタムアクションプラン**: 短期（30 日以内）・中期・長期に分類。Word/PDF でエクスポート可能
- **リソースレベル優先度（Landing Soon）**: 各推奨事項内でどのリソースから着手すべきかを環境のライブシグナルに基づいて個別ランク付け
- **ワークアイテム連携（予定）**: Jira・ServiceNow への直接ワークアイテム生成

デモでは Java Ask HR アプリの 2 サブスクリプションに対し、ストレージアカウントのサービスリタイアメント（30 日以内にサポート終了）を最優先、次に Cosmos DB のゾーン冗長未設定（データ層全体への影響）を提示。

### Stay Resilient: Chaos Studio Workspaces（Public Preview、6/11）

レジリエンシーの事前検証プラットフォーム:

- **ゾーンダウンシナリオ**: AKS・SQL・Cosmos DB・Storage をまたぐ可用性ゾーン障害の End-to-End シミュレーション
- **AI ワークロード障害**: SQL・VM・Cosmos DB 等への障害注入でエージェント/RAG アプリの耐障害性を検証
- **実障害再現**: カスケード障害、DNS 障害、ゾーンブラウンアウト等の過去障害シナリオをプリビルトで提供
- **アプリケーション中心ワークスペース**: 個別リソースではなくアプリケーション単位で障害テストを管理
