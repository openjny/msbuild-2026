Azure CTO Mark Russinovich がデータセンターからセキュリティまで Azure インフラの最新イノベーションを実演。Azure Boost の 400Gbps ネットワーク、MRC プロトコル、VM 間 RDMA、ACI コンテナライブマイグレーション、Azure Context Cache、10,000 サンドボックス同時起動、Confidential VM ライブマイグレーション、Azure Integrated HSM、Project Mosaic の micro LED 光通信を公開デモ。

## 概要

Azure CTO の Mark Russinovich が、データセンターのハードウェアからサイバーセキュリティ、将来のネットワーキング技術まで、Azure インフラストラクチャ全層にわたるイノベーションをライブデモを交えて紹介する恒例セッション。Infrastructure → Core → AI → Security → Future の 5 セクション構成で、多くが未出荷または開発中の技術を含む先行公開となった。

## キーポイント

1. **Azure Boost 次世代**: ネットワーク帯域 400Gbps（前年比 2 倍）、リモートストレージ 20GB/s・100 万 IOPS、ローカル NVMe 36GB/s・660 万 IOPS を達成。フリート 30% 以上に展開済み
2. **Bare Metal GPU Instance**: OpenAI の Fairwater データセンターで稼働中。管理サービスを Azure Boost カードにオフロードし、ベアメタルで GPU に直接アクセス可能
3. **MRC（Multipath Reliable Connection）**: 10 万 GPU 規模のトレーニング向けに開発された耐障害性ネットワークプロトコル。スイッチ障害時もパケットスプレーで帯域を維持
4. **VM 間 RDMA**: Azure Boost 経由の RDMA で推論ワークロード（Prefill/Decode 分離）のトークンスループットがほぼ 2 倍に向上
5. **ACI Direct Virtualization とコンテナライブマイグレーション**: Azure Container Instances を L1 直接仮想化に移行中。コンテナのライブマイグレーションを初公開デモ
6. **Azure Context Cache（Preview）**: リモートストレージに KV キャッシュを共有し、サーバー間でプロンプトキャッシュヒット率を 96% に向上
7. **ACI サーバーレス AI サンドボックス**: メモリスナップショットにより 10,000 コンテナサンドボックスを平均 2 秒で同時起動
8. **Confidential VM ライブマイグレーション**: Intel TDX ベースの CVM を MigTD によるポリシー制御下で別サーバーに移行。世界初の公開デモ
9. **Azure Integrated HSM**: FIPS 140-3 Level 3 認定の HSM を全サーバーに内蔵。AKV からのキー転送で署名性能 640 ops/s → 18,800 ops/s
10. **Project Mosaic**: micro LED アレイとイメージングファイバーによる低消費電力・広帯域データセンター内光通信。Microsoft Research Cambridge からライブデモ

## 詳細

### Infrastructure: Azure Boost と Bare Metal

#### Fairwater データセンター

Mark Russinovich が 2025 年 11 月に訪問した Fairwater Atlanta データセンターのツアー映像を上映。液冷ケーブル配線、GPU サーバーのラッキング、ブレークフィックス作業を紹介した。1 施設内のケーブルは地球 4 周分に達する。

#### Azure Boost 最新世代

Azure Boost はストレージ・ネットワークのデータプレーンを DPU 搭載のオフロードカードに移動し、ホスト CPU をワークロードに解放する。最新世代カードは ARM コア・DPU・FPGA を統合した SoC を中心に構成される。

主要スペック:
- リモートストレージ: 20GB/s（1.4 倍）、100 万 IOPS（1.6 倍）
- ローカル NVMe: 36GB/s、660 万 IOPS
- ネットワーク: 400Gbps（2 倍）、40 万接続/秒
- デュアル ToR Active-Active で sub-second メンテナンス
- Guest RDMA サポート（レイテンシ 2.5 倍低減、CPU 使用量 2.5 倍低減）
- Confidential Device TDISP サポート

#### Bare Metal GPU Instance

管理サービスを Azure Boost カードに完全オフロードすることでベアメタルインスタンスを実現。OpenAI が Fairwater データセンターで限定利用中。デモでは ND144V6 インスタンスに SSH し、仮想化サービスが動作していないこと、PCI バス上の 8 基の Mellanox RDMA 対応 NIC を確認した。一般提供は近日予定。

### Infrastructure: ネットワーキング

#### MRC（Multipath Reliable Connection）

OpenAI と共同開発し、Intel・Broadcom・NVIDIA・AMD を含むコンソーシアムに拡大。RoCE（RDMA over Converged Ethernet）に代わり、10 万 GPU 規模の同期トレーニングジョブ向けに設計された。

特徴:
- パケットを独立したネットワークドメイン（Plane）にスプレー
- リンク障害時は残存 Plane に自動リバランス
- 2 層マルチパストポロジーで省電力

デモでは 4 スイッチ経由で 95GB/s の集約帯域を維持しつつ、1 スイッチを停止しても帯域低下なし（残り 3 スイッチに自動リバランス）を実証した。

#### VM 間 RDMA over Azure Boost

TCP/IP スタックを経由する従来方式では、メモリコピーと同期処理がボトルネックとなる。RDMA ではアプリケーションがリモートメモリに直接データを配置し、Azure Boost がトランスポートを担当する。

デモ（vLLM Prefill/Decode 分離シナリオ）:
- TCP/IP: トークンスループット ~3,400、TTFT 123,000ms
- RDMA（MANA link）: トークンスループット ~7,637（約 2.2 倍）、TTFT 47,000ms（約 62% 低減）

### Core: Azure Container Instances

#### Direct Virtualization

従来の ACI はネステッド Hypervisor 内でコンテナを隔離していたが、Direct Virtualization（L1 直接仮想化）では管理 VM と並列にコンテナポッド（L2 子 VM）を直接配置する。これにより仮想化オーバーヘッドを削減しつつ、敵対的マルチテナント隔離を維持する。フリート全体の移行には 1〜2 年を要する見込み。

ACI の採用サービス: GitHub Actions、Python in Excel、Azure AI Foundry、Copilot Tasks、Copilot Cowork、Azure OpenAI Code Interpreter、Horizon Database Serverless 等。

#### コンテナライブマイグレーション

初公開デモとして、ACI 上のコンテナを 1 サーバーから別サーバーへライブマイグレーション。1 秒間隔の tick 書き込みが途切れなく継続し、695 → 696 のギャップなしを実証した。サーバー障害予兆時のワークロード退避やホストサービシング影響の最小化に活用される。

### AI: Manifold と Azure Context Cache

#### Manifold

Microsoft 全体の推論・トレーニングを支える GPU 統合管理システム。全 GPU プールを単一プールとして扱い、優先度ベースのワークロードデプロイと AI トポロジ対応の効率的キャパシティ活用を行う。GPU・FPGA・ASIC・CPU をすべて抽象化。

Direct Virtualization 上の Manifold では、CPU コンテナと GPU コンテナを完全に隔離しつつ、GPU コンテナには物理ハードウェアへの直接アクセスを付与。デモでは Llama 4 Maverick モデルの推論を GPU ポッドから実行し、隣接する CPU ポッドからは GPU が不可視であることを確認した。

#### Azure Context Cache（Preview）

LLM 推論の KV キャッシュをリモートストレージ（Azure Storage）に共有キャッシュとして配置し、どのサーバーにリクエストが到達してもプロンプトの再 prefill を回避する。

デモ結果:
- 従来（同時実行 10、サーバー分散）: キャッシュヒット率低下
- Azure Context Cache 有効時: ヒット率 96.2%

コスト削減と TTFT 改善の両方に寄与する。

#### ACI サーバーレス AI サンドボックス

エージェントがコード実行・ファイル操作を行うためのサンドボックスを超高速に起動する仕組み。メモリスナップショットにより、同一ベースイメージのコンテナは初期化をスキップして即座にマッピングされる。

デモ: 10 ワーカー × 1,000 = 10,000 コンテナサンドボックスを平均 2 秒で同時起動。

### Security: Confidential Computing

#### Confidential Computing の概要

データ保護の 3 要素（転送中・保存時・使用中）のうち「使用中」を保護する技術。ハードウェアエンクレーブにより、Hypervisor・管理者・侵害者からもデータを隔離する。ハードウェアによる測定（Attestation）と鍵管理ポリシーを組み合わせ、コードの完全性を保証する。

Azure のポートフォリオ: Intel SGX、AMD SEV-SNP CVM、Intel TDX CVM、Confidential AKS、Confidential ACI、Managed HSM、Confidential Inferencing 等。

#### Confidential VM ライブマイグレーション

Intel と共同開発。Migration TD（MigTD）により、送信側が「承認済み MigTD を持つサーバーにのみ移行を許可」するポリシーを強制し、信頼境界をサーバー間に拡張する。

世界初の公開デモとして、Hyper-V Manager 上の CVM を別サーバーへライブマイグレーション。TD info attestation の検証を経て移行が完了し、RDP 接続が切り替わることで移行を確認した。数週間前に初めて達成された成果であり、未出荷技術。

#### Azure Integrated HSM

全 Azure サーバーに FIPS 140-3 Level 3 認定の HSM を組み込み、鍵がハードウェア信頼境界を離れることなくライフサイクル全体を保護する。Azure Key Vault / Managed HSM からポリシーベースで鍵を転送し、VM から NCrypt（Windows）/ OpenSSL（Linux）の標準 API で直接利用可能。

デモ（5,000 ペイロード署名、32 スレッド）:
- AKV リモート署名: ~640 ops/s
- Azure Integrated HSM ローカル署名: ~18,800 ops/s（約 29 倍）

AMD D/E v7 シリーズ VM で GA 予定（Coming soon）。

### Future: Project Mosaic

データセンター内の銅線は最大 2m、レーザー光学は高消費電力が課題。Project Mosaic は「多チャネル・低速度」アプローチで、micro LED アレイ → イメージングファイバー（数千コア/本）→ シリコンカメラセンサーアレイの構成により、VR ゴーグルやカメラ向けに量産される民生技術を活用する。

Microsoft Research Cambridge からのライブデモでは、数百の micro LED を個別制御し、10m 以上のイメージングファイバー経由で受信側スクリーンにパターンを表示。各 micro LED が Gbps 級で変調可能であることを実証した。
