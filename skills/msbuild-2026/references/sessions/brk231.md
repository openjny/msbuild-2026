# Deploy. Observe. Learn. Reinforcement learning for production agents

Microsoft Foundry のファインチューニング・強化学習パイプラインを活用し、本番エージェントのトレースから小型モデルを蒸留・RFT で最適化する手法を4段階のデモで実証。SFT による蒸留、Reinforcement Fine-Tuning、新 Interactive Training API（PyTorch as a Service）、自然言語によるファインチューニングスキルを紹介し、コスト・レイテンシ・精度の同時改善を実現する方法論を解説。

## 概要

エージェントは通常のチャットと比較して約 25 倍のトークンを消費し、フロンティアモデルをそのまま使い続けるとコストが破綻する。本セッションでは Microsoft Foundry 上でファインチューニングと強化学習を段階的に活用し、本番エージェントを「より安く・より速く・より賢く」する 4 段階のアプローチを実演した。

小売返品処理の顧客サービスエージェントを一貫したシナリオとして使用し、(1) SFT による蒸留で GPT-4.1 mini が GPT-5.4 を上回る精度を達成、(2) RFT で o4-mini が 84% の精度を達成、(3) Interactive Training API で Qwen3 32B がさらに上回り、(4) 自然言語のファインチューニングスキルで全プロセスを自動化する流れを示した。実運用事例として Decagon AI、Discovery Bank、Docusign の成果も共有された。

## キーポイント

1. **エージェントタスクはチャットの約 25 倍のトークンを消費し、ファインチューニングによる小型モデル化がエージェント経済性の鍵**
2. **本番トレースからワンクリックで訓練データセットを生成し、SFT で蒸留可能（重複除去・PII 除去を自動実行）**
3. **Reinforcement Fine-Tuning（RFT）は教師モデルの性能上限を突破し、モデル自身のロールアウトから学習**
4. **新 Interactive Training API は「PyTorch as a Service」として GPU インフラをマネージドで提供し、GRPO ループの完全制御が可能（**Preview**）**
5. **ファインチューニングスキルにより、コーディングエージェントと自然言語でモデル最適化の全プロセスを自動化**
6. **SFT ジョブの中央値コストは約 1 ドル、Developer Tier で 50% オフ、Developer Tier Hosting はホスティング費用なし**
7. **MAI-2-5B Frontier Tuned は GPT-5.5 と同等精度で 10 倍以上のコスト効率を実現**

## 詳細

### ファインチューニングの動機: エージェント時代のコスト課題

エージェントの普及によりトークン消費量が急増している:

- 2028 年までにエンタープライズソフトウェアの 51% がエージェント AI を組み込む見込み（2024 年の 1% 未満から急増、Gartner 調査）
- 33% のチームが既にエージェントを本番運用（LangChain — State of AI Agents 2024）
- エージェントタスクはチャットターンの約 25 倍のトークンを消費（ツールコール + 反復ループ、繰り返しコンテキスト）

ファインチューニングの 3 つの柱:

| 目標 | 効果 |
|------|------|
| 品質向上 | ツールの正確な呼び出し、フォーマット、エッジケースをモデルの重みに焼き込む |
| コスト削減 | 小型チューニング済みモデルはトークンあたり 10〜30 倍安価 |
| レイテンシ削減 | 小型モデルはトークンストリーミングが 3〜10 倍高速、推論トークンも 10% に削減可能 |

Insight Partners の 2024 年調査では、エンタープライズ技術リーダーの 48% がファインチューニングを採用し、既製モデル利用の 29% を上回っている。

### Foundry における最適化の位置づけ

Foundry のエージェント最適化は段階的なジャーニーとして整理される:

1. **プロンプティング** — モデルへの指示で挙動を制御
2. **コンテキスト管理** — データ、グラウンディング、メモリの追加
3. **ツール管理** — ツール呼び出し指示、命名、ルーティング
4. **ファインチューニング** — SFT、RFT、RL によるモデル重みの最適化

### デモ 1: 蒸留による低コスト・高速エージェント

顧客サービスエージェント（小売返品処理）のシナリオを使用。9,000 億ドル規模の米国オンライン返品市場で、返品判断の精度が直接的にビジネスインパクトに直結する設定。

**エージェントの構成:**

- GPT-5.4 を教師モデルとして Foundry Hosted Agents で稼働
- ツール: 注文詳細取得、配送状況確認、返品ポリシー確認、支払い処理
- トレース（軌跡）が自動的に蓄積され、評価・ファインチューニングに利用可能

**評価グレーダーの設計:**

Python グレーダーで 3 軸を採点:
- 判断の正確性: 返金すべきか否かの決定（50% ウェイト）
- 金額の精度: 正しい返金額（30% ウェイト）
- 出力フォーマット: ダウンストリームシステムとの互換性（20% ウェイト）

加えて Foundry 組み込みの LLM グレーダー（intent resolution、task completion）でベースモデルの能力劣化を監視。

**蒸留の手順:**

1. 約 1,000 件のトレースを蓄積
2. 「Create Dataset」でトレースを SFT 用データセットに変換（重複除去・PII 除去を自動実行）
3. GPT-4.1 mini を対象に SFT ジョブを実行
4. Training Tier の選択肢: Standard / Developer Preview（50% オフ、低優先度 VM）/ Data Zone（米国リージョンのデータレジデンシー保証、**Build で発表**）

**結果:**

| モデル | 品質スコア |
|--------|-----------|
| GPT-5.4（教師） | ~65% |
| o4-mini（ベース） | ~65% |
| GPT-4.1 mini（ベース） | 低い |
| GPT-4.1 nano（ベース） | 低い |
| **GPT-4.1 mini（SFT）** | **~74%（新リーダー）** |

GPT-4.1 mini ファインチューニング版が o4-mini および教師モデル GPT-5.4 を上回り、かつコストは数分の一。intent resolution と task completion の組み込みグレーダーでベースモデルの能力劣化がないことも確認。

### デモ 2: Reinforcement Fine-Tuning（RFT）

蒸留には教師モデルの性能という上限がある。RFT はモデル自身のロールアウトから学習するため、教師を超える性能が実現可能。

**RFT の仕組み:**

1. プロンプトをモデルに送信
2. モデルが複数のサンプル（ロールアウト）を生成
3. グレーダーが各サンプルをスコアリング（0〜1）
4. トレーナーがスコアに基づき、最良の Chain of Thought を生成するようモデルの重みを更新

**SFT との違い:**

| 項目 | SFT（蒸留） | RFT |
|------|------------|-----|
| 学習方法 | 教師の出力をコピー | 自身のロールアウトから学習 |
| 上限 | 教師モデルの品質に制約 | リワード関数が目標を定義 |
| 訓練データ | 完全な軌跡（ツール呼び出し履歴含む） | ユーザーメッセージ + グラウンドトゥルスのみ |
| ツール | 不要 | 訓練中にリアルタイムで呼び出し |

**実装のポイント:**

- ツールは Azure Functions にホスト（MCP サーバーも利用可能）
- 訓練中にモデルがリアルタイムでツールを呼び出し、応答を取得して推論
- グレーダーにツール呼び出しカバレッジを含め、リワードハッキングを防止（例: 「get orders」を呼ばないロールアウトにペナルティ）
- 推論モデル（o4-mini）の reasoning effort を low/medium/high で設定可能

**モニタリング指標:**

- rewards（訓練/検証）: 時間とともに増加
- reasoning token mean: 訓練が進むにつれ減少（最終モデルはベースより少ないトークンで推論）
- tool calls per rollout: ツール呼び出し回避の検出に有用

**結果:** o4-mini RFT 版が 84% の精度を達成（ベースラインから約 11 ポイント向上）。コストも GPT-5.4 より大幅に低い。

### デモ 3: Interactive Training API（Preview）

完全なアルゴリズム制御を求めるデータサイエンティスト向けの低レベル訓練 API。「PyTorch as a Service」のコンセプトで、GPU インフラのみをマネージドで提供。

**Managed RFT との比較:**

| 項目 | Managed RFT（Path A） | Interactive Training API（Path B） |
|------|----------------------|----------------------------------|
| ループ | クローズドループ（設定後は自動反復） | オープンループ（実践者が各ステップを制御） |
| グレーダー | サービス側で実行 | 実践者側で実行（言語・フレームワーク自由） |
| 制御 | 限定的なハイパーパラメータ | 完全なアルゴリズム制御 |
| インフラ | マネージド | GPU のみマネージド、オーケストレーションはローカル |

**7 ステップ GRPO ループ:**

1. `sample()` — Sampling Node でロールアウト生成（N completions per prompt）
2. グレーダーでスコアリング（クライアント側、カスタム実装）
3. advantages 計算（グループ内の平均との差分）
4. `forward_backward()` — Training Node で勾配計算（importance sampling）
5. `optim_step()` — 最適化ステップ
6. `sync_weights()` — LoRA アダプタを Sampling Node に同期
7. `save_checkpoint()` — Azure Blob にチェックポイント保存

**アーキテクチャ:**

- クライアント（CPU、ローカル/Azure VM）: オーケストレーションコード、レシピ、グレーダー
- Azure GPU クラスター: Sampling Node（ベース LLM + LoRA、トークン生成）+ Training Node（勾配計算、最適化）
- デモでは Qwen3 32B + LoRA rank=32 を使用

**高度な機能:**

- カスタムロールアウト環境（シミュレーター、ツールサーバー、マルチターン環境）
- カリキュラム学習（簡単なタスクから段階的に難度を上げる）
- アルゴリズム変更（GRPO → PPO、DPO 等）
- 訓練中のサンプリング戦略変更
- ローカルダッシュボードで詳細テレメトリを可視化（KL ダイバージェンス、エントロピー、勾配ノルム、グループ構成）

**結果:** Qwen3 32B が Interactive Training API による RFT で o4-mini RFT 版をさらに上回る精度を達成。より安価・小型の OSS モデルで最高性能。

### デモ 4: ファインチューニングスキル

データサイエンスの専門知識がなくても、自然言語でファインチューニングの全プロセスを実行可能にするコーディングエージェントスキル。

**利用方法:**

- GitHub Copilot for Azure スキルの一部として提供
- スタンドアロンのファインチューニングスキルとしても利用可能
- Copilot CLI 等のコーディングエージェントから自然言語で指示

**実演フロー:**

1. 「ホストされたエージェントのグレーダーを作成し、ベースライン評価を実行」と指示
2. スキルがグレーダーを自動生成、教師モデル（GPT-5.4）の評価を実行（mean score 8.37、pass rate 78%）
3. 「cheaper and faster を目標に蒸留ジョブを実行」と指示
4. スキルがトレースのエクスポート、フィルタ、重複除去、autopilot 実行（モデル選定・ハイパーパラメータ設定を自動化）
5. ベースモデル評価とファインチューニングジョブを並列実行
6. GPT-4.1 mini ファインチューニング版が教師モデルと同等以上のスコアを達成
7. 結果が悪い場合はログを分析し、データ生成・パラメータ変更を自動反復

### 実運用事例

| 企業 | 手法 | 成果 |
|------|------|------|
| Decagon AI | 蒸留 | 小型モデルで大規模 SOTA モデルを超える性能、大幅な高速化 |
| Discovery Bank | 蒸留 | レスポンス時間を 5〜6 秒から 1.5〜2 秒に短縮（50% のレイテンシ削減） |
| Docusign | 蒸留 | AI 文書処理コスト 50% 以上削減、スループット 8 倍拡大、精度差 2 ポイント未満 |

### コスト分析: ファインチューニングの経済性

**精度の神話を覆す:**

- MAI-2-5B Frontier Tuned は GPT-5.5 と同等の Weighted Pass Rate（72.1% vs 72.2%）を達成
- GPT-5.4（68.8%）や GPT-5（71.9%）を上回る

**コスト効率:**

- MAI-2-5B Frontier Tuned は GPT-5.5 比で 10 倍以上の Output Tokens per Dollar
- GPT-5 は約 3 倍、GPT-5.4 は約 2 倍

**年間コストシミュレーション（10M TPM 利用時）:**

- GPT-5.5: 約 6,000 万ドル/年
- ファインチューニング済み GPT-4.1 nano: 約 110 万ドル/年
- **年間約 5,400 万ドルの削減**

**訓練コスト:**

- SFT ジョブの中央値: 約 1 ドル
- Developer Tier: 50% オフ（低優先度 VM）
- Developer Tier Hosting: ホスティング費用なし
