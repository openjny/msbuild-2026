## 概要

Azure DevOps と GitHub を統合し、AI 駆動の SDLC を実現するための新機能群を紹介するセッション。前半は Dave Burnison が Azure DevOps Remote MCP Server を活用した GitHub Copilot App/CLI からのワークアイテム操作、セキュリティキャンペーン連携、Copilot Cloud Agent によるコード生成を実演。後半は Dan Hellem が Enterprise Live Migrator（レポジトリ移行ツール）、Azure Repos 向け Copilot Code Review、GHAzDO Autofix を Technical Preview として発表した。

メッセージ: 「GitHub に移行してもしなくても、AI の恩恵を受けられるようにする」。移行する企業にはスムーズなパスを、Azure Repos に残る企業には Copilot Code Review を提供。

## キーポイント

1. **Azure DevOps Remote MCP Server**: GitHub Copilot App/CLI から Azure Boards・Pipelines・Test Plans・Wiki にアクセス。Service Principal / Managed Identity 対応（**GA**）
2. **Copilot Cloud Agent + Azure Boards 統合**: ワークアイテムを Copilot に割り当て、コード生成→PR→CodeQL スキャン→Code Review を自動実行
3. **Enterprise Live Migrator (ELM)**: Azure Repos → GitHub へのフルフィデリティ移行（PR・コミット履歴・ブランチすべて保持）。ダウンタイム 20-30 分。Pipelines・Boards の接続も自動（**Technical Preview**）
4. **Copilot Code Review for Azure Repos**: GitHub アカウント不要。Azure サブスクリプション課金。動的パイプラインで実行（**Technical Preview**）
5. **GHAzDO Autofix**: GitHub Advanced Security for Azure DevOps の CodeQL アラートに対する自動修正（**Technical Preview**）
6. **Apple Mac Pay-per-minute billing**: Azure Pipelines の Microsoft ホステッドエージェントで Mac の従量課金（**Technical Preview**）

## 詳細

### Azure DevOps MCP Server と Copilot 統合（Dave Burnison）

Azure DevOps Remote MCP Server を GitHub Copilot App に接続するデモ:

- **設定**: Copilot App の MCP servers 設定にサーバー名・組織 URL・使用ツール（work items, pipelines, test plans, wiki）を指定
- **ワークアイテム操作**: 「アサインされた新規/アクティブなユーザーストーリーを表示」→ 詳細確認 → 品質基準に基づく自動リファインメント
- **Wiki → Copilot Instructions 変換**: Wiki ページの品質基準をカスタムエージェント（backlog manager）とカスタム Instructions ファイルに自動変換
- **セキュリティキャンペーン連携**: GitHub Advanced Security のキャンペーンから Azure Boards のフィーチャー・ユーザーストーリーを自動生成。アサイン情報も連動
- **Copilot CLI**: 同じ MCP 設定が CLI でも有効。コマンドラインからワークアイテム・PR ステータスを自然言語で照会

### Copilot Cloud Agent によるワークアイテム駆動開発

- ワークアイテムを「Copilot にアサイン」→ エージェントがコード生成 → PR 作成
- PR に対して自動で **Copilot Code Review** と **CodeQL** スキャンが実行
- CodeQL は Copilot ライセンス不要でエージェント生成コードに対して動作
- カスタムエージェント選択 + モデル選択（モデルは sticky で保持される）

### MCP Server の構成（Dan Hellem）

| 項目 | Local MCP | Remote MCP |
|------|-----------|------------|
| ツール | Work Items, Repos, Wiki, Test Plans | 同左 |
| 認証 | ユーザー認証 | Service Principal, Managed Identity |
| 対応クライアント | VS Code, GitHub Copilot | GitHub Copilot App/CLI, Foundry |
| 制限 | なし | Entra 制約により Claude/Cursor/Codex は未対応 |

Remote MCP は Entra の制約で一部クライアントが未対応のため、Local MCP も並行維持。Copilot Studio 対応は「数週間以内」に予定。

### Enterprise Live Migrator — ELM（Dan Hellem）

Azure Repos → GitHub Enterprise Cloud（Data Residency 対応）へのフルフィデリティ移行ツール:

- **移行対象**: リポジトリ全体（コミット履歴、PR、ブランチ、すべて）
- **プロセス**: Queue → Validation → Synchronization（継続同期）→ Cutover（20-30 分 read-only）
- **自動接続**: 移行後に Pipelines と Boards を自動接続
- **既存ツールとの違い**: GitHub Enterprise Importer では 4-5 日のダウンタイムが発生する大規模リポジトリでも、ELM なら 20-30 分
- **インターフェース**: CLI + UX（プレビュー）+ MCP Server からのエージェント操作
- **スケール課題**: 顧客から 30,000～80,000 リポジトリの接続要求あり。継続的に改善中

### Copilot Code Review for Azure Repos（Dan Hellem）

GitHub アカウント不要で Azure Repos の PR に Copilot Code Review を提供:

- **仕組み**: 動的パイプラインが shallow clone → diff を Copilot に送信 → 提案を PR に付与
- **課金**: Azure サブスクリプション課金。1 実行あたり数セント
- **制御レベル**: Organization レベル有効化 → Repository レベル有効化（2 段階）
- **現時点の制限**: Copilot Instructions 未対応（GA 前に対応予定）、前回の提案をコンテキストに含まない（改善予定）
- **UI**: PR 上の「Request」ボタンでオンデマンド実行。提案には「Apply changes」ボタン

### GHAzDO Autofix（Dan Hellem）

GitHub Advanced Security for Azure DevOps の CodeQL アラートを自動修正:

- Repository 設定で Autofix を有効化
- 「Generate Fix」ボタンで修正 PR を自動生成（3-5 分）
- セキュリティキャンペーンへの Autofix 適用も予定
- 将来的にはサードパーティツールのアラートにも拡張予定
