## 概要

Rayfin は、AI コーディングエージェント時代のためのアプリケーションバックエンド基盤である。本セッションでは、Rayfin SDK/CLI を使ってバックエンド全体（データベース、関数、認証、アクセスポリシー）をコードで定義し、`rayfin up` コマンド一つで Microsoft Fabric にデプロイするワークフローをデモを交えて紹介した。

エージェンティックコーディングにより「バイブコーディング」でアプリを素早く構築できるようになった一方、エンタープライズ環境ではデータベース構築、認証設定、コンプライアンス対応などのバックエンド課題が依然として残る。Rayfin はこのギャップを埋め、Fabric のセキュリティ・ガバナンス基盤を活用して初日からプロダクションレディなアプリを提供する。

また、Power BI チームと共同開発したデータアプリテンプレート、Replit との統合パートナーシップ、ランタイムのオープンソース化計画についても発表された。

## キーポイント

1. **Rayfin SDK/CLI**: TypeScript でデータモデル・アクセスポリシー・関数をコードとして定義し、`rayfin up` で Fabric に直接デプロイ（**Preview**）
2. **エンティティデコレータによるスキーマ定義**: クラスデコレータでテーブル・カラム・制約・リレーションシップを宣言的に記述。スキーマ変更時のマイグレーションも自動処理
3. **ロールベースアクセスポリシー**: コード内でロールデコレータとポリシー関数を使い、行レベルセキュリティを定義
4. **Fabric Functions**: TypeScript で記述したバックエンドロジックがデプロイ時に Fabric のセキュアサンドボックスで実行される関数に変換
5. **コネクタ**: OneLake 内の既存データ（データベース、ウェアハウス、セマンティックモデル）への接続・読み書きが可能
6. **データアプリテンプレート**: Power BI チームと共同開発。React + Vega Lite コンポーネントによるエンタープライズグレードのビジュアル、セマンティックモデルへの直接クエリ
7. **Replit 連携**: 非技術ユーザーがブラウザベース IDE でアプリを構築し、Fabric にデプロイ（**Private Beta**）
8. **セルフホステッドランタイム**: Rayfin ランタイムの一部をオープンソース化予定。コミュニティと協力して多様なインフラ上での実行を目指す

## 詳細

### エージェンティックコーディング時代のバックエンド課題（Ben Zulauf）

エージェンティックコーディングにより、ソフトウェアエンジニアでなくてもアプリを構築できる時代が到来した。しかしエンタープライズ環境では以下の課題が残る:

- **開発者側**: データベース構築、認証設定、バックエンドサービスの構成、既存データへの安全な接続、コンプライアンス遵守
- **IT管理者側**: データアクセス権限管理、セキュリティ・コンプライアンスの維持、クラウドコスト管理、監査対応

Rayfin はこれらを解決し、エージェントと人間の両方がバックエンドをコードで定義・管理できるようにする。

### Rayfin の基本ワークフロー（デモ #1）

Ben による配送アプリ（Zaba 社のホームデリバリーサービス）構築デモ:

1. `rayfin create` でプロジェクトスキャフォールディング（テンプレート選択、SDK・CLI・エージェント指示を含むファイル生成）
2. `rayfin up` で Fabric にリソース（データベース・アプリ）をプロビジョニング
3. VS Code の Copilot にアプリ要件を伝え、エージェントがフロントエンド＋バックエンドを生成
4. 生成されたアプリは Fabric ワークスペース上にアーティファクトとして管理
5. アプリデータは Fabric SQL Database に格納され、ワークスペースから確認可能

### コードによるバックエンド定義（Sachin Patney）

Rayfin の TypeScript SDK でバックエンドを宣言的に定義:

- **エンティティデコレータ (`@Entity`)**: クラスが Fabric SQL Database のテーブルにマッピング
- **プロパティデコレータ**: カラム型、min/max 制約をデータベース制約に変換
- **ロールデコレータ (`@Role`)**: ポリシー関数で行レベルセキュリティを実現（例: 認証済みユーザーが自分の配送のみ閲覧可能）
- **自動マイグレーション**: スキーマ変更を検出し、マイグレーションスクリプトを自動生成・実行

### 構成ファイル（rayfin.yml）

YAML 設定で以下を制御:

- **データベースダイアレクト**: 現在は Microsoft SQL。将来 Postgres もサポート予定（アプリコードの変更不要）
- **フロントエンド/バックエンド分離デプロイ**: 複数アプリで同一バックエンドを共有可能。スコープでアプリごとの読み書き権限を制御
- **認証オプション**: 現在は Fabric (Entra) 認証必須。将来は匿名アクセス、ソーシャルログイン（Gmail 等）、OIDC にも対応予定

### コネクタによる既存データ接続

- OneLake 内の既存データソース（データベース、ウェアハウス、セマンティックモデル）に接続
- エージェント指示により必要なテーブルのスキーマを取得し、型安全なクラスを自動生成
- 必要なテーブルのみ選択的にインポート可能

### Fabric Functions

- TypeScript で記述したバックエンドロジック（通知送信、外部サービス接続等）
- `rayfin up` 時に Fabric 上のセキュアサンドボックスにデプロイ
- アプリからは通常の関数呼び出しのように利用可能
- 認証情報を安全に注入できるサーバーサイド実行

### テンプレートシステム

テンプレートは以下を含むパッケージ:

- エージェント指示（AGENTS.md）
- MCP サーバー
- サンプルコード
- ライブラリ
- 設定ファイル（rayfin.yml）

組織固有のテンプレートをリポジトリで共有可能。CLI または Fabric UX からテンプレートを選択してプロジェクトを開始できる。

### データアプリテンプレート（デモ #2）

Power BI チームと共同開発したデータアプリテンプレートの構成:

- **データパッケージ**: Fabric エンドポイント接続の正規化、クエリリトライ、キャッシュ
- **ビジュアルパッケージ**: React + Vega Lite コンポーネント。Power BI チームのデータビジュアライゼーション経験に基づくエンタープライズグレードのデフォルト設定
- **エージェントスキル**: コーディングエージェントにパッケージの効果的な使い方を教示。デザインの質（good taste）を確保

デモでは Fabric ポータルから新規アプリアーティファクトを作成し、セマンティックモデル上に顧客満足度分析ダッシュボードを構築。アクションボタン（イシューのエスカレーション等）を含むインタラクティブな分析アプリを数プロンプトで生成した。

### Replit 連携（Carl）

- Replit はブラウザベースのエージェンティック IDE で、非技術・半技術ユーザー向け
- Replit 上で構築したアプリを Rayfin 経由で Fabric にデプロイ
- Fabric のセキュリティ・ガバナンスプロパティを継承
- **Private Beta** として提供中
- Leatherman 社がテストユーザーとして「高速な開発と Fabric のデータガバナンスの両立」を評価
- Satya との基調講演で前日にパートナーシップを発表済み

### Fabric がアプリ基盤に適する理由

- すべてのアーティファクトに既存のセキュリティ・コンプライアンスルールが適用
- アプリデータも OneLake に格納 → ビジネスユーザーがデータサイエンス・分析に即座にアクセス可能
- SaaS オファリングとしてインフラ管理が不要
- 単一の Fabric Capacity で分析ワークロードとアプリワークロードの両方を課金
- ワークスペース間移動により dev → staging → production のデプロイフローを構築可能
- Fortune 500 の 90% 以上が利用。毎日 370 億インタラクションを処理
- HIPAA、SOC 2、FedRAMP High、PCI DSS、GDPR、ISO 27001、HITRUST 認証取得

### ローカル開発環境

- ユーザーコード（関数、Web アプリ）はローカルで開発
- バックエンドサービスは 2 つの選択肢:
  - Fabric dev ワークスペースに接続
  - ローカル Docker コンテナ（**Experimental**）

### ロードマップ

| 時期 | 機能 |
|------|------|
| 現在 | データベース、Enterprise Auth (Entra)、Fabric アプリ・DB 管理、Fabric データへの接続 |
| Coming Soon | Functions、RBAC、追加データコネクタ、Observability、Free Tier |
| 年内 | OIDC 認証、Blob Storage、リアルタイムサービス、エージェンティックアプリ、ソーシャルログイン |

### オープンソース化

Rayfin ランタイムの一部をオープンソースとして公開予定。コミュニティと協力し、Fabric 外の多様なインフラ上でもセルフホステッドアプリを実行可能にすることを目指す。
