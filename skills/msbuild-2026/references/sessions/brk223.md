## 概要

AI アプリケーションとエージェントには、トランザクション処理だけでなく推論に最適化されたデータプラットフォームが必要である。従来のアーキテクチャではデータシステムを個別に接続する必要があり、レイテンシと複雑性を増大させていた。本セッションでは Azure の 3 つの主要データベース—Azure Cosmos DB、Azure SQL Database Hyperscale、Azure HorizonDB—における最新イノベーションをデモ中心に紹介した。

各データベースで「AI をデータベース内部に統合する」というテーマを共通に掲げ、Cosmos DB ではエージェントメモリ管理、SQL ではインデータベース AI とローカル/クラウド同一アーキテクチャ、HorizonDB では AI パイプラインによるデータインジェストからハイブリッド検索までの一気通貫を実演。すべてのデモコードは GitHub リポジトリ（aka.ms/build26/BRK223）で公開されている。

## キーポイント

1. **Azure Cosmos DB Agent Memory Toolkit**: Python SDK によるエージェントメモリの CRUD・セマンティック検索。短期メモリをローカルに保持し、Cosmos DB に永続化。Change Feed + Azure Durable Functions + Microsoft Foundry モデルで自動的に長期メモリ（要約・ファクト抽出・ユーザープロファイル）へ変換
2. **Azure SQL Database コンテナ（Private Preview）**: ローカルで Azure SQL と同等の動作をするコンテナイメージ。x64/ARM 対応、VS Code・Docker・Apple Container Engine・WSL 統合。ローカル開発から Azure Hyperscale への移行がコード変更最小限で可能
3. **インデータベース AI（Azure SQL）**: sp_invoke による外部モデル呼び出し、ベクトルハイブリッド検索（DiskANN + JSON Index Seek）、Ledger による不変ログ、Data API Builder による REST/MCP エンドポイント自動生成をすべて SQL 内で完結
4. **Azure HorizonDB（Public Preview）**: PostgreSQL 完全互換のクラウドネイティブサービス。コンピュートとストレージを分離し、ゾーン耐障害性ストレージ最大 128TB、セルフマネージド比 3 倍のトランザクション/ベクトル検索性能
5. **AI Model Management（HorizonDB）**: プロビジョニング時に Microsoft Foundry モデル（Embedding・GPT・Reranking）を自動登録。SQL から AI Functions で直接呼び出し可能
6. **AI Pipelines（HorizonDB）**: PG Durable 拡張によるデータベース内非同期バックグラウンドジョブ。チャンキング→エンベディング生成→格納を外部オーケストレーター不要で実現。チェックポイント・リトライ・スロットリング内蔵
7. **VS Code PostgreSQL 拡張**: ビジュアルクエリプラン表示、AI パイプラインデバッグ UI、リアルタイム監視ダッシュボード、各ノードから GitHub Copilot エージェントモードへの直接デバッグ連携
8. **Microsoft の Postgres 貢献**: Postgres 19 に対し全コミットの約 11%（345 コミット）、約 64,000 行（8%）を貢献。非同期 IO・クエリエンジン・オブザーバビリティが主な領域

## 詳細

### Azure Cosmos DB: Agent Memory Toolkit（James Codella）

エージェントが過去のコンテキストを保持し最適な判断を行うためのメモリ管理ソリューション。Python SDK（`CosmosMemoryClient` / `AsyncCosmosMemoryClient`）として提供される。

アーキテクチャ:
- **短期メモリ**: クライアントローカルに保持（`add_local` / `get_local`）。低レイテンシアクセス
- **永続化**: `push_to_cosmos` で Cosmos DB NoSQL に保存。セマンティック検索・ハイブリッド検索が可能
- **自動処理**: Cosmos DB Change Feed → Azure Durable Functions が起動し、Microsoft Foundry の LLM/Embedding モデルを使って短期メモリを長期メモリへ変換
  - スレッド要約
  - エピソード・手続き・ファクトメモリの抽出
  - ユーザープロファイル構築
  - エンベディング生成

デモでは Jupyter Notebook 上でカスタマーサポートエージェントのシナリオを実演。短期メモリの追加→Cosmos DB への永続化→メモリ処理（要約・ファクト抽出）→セマンティック検索の一連のフローを示した。

### Azure SQL Database: Zava Live Site Support（Bob Ward）

架空の企業「Zava」のオンコールエンジニアが直面する課題—運用データ・ナレッジベース・ライブシステム状態・AI が別々のシステムに存在する—を、すべてデータベース内に統合するアーキテクチャを構築。

**ローカルアーキテクチャ（Azure SQL コンテナ）**:
- Blazor WASM + .NET Aspire フロントエンド
- Azure SQL コンテナ（ローカル）: ベクトル埋め込み（1024 次元）、JSON、DiskANN インデックス
- コンテナ内サイドカー: Caddy + Ollama（mxbai-embed-large / phi4-mini）
- Data API Builder: REST + MCP エンドポイント（ゼロコード）
- VS Code Copilot エージェント: 6 つの MCP ツールでインシデント対応を自動化

**Azure アーキテクチャ（Hyperscale）**:
- Azure SQL Hyperscale（システム MI 認証）
- Azure OpenAI: text-embedding-3-small（1536 次元）+ gpt-5-4-mini
- Azure API Management: GenAI ゲートウェイ（MI 認証・Content Safety・TPM 制限）
- Azure Container App 上の Data API Builder
- Azure Static Web App フロントエンド
- **同一の T-SQL コード・ストアドプロシージャ本体**: ローカル↔Azure で変更不要

デモのハイライト:
- MSSQL 拡張 Schema Designer での GitHub Copilot 連携
- T-SQL 正規表現によるインシデントの非構造化テキスト解析→JSON 格納
- ベクトルハイブリッド検索（`TOP APPROXIMATE` + `VECTOR_DISTANCE` + JSON Index Seek）
- Copilot エージェントによるインシデント緩和計画の自動生成（MCP 経由で DMV 参照・インデックス存在確認・リソース圧力診断→`sp_invoke` で LLM 呼び出し→緩和計画＋信頼度スコア出力）

### Azure HorizonDB: AI ネイティブアプリケーション構築（Charles Feddersen & Abe Omorogbe）

#### アーキテクチャと性能

HorizonDB は PostgreSQL 互換のコンピュート層と、新設計のストレージ層（WAL ストレージフリート + データストレージフリート + Azure Blob Storage）から構成される。ストレージはデフォルトでアベイラビリティゾーン間にストライプされ、1 ミリ秒未満のマルチゾーンコミットレイテンシを実現。

TPC ベンチマークデモ:
- セルフマネージド PostgreSQL（Patroni、ゾーン HA）: 約 4,000 TPS
- HorizonDB（同一ハードウェア・同一リージョン）: 約 11,500–12,000 TPS

#### AI Model Management

プロビジョニング時に Microsoft Foundry から以下のモデルを自動登録:
- Embedding モデル（text-embedding-3-small）
- GPT モデル（gpt-5-4-mini）
- Semantic Ranking モデル（Cohere）

SQL から `ai.create_embeddings()`、`ai.generate()`、`ai.rank()` 等の AI Functions で直接呼び出し。

#### AI Pipelines（PG Durable）

```sql
SELECT ai.create_pipeline(
    name    => 'product_rag_pipeline',
    source  => ai.table_source('product_sample'),
    steps   => ARRAY[
        ai.chunk(input => 'content',
                 chunk_size => 1024,
                 overlap => 128),
        ai.embed(model => 'text-embedding-3-small',
                input_column => 'chunk_text',
                dimensions => 1536,
                batch_size => 10)
    ],
    trigger => 'on_change',
    sink    => ai.table_sink('product_rag_pipeline_output')
);
```

特徴:
- テーブルへの INSERT がミリ秒で完了（OLTP スループット維持）
- 変更閾値（batch_size）到達後に非同期でエンベディング生成
- チェックポイント・リトライ・スロットリング・クラッシュリカバリ内蔵
- PG Durable はオープンソース（github.com/microsoft/pg_durable）

#### ハイブリッド検索（AI Search）

DiskANN ベクトルインデックス + PostgreSQL Full Text Search（PG FTS）を組み合わせたハイブリッド検索。AI Search 関数は 2026 年 6 月リリース予定。

デモでは EC サイトの家具データに対し「mid century furniture for my Brooklyn apartment with dark wood tones」で検索し、セマンティック類似度 + テキスト一致の複合結果を取得。

#### Zava Room Designer Agent

HorizonDB のハイブリッド検索 + AI モデル管理 + リランキングを組み合わせたエージェント:
- AI Search でハイブリッド検索（ベクトル + フルテキスト）
- GPT モデルで部屋スタイル分析
- Cohere リランキングモデルで結果を最適化
- ユーザーの部屋に合った家具をキュレーション提案

### VS Code PostgreSQL 拡張

任意の PostgreSQL（ローカル含む）に接続可能な統合開発ツール:
- Object Explorer: テーブル・インデックス・プロパティのブラウズ
- ビジュアルクエリプラン: ツリービュー + テーブルビュー。各演算子に Copilot アイコンを配置し、クリックでエージェントモードによるクエリデバッグを起動
- AI パイプラインデバッグ UI: パイプライン実行状況のリアルタイム可視化
- リアルタイム監視ダッシュボード: CPU・リソースチャート + 異常検知時の Copilot 連携
- バックアップ管理・サーバーパラメーターチューニング
