# Build secure and enterprise-ready agents with Agent 365

Microsoft Agent 365 の GA を発表し、任意のフレームワーク（LangChain、CrewAI 等）で構築されたエージェントに対して、Agent 365 SDK を通じたオブザーバビリティ・ガバナンス・セキュリティの統合的な制御プレーンを提供する方法を実演。エージェントブループリント・エージェントアイデンティティの概念、Registry Sync による他クラウドプラットフォームのエージェント取り込み、Genspark のパートナー統合事例を紹介。

## 概要

AI エージェントが本番環境に移行する中で、開発者がエージェントライフサイクル全体にわたるオブザーバビリティ・ガバナンス・セキュリティを確保する方法を解説するセッション。Microsoft Agent 365 の GA を発表し、Agent 365 SDK を使って任意のフレームワーク（LangChain、CrewAI、LlamaIndex 等）で構築されたエージェントをエンタープライズ対応にする実践的なデモを実施した。

## キーポイント

1. **Microsoft Agent 365 GA**: あらゆるエージェント（Microsoft 製・サードパーティ・カスタム）を一元管理するコントロールプレーンが一般提供開始
2. **Agent 365 SDK**: エージェントビルド SDK ではなく、既存エージェントをラップしてオブザーバビリティ・アイデンティティ・ガバナンス・データセキュリティを付与する SDK
3. **エージェントブループリント**: エージェントの再利用可能な定義（ツール・データ・ルール・ガードレール）を記述するレシピ
4. **エージェントアイデンティティ**: サービスプリンシパルを超えたユーザーレベルのアイデンティティ。On-behalf-of と独自ユーザーアイデンティティの 2 モード
5. **Registry Sync**: Amazon Bedrock・Google Vertex AI 等のサードパーティプラットフォームからエージェントをレジストリに取り込み、可視化・ガバナンスを適用
6. **Genspark パートナー統合**: Agent 365 の Identity・MCP・Observability・Purview を活用し、柔軟なバックエンド（チャットベース AI / 専用 VM）を Microsoft Teams 上で提供

## 詳細

### Agent 365 のアーキテクチャ

Agent 365 は 3 つの柱で構成される:

- **Observe（観察）**: エージェントのリアルタイム監視と管理。レジストリ、利用状況分析、例外追跡
- **Govern（ガバナンス）**: エージェントライフサイクル全体のガードレール。テンプレートによるポリシー一括適用、ルールによる自動化
- **Secure（セキュリティ）**: Microsoft Entra（アイデンティティ管理）、Microsoft Defender（リアルタイム脅威検出）、Microsoft Purview（データガバナンス）、Microsoft Intune（シャドー AI 検出）

### Agent 365 SDK の機能

SDK はエージェントホスティング SDK ではなく、既存エージェントに以下の機能を追加する:

- **Identity**: Entra によるエージェントアイデンティティの付与
- **Observability**: OpenTelemetry ベースのテレメトリ計装
- **Messaging**: Teams やドキュメントコメントからの通知受信
- **Tools**: Work IQ MCP サーバー（Word、OneDrive 等）との連携
- **Threat Protection**: Defender によるリアルタイム脅威ブロック
- **Data Security**: Purview API/SDK によるデータ保護
- **Governance**: ポリシーベースのガバナンス適用

### デモ: LangChain エージェントの Agent 365 対応

Aarthi Saravanakumar が VS Code 上で LangChain（Node.js / TypeScript）製の旅行エージェントを Agent 365 対応にするデモを実施:

1. **Agent 365 Skills の利用**: GitHub Copilot からスキルを呼び出し、約 10 分でエージェントを自動構成
2. **構成内容**: パッケージインストール → ビルド設定 → Entra リソース作成（ブループリント）→ オブザーバビリティ計装 → Work IQ MCP サーバー設定 → 登録・公開
3. **Teams での利用**: テナント内の Teams アプリとしてエージェントを利用。旅行計画を依頼し応答を取得
4. **Word ドキュメント連携**: エージェントを @メンションして Word コメント内でやり取り。通知機能により非同期応答
5. **アクティビティ確認**: エンドユーザーとして自分のエージェントの活動履歴を確認

### IT 管理者向けエクスペリエンス

Microsoft 365 Admin Center での管理画面:

- **概要ダッシュボード**: 総エージェント数、総ユーザー数、ランタイム時間、アクション項目（保留リクエスト、リスクあるエージェント、オーナー未設定エージェント）
- **レジストリ**: 全テナントのエージェント一覧（Microsoft・サードパーティ・カスタム）。メタデータ、指示、アイデンティティ履歴、共有ユーザー、適用ポリシー、権限を確認可能
- **マルチエージェント可視化**: エージェント間の依存関係をビジュアルで表示。ブロック時の影響範囲を把握
- **テンプレート**: Entra・Defender・Purview・SharePoint のポリシーを集約した再利用可能テンプレート。エージェント承認時に適用
- **ルール**: ライフサイクル管理の自動化（オーナー離職時の再割り当て、リスク検出時の自動ブロック）
- **Registry Sync**: Amazon Bedrock・Google Vertex AI からのエージェント取り込み。プラットフォーム側の権限も Agent 365 内で利用可能

### Genspark の統合事例

Genspark（2,000 以上の組織が利用する統合 AI ワークスペース）が Agent 365 と統合:

- **アーキテクチャ**: Microsoft Teams → Agent 365 リレーコンテナ（Identity / Streaming / MCP / Purview / Audit）→ Genspark バックエンド
- **Identity**: 各リクエストごとに認証を実施。Teams グループチャットでの不正アクセスを自動拒否
- **MCP 活用**: PowerPoint・Word・OneDrive への書き込みを Agent 365 の MCP サーバー経由で実現
- **Observability**: 全エージェント活動が Microsoft Purview に記録。テナント管理者が統一ダッシュボードで監視
- **ポリシー継承**: テナントの既存 Purview ポリシー（感度ラベル、DLP）がエージェントにも自動適用
