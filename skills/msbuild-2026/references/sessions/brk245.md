## 概要

OpenClaw プロジェクトの作者 Peter Steinberger が、AI コーディングエージェント時代における開発者の役割転換と、自身のオープンソースプロジェクト運営で構築したツールエコシステムについて語ったセッション。「自分がソフトウェアを速く書く」のではなく「エージェントがソフトウェアを速く書けるようにループを閉じる」というパラダイムシフトを実践し、不満や怠惰を原動力にして次々と自作ツールを開発してきた経験を共有した。

Issue/PR の爆発的増加（10,000 件超）への自動トリアージから始まり、クラウド VM テスト環境、マルチラウンド自動コードレビュー、ビジュアル回帰テストエージェント、マルチプレイヤーコーディングセッションまで、小さなツールが複利的に積み重なり少人数チームで大規模プロジェクトを運営可能にする様子を実演した。

## キーポイント

1. **開発者の仕事は「自分がコードを速く書く」から「エージェントがコードを速く書けるようにする」に転換した**
2. **Issue と PR は本質的に同じもの（変更要求のシグナル）であり、「Prompt Request」と呼ぶべき**
3. **Clause Reaper により約 15,000 件の Issue を自動トリアージ**: vision.md に基づきコメント・クローズを自動実行
4. **main へのマージ後に過去 Issue を週次で再評価し、未読 Issue の自動解決を実現**
5. **CrabBox でクラウド VM 上のテスト実行・クロスプラットフォーム検証・UI テストを統合**
6. **Auto Review で Codex が Codex を呼び出し、新鮮なコンテキストでマルチラウンドレビューを実現**
7. **Mantis で PR のビジュアル検証を自動化（バグ再現動画 + 修正確認動画の自動生成）**
8. **CrabFleet でマルチプレイヤー Codex セッションを実現し、チーム協業を効率化**
9. **不満や面倒さを感じたら自動化のシグナル**: 小さなツールは複利的に積み重なる
10. **agent.md に不変条件を明記し、定期的にエージェント自身に混乱箇所を確認させて改善する**

## 詳細

### エージェント時代の開発者の役割転換

Peter はコーディングエージェントが実用化されて約半年で、ソフトウェア構築の多くのアプローチが陳腐化したと指摘。「ソフトウェアを速く書く」のではなく「エージェントがソフトウェアを速く書けるようにループを閉じる」ことが新しい開発者の仕事であると述べた。

例としてスクリーンショット取得機能を挙げ、人間が繰り返し行う作業をエージェントが自律的に実行できるようにするだけで、開発ループ全体が劇的に効率化されると説明した。

### Clause Reaper: Issue/PR の自動トリアージ

OpenClaw の Issue 数が 5,000 件（実質 10,000 件以上）に達し、CodeRabbit 等の既存ツールではレビューのみで不十分だったため、自作の Clause Reaper を開発した。

仕組みは単純で、Issue または PR がオープンされるたびに GitHub Action で Codex をクラウド起動する。リポジトリの vision.md（プロジェクトの方向性・受け入れ/拒否基準を記載）を読み込み、その基準に基づいてコメント付与またはクローズを自動実行する。これにより約 15,000 件の Issue を処理した。

さらに定期的（週次）に全 Issue/PR に対して Clause Reaper を再実行する。main にマージされた修正が過去 Issue を解決している可能性があるため、数カ月前に報告され一度も人間が読んでいない Issue が自動的にクローズされるケースが常態化した。

### メンテナーダッシュボードと Discord 連携

プロジェクト拡大に伴いメンテナーの活動状況を把握するためのダッシュボードを構築。GitHub の活動だけでなく Discord での議論参加も評価指標に含め、Discord データをクロールするツール（Disk Crawl）を自作した。

Discord のデータは GitHub リポジトリにシリアライズして保存し、bot トークンを持たない他のメンテナーもデータにアクセスできるようにした。

### Octopus: GitHub API トークン管理

大量の Issue 検索と関連 Issue の照合で個人の GitHub API トークン（5,000 リクエスト/時）が常に枯渇する問題に対処。GitHub App をインストールすると得られる 15,000 リクエスト/時のトークンを活用する Octopus を開発した。

Cloudflare Workers 上で動作し、パブリックリポジトリへの読み取り専用リクエストには GitHub App トークンを使用し、ミューテーション操作のみ個人トークンを消費する仕組みで、レートリミット問題を解消した。

### CrabBox: クラウド VM テスト環境

テスト実行時間の増大（10 秒→5 分）とローカルマシンのリソース不足を解決するため、クラウド上に一時的な VM を起動してテストを並列実行する CrabBox を開発した。rsync でコード変更を同期し、高スペックハードウェアでテストを実行する。

主要機能の進化:
1. **マルチプロバイダーフェイルオーバー**: メインプロバイダーがダウンしても AWS や Azure に自動切り替え（現在約 20 プロバイダー対応）
2. **クロスプラットフォームテスト**: Windows/Linux/macOS の同時テストをワンコマンドで起動
3. **VNC 統合**: CrabBox 内でブラウザ・ターミナル等の UI を表示し、エージェントがコンピュータビジョンで操作・検証

### Mantis: ビジュアル検証エージェント

CrabBox + VNC + コンピュータビジョンを組み合わせたビジュアル検証エージェント。PR に対して以下のフローを自動実行する:

1. CrabBox を起動しバグを再現
2. 再現状況を動画撮影
3. バグを修正
4. 修正後の状態を動画撮影
5. 動画を確認し修正を検証

Issue → Clause Reaper が PR 作成 → Mantis がビジュアル検証 → 人間は動画を見てマージ、というワークフローにより、UI バグ修正の人間の関与を最小化した。

### Auto Review: マルチラウンド自動レビュー

Codex の `/review` を手動で繰り返す問題（1 回 10〜30 分、毎回新しい問題を発見、1 日中ループした経験あり）を解消するため、agent.md に 1 行「コミット前に Auto Review を実行せよ」を追加。

Codex が別の Codex セッションを呼び出すことで新鮮なコンテキストでレビューを行い、問題がなくなるまで自動ループする。skill.md ファイルでレビュー基準を調整し、プロジェクト固有の不変条件（例: 実行中のプラグインファイル編集は想定しない）を明記することで誤検知を削減した。

### CorePatch: 大規模コードベースの事後レビュー

Auto Review 導入前にコミットされた大量のコードに対する品質不安を解消するため開発。100 万行規模のコードベースを機能単位の 50 程度のサブセクションに分割し、各セクションを個別の Codex セッションでレビューする。完璧ではないが、コードベースのクリーンアップに有効。

### CrabFleet: マルチプレイヤー Codex

CrabBox 上で Codex を実行し、セッションリンクを他のメンテナーと共有可能にしたもの。チームメンバーがセッションを閲覧・発言・引き継ぎできるマルチプレイヤーコーディング環境を実現した。チーム内でエージェントのやり取りを仲介する手間を排除した。

### プロンプト透明性の仕組み

コントリビューターにプロンプト共有を依頼しても誰も従わなかった問題を、OpenClaw のスキルファイルに機能として組み込むことで解決。PR 作成時にエージェントがコントリビューターに確認し、同意すればプライバシーフィルタリング済みのトランスクリプトを PR に自動添付する。PR に投入された労力を判断する材料として活用。

### vision.md と agent.md のベストプラクティス

- vision.md にプロジェクトの方向性（受け入れる/拒否する変更の基準）を明文化し、すべてのエージェントが参照
- agent.md には意味のある不変条件を記載（短すぎず長すぎず）
- エージェント向けの記述はエージェント自身に書かせる（「人間がクランカーへの書き方を知っている」わけではない）
- 定期的にエージェントに「agent.md で混乱する箇所はないか」を確認し、矛盾や冗長を整理

### その他のツール群

- **Release Bar**: 各プロジェクトの最終リリースからの経過日数を可視化するダッシュボード。リリースすべきプロジェクトを素早く判別
- **GOG**: Google サービス（Gmail、Calendar 等）を操作する自作 CLI。エージェントが Google サービスにアクセスするためのインターフェース
- **Issue Browser**: テキスト選択から Issue を素早く閲覧・フォーマット変換する自作ツール
- **Click Clock**: エンドツーエンドテスト用の自作メッセージングプラットフォーム。外部サービスの Captcha 問題を回避
