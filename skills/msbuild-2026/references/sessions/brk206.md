## 概要

GitHub Copilot SDK が 6 言語（TypeScript、Python、.NET、Go、Java、Rust）で GA に到達したことを発表し、SDK の設計思想・デプロイモデル・アイソレーション技術・将来計画を詳細に解説するセッション。SDK は Copilot CLI および Copilot Cloud Agent と同一のランタイムハーネスを使用しており、ローカルマシンからエアギャップ環境・マルチテナントサービスまで幅広い環境で動作する。

Steve Sanderson によるライブデモでは、C# でのエージェントオーケストレーション構築、OS サンドボックスによるプロンプトインジェクション防御、仮想ファイルシステムによるマルチテナント分離を実演。リモートクラウド環境によるデバイス横断セッション、Rust への書き換えによる 10 倍以上の性能向上計画、ブラウザプラグインへの Wasm コンパイルなど将来の展望も示された。

## 詳細

### CLI/SDK の概要と GA 発表 (Patrick Nikoletich)

GitHub Copilot SDK は、Copilot Cloud Agent と Copilot CLI を駆動する同一のランタイムハーネスをプログラマブルに公開するもの。2025年末から開発を開始し、2026年1月に Technical Preview、Build 2026 で GA 1.0 に到達した。

SDK の核心的な価値は、github.com 上の Copilot Cloud Agent、ローカルの Copilot CLI、そして SDK に組み込まれたエージェントがすべて同一のコアランタイムを使用する点にある。CLI での体験がそのまま SDK での体験に変換される。

GA までに 66 名のコントリビューター、450 以上のイシュー解決、500 以上の PR マージを達成。主な改善領域:

- セッション管理
- パーミッション
- Skills、Tools、Hooks & MCP
- BYOK / カスタムプロバイダー
- OpenTelemetry
- ストリーミング
- カスタムエージェント / サブエージェント
- システムメッセージカスタマイズ
- リモートセッション
- ドキュメント・E2E テスト

### デプロイ環境の多様性 (Patrick Nikoletich)

SDK は以下の環境で動作する:

| 環境 | 特徴 |
|------|------|
| ローカルマシン | パーソナルアシスタント、開発ワークフロー |
| アプリ組み込み | デスクトップ・Web アプリのバックエンド |
| クラウドコンテナ | シークレットをコンテナ外に配置、プロキシ分離 |
| エアギャップクラウド | インターネットアクセスなしの政府クラウド |
| プライベート VNet | Azure VNet 推論、データレジデンシー制御 |
| マルチテナントサービス | 起動時間・メモリフットプリント・処理要件の制約 |

認証は GitHub AUTH、BYOK（Bring Your Own Key）、BYOM（Bring Your Own Model）に対応し、GitHub Copilot サブスクリプションなしでも利用可能。

### エージェント構築デモ (Steve Sanderson)

Steve Sanderson が C# を使ったリアルタイムデモで、不動産の顧客問い合わせ処理エージェントを構築。API は全言語で類似しているため、他言語でも同様に実装可能。

#### 基本セットアップ

```csharp
// CopilotClient インスタンスの作成（ランタイム起動）
var client = new CopilotClient();

// 複数セッションを並列実行可能
var session = client.CreateSession(new SessionConfig {
    Model = "...",
    Permissions = ApproveAll,
    Tools = new CustomTools { ... }
});
```

#### システムプロンプトのカスタマイズ

デフォルトでは Copilot CLI と同じシステムメッセージを使用。全体を置き換えることも可能だが、推奨はセクション単位の置き換え（例: Identity セクションのみを業務固有の指示に変更）。

#### カスタムツールの定義

C# メソッドとして定義し、メタデータ（説明文・パラメータ情報）を付与:

- `ReportPhase`: エージェントの処理フェーズを UI に報告
- `ReportIntent`: エージェントの意図を UI に報告
- `SearchProperties`: SQL データベースから物件検索（複数の検索条件パラメータ）

#### ビルトインツールの有効化

すべてのビルトインツールを有効にすることも可能だが、最小権限の原則に従い必要なもののみ有効化を推奨。デモでは Web fetch ツールのみを追加し、エージェントが Google 検索で最新情報（新設校の位置情報等）を取得できるようにした。

### エージェント実行モデルとアイソレーション (Steve Sanderson)

4 つの実行モデルを紹介:

#### 1. Direct Execution（直接実行）

ホスト OS 上でエージェントループとツールを直接実行。ツールの制限または Human-in-the-loop で制御。

#### 2. Container-per-session（セッション毎コンテナ）

セッションごとに独立したコンテナ/VM を用意。高い分離性を持つが、コストが高い。

#### 3. OS-managed sandbox（OS管理サンドボックス）— Preview

新技術 **MXC** を使用したクロスプラットフォームサンドボックス:

- **Windows**: プロセスコンテナ分離
- **macOS**: macOS サンドボックスプロファイル
- **Linux**: Namespace と cgroup による分離

ファイルシステムアクセス、ネットワーク要求、サブプロセス生成を OS レベルで制御。JSON 設定ファイルでポリシーを定義:

```json
{
  "filesystem": { "read": [...], "write": [...] },
  "network": { "allow": [...] }
}
```

デモではプロンプトインジェクションにより PowerShell でファイル作成を試みる攻撃を、サンドボックスが OS レベルでブロックする様子を実演。プロンプトだけに依存するのではなく、OS のセキュリティ機構で防御する重要性を示した。

#### 4. Multi-tenant Virtualization（マルチテナント仮想化）

仮想ファイルシステム（Session FS）によるセッション分離:

- **軽量**: コンテナ起動なし、10〜100 倍の密度
- **高速**: プロセス内メモリ管理
- **状態仮想化**: インターフェースによる抽象化
- **テナント毎設定**: 認証・設定を抽象化
- **スケール**: N テナント ≈ 1 プロセス

### セッション仮想化デモ (Steve Sanderson)

公開サンプルアプリ（`aka.ms/CopilotSDKServerSample`）を使用したデモ。TypeScript で実装。

#### モードの選択

```typescript
const client = new CopilotClient({
    mode: "empty"  // サーバーアプリ向け（全機能オフからスタート）
    // mode: "copilot-cli" — ローカルツール向け（CLI相当の全機能オン）
});
```

#### Session FS の実装

ファイルシステム操作のコールバックを実装:

- `readFile`: ファイル読み取り
- `writeFile`: ファイル書き込み
- `mkdir`: ディレクトリ作成
- `remove`: 削除

バックエンドストレージは任意（ディスク、メモリ、Blob ストア等）。

#### 仮想 Bash

Just Bash 等のサードパーティライブラリを使い、仮想ファイルシステム上で動作する bash を提供。エージェントから見ると独立した OS 環境のように振る舞う。

#### セッションの永続化とマイグレーション

セッション状態は `events.json` ファイルに記録される。セッションディレクトリを Blob ストアや他サーバーに非同期同期することで:

- ユーザー離脱時にディスクを回収
- 再接続時にセッションを復元
- サーバー間マイグレーション

この手法は github.com の Chat 機能のリライトにも使用されており、コスト削減と機能向上を同時に実現している。

### 今後の展望 (Patrick Nikoletich)

#### GitHub 全体への SDK 統合

Visual Studio、VS Code、github.com の各体験が 30〜60 日以内に統一される方向。プラグインのロード方法、スキルの共有、エージェントランタイムの動作が全サーフェスで共通化される。

#### リモートクラウド環境 (Preview)

SDK を任意のアプリに組み込み、エフェメラルクラウド環境にエージェントをデプロイ可能:

- アプリを閉じてもエージェントは動作を継続
- Web ブラウザ・電話など別デバイスからセッションに再接続
- GitHub のリソース（Task サービス、Mission Control）にアクセス
- デバイス横断でエージェントを操作

#### Cross-surface 配信

| 領域 | 内容 |
|------|------|
| Plugins | 統一フォーマット（Open Plugins）で全サーフェスにロード |
| Memory | パーソナライズされたユーザーコンテキストをサーフェス間で共有 |
| Tool Search | 大規模ツールカタログの品質維持 |
| Citations | プラットフォーム横断で一貫した知識のグラウンディング |
| MCP | 統一認証・標準サポート |
| Compliance | ポリシー管理・IT 管理コントロール |

GitHub 内では User Preference Memory Service を出荷済み。コーディング中に記録されたファクトと好みがサービス間で共有される。

#### Rust ランタイム書き換え

現在の Node.js ベースのランタイムは起動時に約 130 MB の RAM を消費。Rust への書き換えにより:

- 起動時間 > 10 倍改善
- メモリフットプリント > 10 倍改善
- バンドルサイズの大幅削減
- 対応プラットフォームの拡大

2026年2月時点で POC を複数構築済み。正式に開発が進行中。

#### ブラウザプラグインデモ (Preview)

Rust ランタイムを Wasm にコンパイルし、約 500 KB のサイズで Chrome アドオンとして動作させるデモ。V8 サンドボックス内で安全に実行され、ホストマシンへのアクセスなし。リモート機能により macOS から Windows の Cloud PC ブラウザプラグインに接続可能。
