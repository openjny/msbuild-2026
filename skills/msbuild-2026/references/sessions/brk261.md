## 概要

Windows 上での開発体験を高速化・効率化するツール群をデモ中心に紹介するセッション。前半「Building on Windows」では開発環境セットアップ自動化、AI エージェント統合ターミナル、WSL コンテナ CLI、Coreutils、Windows UI 改善を、後半「Building for Windows」では WinUI 3 アプリ開発を支援する WinApp CLI・Win Dev Skills エージェントプラグイン、WSL containers API による Windows アプリへの Linux 統合、SPGO によるコンパイル最適化を扱った。すべてのツールがオープンソースで提供されていることが繰り返し強調された。

## キーポイント

1. **Windows Developer Config**: WinGet 設定ファイルにより WSL（Ubuntu）、Git、GitHub CLI、Copilot CLI、VS Code 等を冪等にインストール。Comfort Shell として Homebrew・ZSH・Starship・btop がプリインストールされた Ubuntu 環境も提供
2. **Intelligent Terminal**: Windows Terminal の実験ブランチとして AI エージェントをターミナルに統合。GitHub Copilot・Claude・Codex・Open Code 等のエージェントを選択可能で、コマンド失敗時に自動検出・修正提案を行う
3. **WSL containers**: `wslc`（別名 `container`）CLI によりサードパーティ不要でコンテナのビルド・実行が可能。各 Windows アプリに専用 WSL VM が割り当てられ、C# NuGet パッケージ経由の API も提供
4. **Coreutils for Windows**: grep・wc・tail・env 等 75 以上の Unix コマンドを Windows ネイティブ `.exe` として WinGet でインストール可能
5. **WinApp CLI & Win Dev Skills**: WinApp CLI は Windows SDK 管理・パッケージング・証明書生成を一元化。Win Dev Skills エージェントプラグインは WinUI 3 アプリ開発の全ワークフローをスキルとして提供
6. **Sample Profile Guided Optimization（SPGO）**: ハードウェアカウンターベースのプロファイリングにより、インストルメンテーション不要でリリースビルドから最適化プロファイルを収集。5〜15% の典型的性能向上、Adobe Photoshop では最大 20% の改善
7. **Windows UI 改善**: PowerToys Command Palette アーキテクチャを活用した新 Run ダイアログ、垂直タスクバーの復活、PowerToys Grab and Move（Alt+ドラッグ）

## 詳細

### Building on Windows

#### Windows Developer Config

`microsoft/WindowsDeveloperConfig` リポジトリで公開された WinGet 設定ファイルを使用し、新規マシンの開発環境を自動セットアップする。インストール対象は Ubuntu（WSL）、Git、GitHub CLI、Copilot CLI、VS Code、Python、Node 等。冪等設計のため既にインストール済みのツールはスキップされ、常に目的の状態を保証する。

Comfort Shell として Homebrew、ZSH、Starship、btop 等がプリインストールされた Ubuntu 環境も同リポジトリで提供。既存の Ubuntu ディストロにセットアップスクリプトとしてプラグインされる。

#### Windows 改善

- **垂直タスクバー**: タスクバーの上下左右配置が Insider Program で利用可能に。長年のユーザーリクエストに応える形で復活
- **新 Run ダイアログ**: PowerToys Command Palette のアーキテクチャを Windows 本体に統合。PowerToys Run → Command Palette → 新 Run という進化の系譜で、OSS コントリビュートが Windows 本体に直接反映
- **Coreutils for Windows**: 75 以上の Unix コマンド（grep、wc、tail、env、test 等）を `.exe` として提供。WinGet でインストール可能。GitHub リポジトリで公開

#### Intelligent Terminal

Windows Terminal の実験ブランチで、AI エージェントとの並行作業を実現する。エージェントペインが端末横に常駐し、コマンド実行中のエラーを自動検出して修正提案を行う。デモではパスワード検証用の正規表現が正しく動作しない状況をエージェントが検出し、修正方針を提案する様子を実演。

初回起動時にエージェント（GitHub Copilot、Claude、Codex、Open Code 等）とモデルを選択可能。マシンにインストール済みのエージェントを自動検出する。オープンソースとして GitHub に公開されフィードバックを募集中。

#### WSL containers CLI

`wslc.exe`（`container` エイリアス）CLI により、Docker Desktop / Podman Desktop / Rancher Desktop なしでコンテナのビルド・実行・管理が可能。既存の Containerfile と互換。

デモでは Python ベースの MarkItDown サービスコンテナを構築し、`-p 8000:8000` でポートフォワーディングを行い Windows ブラウザからアクセス。PDF を Markdown に変換する Web アプリを実演。

独自 CLI を作成した理由:
- `system session` コマンド等の独自拡張を実装するため
- API と CLI の一貫した連携を設計するため
- 基盤の仮想マシン技術の改善は Docker・Podman・Rancher にも無料で還元される

主な特徴:
- クロス OS ファイルパフォーマンス 2 倍改善（予定）
- lazywslc による TUI ダッシュボード
- Dev Containers 対応（attach to running container、reopen in container）
- すべてオープンソース

### Building for Windows

#### WinApp CLI

Windows SDK 管理、パッケージング、アプリ ID 生成、マニフェスト・証明書・ビルドツール管理を単一 CLI で提供。WinGet でワンラインインストール可能。パッケージ ID を持つことでネイティブ通知、Explorer・タスクバー・共有シート連携等が有効化される。オープンソースで GitHub に公開。

#### Win Dev Skills エージェントプラグイン

WinUI 3 と Windows App SDK を使ったアプリ開発を支援するエージェントプラグイン（スキル・MCP サーバー・カスタムエージェントの統合パッケージ）。以下のスキルを含む:
- ビルド＆実行ワークフロー
- WinUI デザイン
- WinUI コードレビュー
- UI テスト
- パッケージング
- WPF からの移行
- セッションレポート
- WinUI セットアップ

エージェントに最新の WinUI 3 / Windows App SDK 情報を直接提供することで、Web 検索のトークン消費を削減し回答精度を向上させる。インストール後は VS Code の Copilot Chat で `WinUI-dev` カスタムエージェントとして利用可能。

デモでは音声プロンプトと Copilot Autopilot モード（Shift+Tab で切替）で WinUI 3 MarkItDown アプリを 1 回のプロンプトから構築し、UI テスト実行まで自動完了。

#### PowerToys Grab and Move

Alt キーを押しながらウィンドウの任意の場所をドラッグすることで、タイトルバーにアクセスできない状況（多数のブラウザタブでタイトルバーが埋まっている等）でもウィンドウを移動可能にするユーティリティ。

#### WSL containers API

C# NuGet パッケージ `WSL Containers` を .csproj に追加することで、`dotnet build` / `dotnet run` の一部としてコンテナを自動ビルド。コンテナ定義は .csproj 内で名前・Containerfile パス・出力先を指定する。

API で制御可能な項目:
- セッション開始（CPU・メモリ数指定）
- イメージプル（ローカルまたはレジストリ）
- ポートマッピング
- ボリュームマウント
- GPU アクセス

各アプリに専用 WSL VM が割り当てられ、アプリ終了時にリソースは自動クリーンアップ。コールドスタートは約 2.5 秒。

デモでは WinUI 版 MarkItDown アプリを C# + WSL containers API で構築。ユーザーからは完全に Windows アプリに見えるが、バックエンドで Linux コンテナが透過的に動作する。

#### MOONRAY レンダリングエンジンデモ

DreamWorks 製オープンソース Linux ベースレンダリングエンジン MOONRAY を WSL containers API で Windows EXE としてパッケージした事例。Windows ファイルパス入力（バックスラッシュ）→ Linux レンダリング → Windows JPEG 出力のワークフローを実演。映画『The Bad Guys 2』『The Wild Robot』等の制作に使用されているツールが WSL containers により Windows ユーザーにもシームレスに提供可能になることを示した。

#### Sample Profile Guided Optimization（SPGO）

従来の PGO の課題（インストルメンテーション必須、ビルドパイプライン分岐、計測時の低速実行でプロファイル品質が低下）を解決するハードウェアカウンターベースのアプローチ。Jianye Lu が解説。

手順:
1. `/SPGO` フラグ付きでコンパイル（まだ最適化なし、PDB 等の準備）
2. リリースビルドを `xperf` でハードウェアカウンター収集しながら実行
3. ETL トレースから SPT ファイルを抽出（ワークフローごとに並列収集可能）
4. 複数 SPT を統合して単一 SPD ファイルを生成
5. SPD を指定して再コンパイル → 最適化バイナリ生成

デモではスタックベース仮想マシンによるフィボナッチ計算で約 33% の性能向上を実演（3 秒 → 2 秒）。switch 文の分岐予測最適化とホット/コールドコード分離が主な効果。

Adobe Photoshop チームの成果:
- 実ワークフローで最大 20% の CPU 性能改善
- 低レベルチューニングよりスケーラブルで持続可能なアプローチと評価
- プロダクションビルドパイプラインに統合済み
- SPGO 最適化版を近日公開予定

運用の推奨: コード変更やユーザー行動の変化に応じてプロファイルを反復的に更新する。Step 3（xperf 収集）から繰り返すだけで最新の最適化を維持可能。

### オープンソースとコミュニティ

Terminal、WSL、WinUI、PowerToys、WinGet の合計で 16,437 名のオープンソースコントリビューターが貢献。PowerToys Command Palette へのコントリビュートが Windows 本体の新 Run に反映される等、コミュニティの成果がプロダクトに直接還元される仕組みが強調された。
