## 概要

AI エージェントがコードを書けるようになった今、開発者の課題は「作れるか」から「出荷できるか」に移行している。本セッションでは GitHub が構築中の Agent-Native Engineering System のビジョンと、その具体的な実装である GitHub Copilot app を中心にライブデモを展開。マクロ委任・マイクロステアリングのワークフローにより、開発者がアイデアから本番デプロイまでのギャップを埋める方法を実演した。

セッションの大部分をデモに費やし、「Demos, not memos」の精神で GitHub Copilot app の新機能群を実際に動作させながら紹介。Canvas によるカスタマイズ可能な UI、Cloud Sandbox による安全なリモート実行、Automations によるプロアクティブなワークフロー自動化、Agent Merge による PR の自律的なマージ、Chronicle によるセッション横断の記憶と検索、Rubber Duck による批評的レビュー、マルチモデルによるコード品質・セキュリティレビューを一連のフローとして披露した。

## キーポイント

1. **SDLC の終焉**: 従来のソフトウェア開発ライフサイクルはプロダクト開発ライフサイクルと融合し、人間とエージェントの協調が中心に
2. **Agent-Native Engineering System の 6 つの柱**: Surfaces（マルチサーフェス）、Runtime（安全な実行環境）、Automation（予測可能な自動化）、Quality（検証システム）、Memory（継続的知識）、Trust（エンタープライズ信頼性）
3. **活動ではなく成果にフォーカス**: エージェントの目的は「もっと多くやる」ことではなく「もっと多く完了する」こと。実装だけでなく、CI の通過・レビュー対応・マージ・デプロイまでを含む
4. **Canvas エコシステム**: エージェントが駆動するカスタム UI を開発者自身が構築・共有可能。マーケットプレイスも提供予定
5. **Cloud Sandbox**: GitHub Sandboxes 上のステートフルな micro VM により、ラップトップを閉じてもエージェントが作業を継続
6. **Agent Merge**: PR のマージコンフリクト解消・CI 修正・レビューフィードバック対応を自律的に実行し、開発者の注意を保護
7. **Chronicle**: 全 Copilot サーフェス横断のセッション履歴を SQL ベースで検索・活用。スタンダップレポートも自動生成

## 詳細

### Agent-Native Engineering System

Mario Rodriguez が冒頭で整理したフレームワーク。GitHub をヒューマン間コラボレーションのプラットフォームから、ヒューマンとエージェントのコラボレーションプラットフォームへ進化させるための設計思想:

- **Surfaces**: IDE・CLI・モバイル・デスクトップ・Web など複数のサーフェスで Copilot を利用可能。マルチモデル対応（Claude、GPT、Gemini 等）
- **Runtime**: ローカルおよびクラウドのサンドボックスで安全にエージェントを実行。資格情報のスコープ制限と分離
- **Automation**: CI/CD の次世代として、エージェントがイベント駆動・時間駆動で自律的にタスクを実行
- **Quality**: マルチモデルコードレビュー、セキュリティレビュー、Rubber Duck による批評で品質を担保
- **Memory**: Chronicle によるセッション横断の記憶。開発者の全作業履歴を検索・活用可能
- **Trust**: エンタープライズスケールでのセキュリティ・ガバナンス。パートナーエコシステム（Sonar、Endor、LaunchDarkly 等）

### GitHub Copilot App のデモ

#### My Work とバッチセッション

GitHub Copilot app のホーム画面から「My Work」ビューでフィルタリングし、関連する複数の Issue を選択して一括でエージェントセッションを起動。各セッションは分離されたワークツリーで並行して作業を進行する。

#### Canvas

Canvas は GitHub Copilot app の右側パネルに表示されるカスタマイズ可能な UI サーフェス。エージェントがターミナル・ブラウザ・カスタム UI を駆動し、開発者との双方向インタラクションを実現:

- **ビルトイン Canvas**: ターミナル、Web ブラウザ
- **カスタム Canvas**: Triage Canvas（チームのデイリースタンダップ用 Issue 分析）、Kanban Board 等を `.github` リポジトリにチェックインして共有
- **Canvas マーケットプレイス**: Miro 等のパートナーによる公式 Canvas 拡張。エージェント駆動のデザインアーキテクチャボード等
- **Create Canvas スキル**: 自然言語の指示からエージェントが Canvas 拡張を動的に生成（デモでホワイトボード Canvas をワンショット生成し、Gist への公開まで成功）

#### Cloud Sandbox (`copilot --cloud`)

GitHub Sandboxes 上の高速 micro VM:

- サブ秒で起動するステートフルな分離環境
- ラップトップがオフラインでもエージェントが夜通し作業を継続
- 資格情報は厳密にスコープ制限
- PR のテストを安全な環境で実行可能

#### Automations (`/every`)

ローカルまたはクラウドで実行されるエージェント自動化:

- **Morning Brief**: 毎朝 6:00 に実行。重要 PR・未レビュー PR・新規 Issue のサマリーを自動生成
- **Issue Repro**: Issue 作成イベントをトリガーに、バグ報告の自動再現テスト作成。再現成功時にラベル追加とコメント投稿
- ローカル自動化はプライベート、クラウド自動化はチーム共有

#### Agent Merge

PR に対して「Agent Merge」ボタンをクリックすると、エージェントが自律的に:

- マージコンフリクトを解消
- CI を監視しグリーンに維持
- CCR（Copilot Code Review）フィードバックに対応
- 人間のレビューコメントに適切に応答（盲目的に従わず、正当な理由で拒否することもある）
- すべて完了後にマージを実行

デモでは、セキュリティ担当者の大規模リファクタリング要求を「このPRのスコープ外」と適切に判断して拒否し、マージした事例を紹介。

#### Chronicle

全 Copilot サーフェス（VS Code、CLI、.com、Cloud Agent）横断のセッション履歴:

- `/chronicle search`: 過去のセッションをセマンティック検索（デモでは以前修正した Rust バグの詳細を検索）
- `/chronicle standup`: 直近 24 時間の作業履歴から自動スタンダップレポート生成
- SQL API を直接エージェントに公開し、構造化クエリとフルテキスト検索を組み合わせ

#### Rubber Duck

開発者同士の壁打ち会話をエージェントで再現:

- **Advisor モード**: アドバイスを提供
- **Critic モード**: 批評的にレビュー
- Copilot CLI と GitHub Copilot app の両方で利用可能

#### マルチモデルコード・セキュリティレビュー

複数のセッションを階層的に作成し、並列でレビューを実行:

- コードレビューセッション: Gemini、GPT-5.5、Sonnet 4.6 を並列使用
- セキュリティレビューセッション: GitHub Advanced Security と連携したセキュリティサブエージェント
- 親セッションで結果を統合し、Rubber Duck で最終批評
- 必要に応じて PR にコメントを自動投稿

### パートナーエコシステム

Trust の柱として紹介されたパートナー:

- **Sonar**: コード品質
- **Endor**: セキュリティ
- **Miro**: Canvas マーケットプレイス（デザインアーキテクチャ）
- **LaunchDarkly**: 本番デプロイ（フィーチャーフラグ）
