# GitHub Copilot in Visual Studio: Agents That Debug, Profile, and Test

Visual Studio のエージェント機能（Debugger Agent・Profiler Agent）を Visual Studio 自身のコードベースで実演。ADO MCP Server 経由でバグ取得→テスト作成→デバッガーエージェントで根本原因特定→修正、Profiler Agent による計測→最適化→再計測のワークフローを紹介。ロードマップとして GitHub Copilot CLI SDK 統合、任意モデル対応、App Modernization Agent、AI マージコンフリクト解決を発表。

## 概要

Visual Studio におけるエージェント機能を、Visual Studio 自身のコードベース（Diagnostics Hub / Profiler）を使ってデモする実践的セッション。コード生成だけでなく、プロフェッショナル開発者が直面するデバッグ・パフォーマンス最適化・テスト作成を AI エージェントで加速する方法を示す。

セッションのフレーミングとして「コードはアセットかアーティファクトか」を提示。長期メンテナンス対象のコードを扱うプロフェッショナル C#/C++ 開発者のためのツールとして Visual Studio を位置づけ、品質ゲート・ガバナンス・コーディング標準を維持しながらエージェントを活用する「プログレッシブ・エンハンスメント」アプローチを強調した。

## キーポイント

1. **Debugger Agent**: テストを右クリック→「Debug this test with Copilot」でデバッガーエージェントが起動。ブレークポイント設定、式評価、メモリ検査を自動で行い根本原因を特定・修正する
2. **Profiler Agent**: テストを右クリック→「Profile with Copilot」でプロファイリング→分析→最適化→テスト実行→再計測を自動実行。「計測なき最適化はリファクタリングに過ぎない」を体現
3. **VS Test Performance Collector**: NuGet パッケージをテストプロジェクトに追加し、CI テスト実行時にプロファイリングトレース（.diagsession）を自動収集。Azure DevOps PR パイプラインに組み込み可能
4. **Azure DevOps MCP Server**: Copilot Chat からワークアイテムを取得し、バグの再現テスト作成にコンテキストとして活用。パイプラインの diagsession ファイル取得にも利用可能
5. **GitHub Copilot CLI SDK への統合**: Visual Studio が VS Code・CLI と同じ SDK ベースに移行。機能の同時提供が可能に。Visual Studio Insiders で翌週から利用可能
6. **任意モデル対応**: ローカル・オンプレミス・クラウド問わず任意のモデルを Visual Studio で使用可能に。エンタープライズ向けにモデル許可リストのガバナンス機能付き
7. **App Modernization Agent**: WebForms（ユーザーコントロール、サーバーコントロール含む）→ Blazor への変換、Aspire 追加、クラウド対応を自動化

## 詳細

### Visual Studio の位置づけ（Mads Kristensen）

Visual Studio はプロフェッショナル C# / C++ 開発者向けの Windows 上のプレミアツールであると明確に位置づけた。エージェント時代における差別化ポイント:

- **品質ゲート**: セキュリティ、アクセシビリティ、コンプライアンス、ガバナンス
- **IT 管理**: グループポリシーによる組織全体のロールアウト制御
- **コード品質**: エージェントがコードを書く時代において、レビューとコーディング標準の維持がますます重要
- **コードはアセット**: 長期メンテナンス対象のコードを扱う開発者のためのツール。チーム開発、コーディング標準、フォーマットが重要な場面（対して使い捨ての PoC やインターナルツールでは「コードはアーティファクト」）

HTML5 時代の「プログレッシブ・エンハンスメント」になぞらえ、基本機能は従来どおり手動で操作でき、その上にエージェント機能を積み重ねるアプローチを採用していると説明。ユーザーは常に手動操作に戻れる選択肢を持つ。

### デモ 1: Debugger Agent によるバグ修正（Nik Karpinsky）

Visual Studio の Diagnostics Hub ソリューション（C++, C#, JavaScript, Assembly, IL を含む 160 以上のプロジェクト）上で実際のバグを修正するデモ。このソリューションが Visual Studio Profiler そのものであり、ビルドされて Visual Studio に組み込まれる。

**シナリオ**: HybridDictionary の EnumKeysIterator で IndexOutOfRangeException が発生する内部テレメトリバグ。新機能追加により未使用だったコードパスが初めて実行され、内部ドッグフーダーからのクラッシュレポートとして報告された。

**ワークフロー**:
1. **Azure DevOps MCP Server 経由でバグ取得**: Copilot Chat にバグリンクを貼り付け、「このバグを再現するユニットテストを書いて」と指示。MCP Server が認証トークンを使用してワークアイテムにアクセス
2. **コンテキスト収集**: エージェントが ADO からワークアイテムを取得し、スタックトレースから HybridDictionary のソースと既存テスト構造を探索。該当コードパスにはテストが存在しないことを発見
3. **テスト生成**: EnumerateKeys と EnumerateValues の 2 テストメソッドを生成。ビルドエラー（型パラメータが非 nullable 値型であるべき制約）を会話的に修正
4. **テスト実行で再現確認**: 最初はアサーション失敗（期待と異なるエラーパス）→ 会話で修正 → IndexOutOfRangeException で正しく失敗
5. **「Debug this test with Copilot」**: 右クリックメニューからデバッガーエージェントを起動
6. **自動デバッグ**: エージェントがブレークポイント設定、式評価、仮説立案を実行。コレクション要素数に関する「off-by-1 エラー」と特定
7. **修正＆検証**: コード修正後、デバッガー下でテスト再実行 → パス確認

**テストファーストの意義**: Nik は「Copilot が登場するまで TDD を信じていなかった」と述べ、エージェントにコード修正を任せる際のガードレールとしてテストハーネスを先に構築する新しい開発パターンを提示。修正後のテストは耐久的なアーティファクトとしてリポジトリに残り、将来のリファクタリング時にも保護を提供する。

### デモ 2: Profiler Agent によるパフォーマンス最適化（Nik Karpinsky）

VS Test Performance Collector を使った計測→最適化→検証のデモ:

**シナリオ**: ExpressStream の大ファイルデコード（.diagsession 内の ETL ファイルの圧縮/展開）のパフォーマンス改善

**ワークフロー**:
1. **「Profile with Copilot」**: テストを右クリックしてプロファイリング開始（従来の「Profile」と同じ UI から選択可能）
2. **動的インストルメンテーション**: メソッド呼び出し間に start/stop を挿入し、正確なパフォーマンスデータを収集。CPU 使用率・メモリアロケーション・オブジェクト割り当てを計測
3. **初期分析**: ホットパスは `ExpressStream.ReadBlock` → `CheckReadEnoughBits` に集中。実行時間 3.88 秒
4. **最適化提案**: バッファ読み取りによる BinaryReader オーバーヘッド削減を提案。Copilot がプロファイリングレポートの行情報とソースコードをマッチングし、最適化対象を正確に特定
5. **コード変更**: 入力ストリームの ByteReader 呼び出しコストを削減するため、新しい byte 配列バッファを作成して直接読み取り。Huffman テーブルの読み取りコードも更新
6. **テスト実行（動作確認）**: 最適化後にテストを実行し、動作が壊れていないことを確認（ガードレール）
7. **再プロファイリング（効果検証）**: 同じテストを再度プロファイリングし、3.88 秒 → 約 2 秒（約 2 倍の性能向上）を確認

**「計測なき最適化はリファクタリングに過ぎない（If you're not using data, you're not improving performance, you're just refactoring）」** — Measure twice, optimize once の原則を強調。

**VS Test Performance Collector の活用パターン**: Azure DevOps の PR パイプラインに NuGet パッケージとして追加し、テスト実行時に自動でプロファイリングトレース（.diagsession）をテスト添付ファイルとして収集。ADO MCP Server と組み合わせ、Copilot にパイプラインの diagsession を取得・分析・最適化提案させることも可能。

**実績紹介**:
- Visual Studio 起動時に 800,000 アロケーションを発見（Azure DevOps キャッシュファイルの XML デシリアライゼーション）→ 初期ヒープ 50 MB 削減
- Azure App Service のランニングコスト削減
- Roslyn コンパイラの switch case 評価の最適化（チームの SDE 2 が Roslyn をダウンロードしてベンチマークに対して実行、コミュニティ貢献として PR マージ）
- diagsession リーダー自体の読み込み性能を 6〜7 件の PR で 2 倍以上に向上

### ロードマップ: 今後の Copilot 機能（Mads Kristensen）

翌週以降の Visual Studio 2026 Insiders / 夏にかけてリリース予定の機能:

#### App Modernization Agent

レガシーコードの維持要員が減少する中、WebForms アプリケーション（ユーザーコントロール、サーバーコントロール含む）を Blazor に変換。Aspire の追加やクラウド対応も Agent が自動化。「上司に 6 ヶ月の移行許可を取る必要がなくなる」とフレーミング。

#### Agent Skills 自動適用

スキルの発見可能性（discoverability）問題を解決。プロジェクトタイプとタスクに基づき、各チーム（Azure チーム、WinForms チーム等）が作成した Microsoft 公式スキルを自動ロード。スキルはオープンソースでコミュニティ貢献を受け付け。

#### ビルド速度最適化

エージェントがビルドを実行する前にエラーリストや他のインジケーターを確認し、既にエラーがある状態での長時間ビルド実行を回避。これは多数のビルド高速化施策の第一弾。

#### AI マージコンフリクト解決

ボタンクリックで AI がマージコンフリクトを解決。人間が解決する場合とほぼ同等の精度。解決できないコンフリクトは人間に委ね、複数のコンフリクトのうち自動解決可能なもののみを処理。開発者の「pull 実行時の不安」を解消。

#### GitHub Copilot CLI SDK 統合

Visual Studio が独自の Copilot バックエンド実装から GitHub Copilot CLI SDK に移行。これにより Visual Studio、VS Code、CLI が同一の SDK ベースとなり、機能の同時提供が可能に。Visual Studio 固有の最適化（MSBuild のウォーム状態活用によるビルド高速化等）は維持。翌週から Visual Studio Insiders で利用可能。Mads は約 1 ヶ月間フルタイムで使用済み。

#### 任意モデル対応（BYOK）

ローカル・オンプレミス・クラウドの任意モデルを使用可能に。GitHub Copilot 資格情報でのサインインも不要。エンタープライズ向けにはモデルの許可リスト制御機能を提供し、組織が使用可能なモデルを制限可能。セキュリティ要件が厳格でクラウドモデルを使用できない企業にも対応。Visual Studio 2026 Insiders で翌週から利用可能。
