# Agent supervision is the new senior engineering skill

AI エージェントがコードを大量生産する時代に「エージェント監督」がシニアエンジニアの新スキルであると主張。マルチエージェントオーケストレーション、ダークファクトリー（人間レビューゼロ）の実態、仕様駆動開発、推論速度の飛躍的向上、リアルタイムインタラクション研究の最前線をインタラクティブ形式で議論した。

## 概要

AI エンジニアコミュニティの主催者である swyx が、エージェント時代におけるシニアエンジニアの新スキル「エージェント監督（Agent Supervision）」について論じたセッション。コーディングエージェントの爆発的普及に伴い、ボトルネックが「コードを書くこと」から「エージェントとの高帯域な意思疎通による信頼構築」へ移行していると主張する。

GitHub Copilot App を含むすべてのコーディングエージェントが「コンダクター」型 UI に収束する現象を出発点に、マルチエージェントオーケストレーション、ダークファクトリー（人間レビューゼロの自動化パイプライン）、仕様駆動開発、推論ハードウェアの飛躍的進化、そして今後の研究方向について幅広くカバーした。

インタラクティブ形式で会場からの質問にも随時回答し、実践的な知見を共有した。

## キーポイント

1. **コンダクター型 UI への収束**: GitHub Copilot App、Cursor、Windsurf、Codex、Cognition すべてが同一の「エージェントコマンドセンター」形式に収束。独立進化で同形態に至る「カニ化（carcinization）」現象と類似
2. **モデル＋ハーネスの時代**: 競争優位はモデル単体にもハーネスコードにもなく、人間とエージェントのマインドメルド（意思疎通の質）にある。MCP・スキル・ワークフロー等のハーネス構成要素が重要
3. **SDLC の変革**: IDE の死 → PR からプロンプトリクエストへ → ゴールループ → ダークファクトリーという段階的進化。GitHub 上のエージェントコミット量は前年比 14,000 倍成長
4. **ダークファクトリーの実態**: OpenAI Symphony の事例。約 2,000 行の仕様 Markdown、1 分以内のコンパイルタイム、API コントラクトを「ナローウェスト」として設計しエントロピーを制御
5. **スイスチーズモデル**: テスト・仕様・オンライン評価・フィーチャーフラグ・プログレッシブロールアウトの多層防御が必須
6. **推論速度革命**: Cerebras の数千 tok/s から Talos の 17,000 tok/s へ。300,000 tok/s も視野に入り、10 倍ごとに利用習慣と製品設計が根本から変わる
7. **リアルタイムインタラクション**: Thinking Machines のインタラクション小型モデルが示す方向性。トレーニングデータにマイクロバッチのインタラクションを組み込む新アプローチ
8. **エージェントワークパラドックス**: エージェントがかつてないほど働いているが、人間もかつてないほど忙しい。AI エンジニアリングは「最後の仕事」

## 詳細

### コーディングエージェントの収束

GitHub Copilot App のリブランドを起点に、すべてのコーディングエージェント（Conductor、Cursor、Windsurf、Codex、Cognition）が同一の「コンダクター」型 UI — エージェントコマンドセンター — に収束している現象を解説。Conductor が最初にこのフォームファクターを確立し、各社が独立してリブランドした結果、地球上で少なくとも 7 回独立進化したカニのように同じ形態に至った。

個別エージェント管理から、12 のターミナルを同時操作する段階を経て、Cognition の「ローカルからクラウドへのハンドオフ」のような整理されたマルチエージェントオーケストレーションへの移行が進んでいる。

### AI エンジニアコミュニティの 6 つのテーマ

swyx が AI エンジニアコミュニティの運営を通じて観察した 6 つのコアテーマ:

1. モデル単体の宗教からモデル＋ハーネスの組み合わせへの転換
2. プロンプトだけでなく MCP・スキル・ワークフロー等のハーネス構成要素の重要性増大
3. チャット以外の新しい UI 形態の必要性
4. モデルラボのスウォーム指向化（Gemini K2 の数百並列スウォーム、Anthropic の恥ずかしいほどの並列処理）
5. エージェント分野における人間の雇用増加
6. ソースコード流出がビジネスに影響しない事実（Cloud Code のリーク事例）

### モデルとハーネスの関係

Cloud Code のソースコード全量が流出しても Anthropic のビジネスに影響がなかった事実を引用し、モートがモデルにもハーネスコードにもないことを論証。競合他社も「彼らに秘密のソースはない。我々も同じことを発見済み」と証言。

真の競争優位は人間とエージェントのマインドメルド — すなわち高帯域な意思疎通と整合性にあると結論づけた。

### 自律性の閾値変化

2025 年後半に発生した閾値変化を指摘。モデルの自律タスク遂行能力が 1〜2 時間から 10〜12 時間、さらに 20〜24 時間のタスクへと一桁跳躍。この 10 倍の自律能力向上が、異なる種類の UX を要求する。

GitHub 上の Cloud Code によるコード量は 2026 年 2 月時点で 4〜5%、現在推定 10%、年末に 50% 到達見込み。GitHub CEO のポッドキャスト出演で前年比 14,000 倍のコミット成長が報告された。

### ソフトウェア開発ライフサイクルの変革

Steve Yegge（Gastown 創設者）による「IDE は 2026 年に死ぬ」という予言が的中したことを紹介。SDLC の次のフロンティアとして:

- **PR → プロンプトリクエスト**: OSS への悪意あるPR対策として、コードではなく意図（プロンプト）を送り、メンテナーが適応する方式
- **ゴールループ**: エージェントに「どう作るか」ではなく「どう評価するか・いつ停止するか」を伝える Ralph Wiggum ループ。全主要コーディングエージェントが採用
- **8 段階の LLM サイコシス**（Yegge の分類）: 単一エージェント → マルチエージェント → 自前オーケストレーター構築 → 完全自律。2025 年 11 月の予測が 2026 年 3 月に現実化

### ダークファクトリーと仕様駆動開発

OpenAI の Ryan Leathereau による「ハーネスエンジニアリング」を紹介。1 日 10 億トークン（約 1 万ドル/日）を消費し、人間レビューゼロで運用。

OpenAI Symphony（オープンソース）の仕様は約 2,000 行の Markdown で構成され、以下を重視:

- API コントラクト（モジュール間の「ナローウェスト」）
- 型とデータ構造
- 1 分以内のコンパイルタイム（高速フィードバックサイクル）

「超人的なコーダーはいるが超人的なレビュアーはまだいない」— 帯域の制約が本質的問題。高速サイクルによる整合性確保が現時点の最善策。

会場からの質問を受け、「ゴール」の本質は「エージェントにどう作るかではなく、どう評価するか」を記述することであり、スイスチーズモデル（テスト・仕様・オンライン評価・フィーチャーフラグ・プログレッシブロールアウト）による多層防御が大規模ソフトウェア組織の実践を小規模チームにも要求すると説明した。

### サンドボックスとインフラストラクチャ

エージェントに必要な計算環境として、Git ブランチのフォークだけでは不十分で、環境全体のフォークが必要と指摘。「Kubernetes on crack」問題 — Kubernetes は生存維持が目的だが、エージェントには極端なサーバーレス性が求められる。

GPU:CPU 比率が 8:1 から 1:1 へ移行中（Intel CEO の決算説明会発言を引用）。Sam Altman も「非常に大きな程度で推論企業になった」と発言。CPU 不足の時代が到来しつつある。

### 推論速度の飛躍

推論速度の現状と展望:

| チップ/プラットフォーム | 速度 |
|----------------------|------|
| 一般的な推論 | 50–100 tok/s |
| Cerebras | 数千 tok/s |
| Talos | 17,000 tok/s（実演済み） |
| 将来目標 | 300,000 tok/s |

すべてハードウェア最適化であり、ML 研究由来ではない。H100 の価格は減価償却曲線が逆転し、4 年物のチップが値上がりする異例の状況。チップの償却期間は従来の 3〜4 年から 8 年程度に延伸。

「小型モデル」の定義も拡大を続けている: 2B → 7B → 24B（Mistral Small）。トップエンドは GPT-4 の約 1 兆パラメータから 10 兆超へ。

10 倍ごとに利用習慣・構築可能な製品・オープンモデル vs クローズドモデルの選択が根本から変わるため、インフラ投資の方向性を理解した上でのプロダクト設計が重要。

### 今後の研究方向

6 つの研究アジェンダを紹介（重み付けは均等ではない）:

1. **リアルタイムインタラクション**（最も実現が近い）: Thinking Machines のインタラクション小型モデル。セミ非同期の「不気味の谷」を解消。マイクロバッチのインタラクションをトレーニングデータに組み込む新アプローチで、単なるハードウェア高速化とは本質的に異なる
2. **ビデオエージェント**: 単パスの動画生成からエージェントオーケストレーション型へ移行。ドメイン特化のオーケストレーター訓練が鍵
3. **ワールドモデル**: ユーザーの意図・嗜好の Theory of Mind。中央値 RLHF ではなくユーザー固有の世界モデル維持
4. **メモリと継続学習**: 最も長期的だが影響大。5 年以上の研究期間を見込む
5. **コーディング以外のドメインへの波及**: 昨年コーディングに起きたことが今年すべてのナレッジワークに波及。OpenAI の Codex と ChatGPT の統合が象徴
6. **エージェントワークパラドックス**: エージェントの仕事量と人間の仕事量が同時に増大する現象。正常であり、開発者は「最初に圧縮された未来を体験する特権」を持つ
