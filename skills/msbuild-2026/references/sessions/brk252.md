# From observability to ROI for AI agents on any framework

Microsoft Foundry Observability のエンドツーエンド機能を実演。任意フレームワーク（LangChain、OpenAI SDK 等）で構築された AI エージェントに対する OpenTelemetry ベースのトレーシング、ルーブリック評価器による多次元品質評価、VS Code 内コードファースト分析、Agent Optimizer による自動最適化（25 分で 27% 向上）、Agent ROI によるビジネス価値証明までの Agent DevOps ライフサイクルを包括的にデモ。

## 概要

AI エージェントは非決定論的であり、従来のモニタリング手法では信頼性と一貫性の担保が困難である。本セッションでは Microsoft Foundry Observability を用いたエージェント DevOps ライフサイクルの全体像を、4 つのライブデモを通じて実演した。トレーシング・評価・モニタリング・最適化の 4 つの柱を軸に、エージェントの品質確保から ROI 証明までを一貫して扱う。

デモシナリオとして「Vendor History Analyst Agent」（データセンター運用のベンダー履歴分析エージェント）を使用。BRK241 で紹介されたデータセンターシナリオを引き継ぎ、ベンダーマネージャーがデータドリブンな意思決定を行うためのエージェントを題材に、Foundry ポータルでのプレイグラウンドテスト、ルーブリック評価器の自動生成、トレースベースの品質分析、VS Code での Foundry Toolkit を使ったコードファースト分析、Agent Optimizer による自動最適化、そして ROI ダッシュボードでのビジネス価値可視化を順に示した。

## キーポイント

1. **任意フレームワーク対応のオープンエコシステム**: OpenTelemetry ベースのトレーシングと評価が LangChain、LangGraph、OpenAI SDK、Microsoft Agent Framework 等すべてのフレームワークに対応。Foundry ホスト以外のエージェントも統一的に観測可能
2. **ルーブリック評価器（Public Preview）**: エージェントのシステムプロンプトとコンテキストから多次元評価基準を自動生成。データなしでも即座に評価を開始でき、オフライン・オンライン両方で利用可能
3. **コードファースト Observability**: Foundry Toolkit（VS Code 拡張）と Foundry MCP サーバーにより、IDE 内からトレース分析・評価実行・比較・最適化を直接実行。スキルベースのガイド付きフローで開発者を誘導
4. **マルチターン評価（Public Preview）**: 会話全体のコンテキスト維持・推論一貫性・タスク完了を評価。ユーザーシミュレーションにより現実的な多ターン会話を自動生成
5. **Agent Optimizer（Private Preview）**: システムプロンプト・ツール定義・スキル・モデル選択の組み合わせを自動的にヒルクライム最適化。デモでは 25 分で 27% のスコア向上を達成
6. **Agent ROI（Private Preview）**: エージェント呼び出しごとのトークンコスト・ツールコストとビジネス価値を紐付け、正味価値をバージョン別・日次トレンドで可視化。低 ROI トレースへのドリルダウンも可能
7. **Azure Monitor 統合**: Foundry の AI プラットフォームレイヤーと Azure Monitor のフルスタック可視性を組み合わせ、AI ワークロードからインフラまでの統合オブザーバビリティを提供

## 詳細

### エージェント Observability の 4 つの柱

Microsoft Foundry Observability は以下の 4 つの柱で構成される:

- **トレース**: エージェントのエンドツーエンド実行フロー（プロンプト → モデル → ツール呼び出し）を可視化
- **評価**: トレースから品質・安全性を定量的に評価（ビルトイン評価器 + カスタム評価器 + ルーブリック）
- **モニタリング**: オフラインと同じ評価をオンラインで実行し、リアルタイムで問題検知
- **最適化**: トレースと評価結果を基にエージェントを継続的に改善

Foundry Observability はコントロールプレーンの一部として Agent Service・Models・IQ・Tools・Machine Learning を横断的にカバーする。従来の DevOps サイクル（Plan → Code → Test → Release → Monitor → Analyze → Optimize）が AI エージェント時代に進化した「Agent DevOps ライフサイクル」を提示し、Getting Started（インナーループ前半）→ コードファースト（インナーループ全体）→ アウターループ（モニタリング・分析・最適化）→ ROI の順にセッションを構成した。

### Getting Started: アウトオブボックスの Observability

Foundry でホストされたエージェントを作成すると、自動的にトレーシングと Observability が有効化される。デモでは Vendor History Analyst Agent をプレイグラウンドでテストし、右パネルにストリーミングされるログからエージェント実行の詳細を確認した。

#### ルーブリック評価器

ルーブリック評価器は、エージェントのシステムプロンプトとターゲットエージェントを指定するだけで、8 次元の評価基準を自動生成する。各次元には重みが割り当てられ、単一の評価器で多角的な品質評価が可能となる。データがない状態からでも即座に評価を開始でき、コンテキストファイルの追加アップロードやコードからの生成にも対応する。

### トレースと継続的評価

プロダクションのトレースは Azure App Insights に流れ、Foundry ポータルのトレースビューに表示される。継続的評価ジョブによりプロダクショントレースに対して自動的にルーブリック評価が実行され、スコアの低いトレースを特定・ドリルダウンして原因分析を行える。

デモでは、特定のベンダー詳細について「ハルシネーション」が発生しているトレース（タスク完了とルーブリックスコアが共に低い）を特定。ユーザーが要求した正確なデータをエージェントが提供できていない問題を発見した。トレース ID をクリックすると、エージェント呼び出しの詳細ビューで各評価次元のスコアと説明を確認できる。

### オープンエコシステム対応

Foundry のトレーシングと評価は Foundry ホストのエージェントに限定されない。OpenTelemetry 標準に基づき、以下のフレームワークで構築されたエージェントを統一的に観測可能:

- LangChain / LangGraph
- OpenAI SDK
- Microsoft Agent Framework
- その他 OTel 対応フレームワーク

エージェントの各ステップ（ツール呼び出し、LLM 呼び出し、ハンドオフ）が単一のトレースビューにキャプチャされ、フレームワークを問わず一貫した評価シグナルを得られる。Microsoft のエンジニアは OpenTelemetry のオープンソースコミュニティに積極的に貢献しており、メモリに関する新しいセマンティクスも近日中に Foundry に導入予定。

### コードファースト Observability と Foundry Toolkit

Foundry Toolkit は VS Code 拡張として提供され、Foundry MCP サーバーと Foundry Skill を含む。VS Code 内の GitHub Copilot Chat から直接以下の操作が可能:

- 評価結果の取得と分析（直近 8 時間の評価サマリー、次元別ブレークダウン）
- 最弱次元の特定と失敗パターンの分析
- リポジトリコンテキストと Foundry スキルコンテキストに基づくグラウンデッドな改善提案
- システムプロンプトの改善ドラフト生成
- プロダクショントレースをデータセットにフィードバックし回帰テストを構築

Skill はオピニオネイテッドなフローとして設計されており、Observability ループのどの段階にいるかを自動認識し、Foundry 側（サーバーサイド）とコード側（クライアントサイド）の両レイヤーにまたがる改善提案を行う。一般的な LLM に評価結果を入力するのと異なり、リポジトリコンテキストと Foundry スキルコンテキストに基づいてグラウンドされた提案が可能。

### 新しい評価機能（Public Preview）

| 機能 | 説明 |
|------|------|
| マルチターン評価 | 会話全体でのコンテキスト維持・推論一貫性・エンドツーエンドのタスク成功を評価。5 分で終わるべきセッションに 20 分かかる場合の検知にも有効 |
| ユーザーシミュレーション | 現実的な多ターン会話を自動生成し、評価データの初期セットを作成 |
| トレース → データセット | プロダクショントレースを選択してオフライン評価データセットに変換し、テストカバレッジを拡大 |
| スマートフィルタリング | 継続的オンライン評価で最も関連性の高いトレースをサンプリング |

### 評価器カタログ

Foundry は品質・リスク/安全性・エージェント評価にわたる包括的な評価器カタログを提供する:

- **品質**: Groundedness、Relevance、Response Completeness、Coherence、Fluency、Similarity、Task Completion、Customer Satisfaction、NLP Metrics（F1 Score 等）、Quality Grader（多くがマルチターン対応）
- **リスク/安全性**: Indirect Attack Jailbreaks、Hate and Unfairness、Sexual、Violence、Self-Harm、Protected Material、Ungrounded Attributes、Code Vulnerability、Prohibited Actions、Sensitive Data Leakage
- **エージェント**: Intent Resolution、Tool Call Accuracy、Tool Selection、Tool Input Accuracy、Tool Output Utilization、Tool Call Success、Task Adherence
- **カスタム**: LLM-as-Judge、コードベース評価器（正規表現チェック、DB ルックアップ等の非決定論的シナリオに推奨）、ルーブリック評価器

Sebastian は特にコードベース評価器を推奨。正規表現チェックや DB 参照は LLM-as-Judge を使う必要がなく、確定的な検証に適している。

### Agent Optimizer（Private Preview）

Agent Optimizer は、エージェントのシステムプロンプト・ツール定義・スキル・モデル選択を自動的に最適化する機能である。`azd ai agent optimize` コマンドで起動する。

**入力要件**:
- ホストされたエージェント
- 評価データセット（代表的タスク群、デモでは 40 クエリ）
- ルーブリック評価器

**最適化変数**:
- システムプロンプト（instructions ファイル）
- スキル構成
- ツール定義（description がコンテキストに入りツール利用に影響）
- ターゲットモデル（コスト・レイテンシ・品質のトレードオフ探索）

**プロセス**:
1. 最適化モデル（リフレクション用）を選択。エージェントモデルより高性能なモデルを推奨
2. 最適化ジョブが新バージョンのエージェントを作成・デプロイ・評価を自動実行
3. うまくいく設定を強化し、うまくいかない設定は別戦略に切り替え
4. 候補を段階的に生成（デモでは 4 候補）

**結果**:
- デモでは 25 分でベースラインスコア 0.577 → 0.73（**27% 向上**）を達成
- 候補 1: システムプロンプト変更で 40 タスク中 38 成功
- 候補 4: システムプロンプト + ツール定義の変更で 40 タスク全成功
- diff ビューでシステムプロンプトとツール定義の変更内容を確認可能
- 各候補の品質・コスト・レイテンシを比較表示

**反復的改善**: ワンショットではなく、プロダクショントレースから新タスクを継続的に取り込み、ドメイン変化・モデル変化・ツール実装変化に対応してヒルクライムを繰り返すことを推奨。

### Agent ROI（Private Preview）

Agent ROI はエージェントの投資対効果を定量化する機能である:

- **設定**: 評価器を選択し、成功時のビジネス価値を割り当て（例: 1 回の成功分析あたり $5 の時間節約）
- **コスト計算**: トークンコストは自動取得、ツールコストはユーザーが平均値を指定
- **可視化**: バージョン別の価値 vs コスト vs 正味価値を日次トレンドで表示
- **分析**: 低 ROI トレース（例: 完全に失敗した呼び出し = 全額コスト）にドリルダウンし、コードファースト機能で原因分析
- **インサイト**: デモではツールコストが LLM コストより高いことが判明し、最適化の方向性を示唆

Foundry ポータルと API の両方で利用可能。バージョン比較と日次トレンドビューをサポート。
