GitHub Copilot Modernization Agent を中心としたエンタープライズ規模のアプリケーションモダナイゼーションを実演。CLI による並列アセスメント、Command Center によるポートフォリオ管理、Rulebook によるガバナンス、Custom Skills による組織固有カスタマイズ、メインフレーム COBOL から Java への変換、ASP.NET Web Forms から Blazor + Aspire + Azure Container Apps へのエンドツーエンド移行を包括的にデモ。

## 概要

Forrester 調査で 94% の IT リーダーがアプリモダナイゼーションを投資優先事項とする中、技術的負債の重さがイノベーションを阻害し続けている。本セッションでは GitHub Copilot Modernization Agent を軸に、エンタープライズ規模のモダナイゼーションを「スケール・カスタマイズ・ガバナンス」の 3 本柱で実現するエンドツーエンドプロセスをライブデモで示した。

Azure Copilot（IT 運用チーム向け）と GitHub Copilot（開発者・アーキテクト向け）が連携し、発見・アセスメント・計画・実行・デプロイ・運用の継続的モダナイゼーションループを形成する。デモではメインフレーム COBOL アプリの Java 変換、レガシー Java Struts から Spring Boot への書き換え、ASP.NET Web Forms から Blazor + .NET Aspire + Azure Container Apps へのモダナイゼーションを実演した。

## キーポイント

1. **GitHub Copilot Modernization Agent が GA**: CLI からポートフォリオ全体のアセスメントとアップグレードを並列実行。クラウドエージェントが数千アプリケーションを同時処理
2. **Custom Skills が GA**: 組織固有のマイグレーションパターン・内部ライブラリ・Azure ベストプラクティスをスキルとしてエンコードし、ポートフォリオ横断で再利用
3. **Command Center（Private Preview）**: ポートフォリオ全体のモダナイゼーション進捗を一覧管理。フェーズ別ステータス・タイムライン・所有者の可視化
4. **Rulebook（Private Preview）**: ガバナンス・セキュリティ・アーキテクチャ標準を中央集権的にエンコードし、エージェントがガードレール付きでモダナイズ
5. **メインフレームモダナイゼーション（Private Preview）**: COBOL/JCL/BMS をリバースエンジニアリングして文書化し、ネイティブ Java + SQL に変換
6. **ASP.NET Web Forms → Blazor（Private Preview）**: Web Forms から Blazor Server への自動変換、.NET Aspire によるオーケストレーション追加、Azure Container Apps へのデプロイまでを一気通貫
7. **Azure Migrate 連携**: Azure Migrate のオンプレミス発見情報から GitHub Copilot Modernization CLI へのシームレスなハンドオフ
8. **顧客実績**: SAP Labs China・FMG・Microsoft 内部チームで 88% のマイグレーション工数削減、10,000 以上のエンジニアリング時間を節約

## 詳細

### モダナイゼーションの 3 本柱

Jeff Fritz がオープニングで提示した GitHub Copilot Modernization の設計思想:

- **スケール（Scale）**: 1 アプリずつではなく、アセスメント・計画・アップグレード・フレームワーク変更をポートフォリオ全体で並列実行
- **カスタマイズ（Customize）**: アプリごとに異なるコードベース・要件に対応。60 以上のビルトインスキルに加え、組織固有の Custom Skills を追加可能
- **ガバナンス（Govern）**: Human-in-the-loop レビュー、ポリシーによる承認・禁止パターン、コンプライアンスルール、監査証跡を全プロセスに組み込み

### 継続的エージェンティックモダナイゼーション

モダナイゼーションは一回限りのプロジェクトではなく継続的ループとして定義:

- **モダナイゼーション側**: Plan（発見・アセスメント）→ Execute（アップグレード・変換）→ Innovate（リファクタ・AI 対応化）
- **オペレーション側**: Observe（モニタリング）→ Troubleshoot（インシデント対応）→ Optimize（コスト・パフォーマンス最適化）
- Azure Copilot が IT 運用の計画・発見を担い、GitHub Copilot が開発者のコード変換・デプロイを担当

### メインフレームモダナイゼーション

Hazem El-Hammamy がデモした COBOL アプリケーションの Java 変換プロセス:

1. **アセスメント**: COBOL・JCL・BMS・DB2・VSAM ファイルをスキャンし、欠損依存関係を早期検出
2. **文書化**: エグゼクティブサマリー、ユーザーペルソナ、機能モジュール、アーキテクチャ図、ユーザージャーニー、ビジネスプロセス、プログラムレベル詳細（データフィールド・ビジネスロジック・コールグラフ・データリネージュ）を自動生成
3. **データレイヤー変換**: VSAM/DB2 のデータフィールドをネイティブ SQL テーブルにマッピング
4. **アプリケーション変換**: メインフレーム固有のニュアンスを考慮した Java ネイティブ実装を生成

パートナー（Amdocs、TCS、Infosys、Kyndryl、Accenture、Capgemini、Avanade、NTT Data、Ensono、Hitachi）とのハイブリッドコンシェルジュアプローチで提供。

### CLI によるスケールアセスメント

Nish Anil がデモした Modernize CLI のフロー:

- JSON 設定ファイルでポートフォリオ内の全リポジトリ URL を定義
- アセスメントドメイン選択（アップグレード・クラウドレディネス・セキュリティ）
- 分析カバレッジ選択（Issue のみ～フル分析: アーキテクチャ図・API コントラクト含む）
- ターゲットランタイム・コンピュートサービスの指定
- ローカル実行またはクラウドエージェント委任による並列実行
- ヘッドレス実行モードで CI/CD パイプラインからのブートストラップ対応
- 実行結果は GitHub PR としてアセスメントレポート・アーキテクチャドキュメントを生成

### Azure Migrate との連携

オンプレミスインフラストラクチャからのモダナイゼーション開始をサポート:

- Azure Migrate でプロダクション環境を発見・アセスメント
- 「Add code insights to GitHub Copilot modernization」でハンドオフ
- 設定ファイルを生成し、Modernize CLI でコードレベルのアセスメントを実行
- レポートを Azure Migrate ストレージにプッシュバック
- HTML 統合レポートでウェーブプランニング・コスト見積もり（Azure 小売価格ベース）・工数見積もりを提供

### Command Center

Private Preview で発表されたポートフォリオ管理ダッシュボード:

- Modernize CLI に同梱のセルフホスト可能ポータル
- アセスメント・計画・実行の各フェーズ進捗を一覧表示
- プロジェクトタイムライン（開始日・デッドライン・エージェント進捗）
- アセスメントレポートの比較機能
- 組織全体での情報共有と意思決定支援

### Rulebook

Private Preview で発表されたガバナンス機能:

- Markdown 形式でポリシー・ガードレールを記述
- Charter（高レベル原則）: 例「クラウドネイティブ認証は DefaultAzureCredential 経由」
- Policies（セキュリティ要件・ハード境界）: エージェントが絶対に行ってはいけない操作を定義
- 承認済み SKU・可観測性要件（OpenTelemetry）等を指定
- 実行計画生成時にエージェントがルールを遵守
- 異なる技術スタック（Struts → Spring Boot、Web Forms → Blazor）でも同一 Rulebook が適用

### Custom Skills

GA として発表された組織固有カスタマイズ:

- 中央リポジトリで組織のスキルライブラリを管理（VS Code の「Manage Repositories」で設定）
- IDE から検索・追加可能
- デモ例: Kafka → Azure Event Hubs のカスタム移行パターン、PII ハンドリングのロギングスキル
- エージェントスキルと同じ形式で記述

### ASP.NET Web Forms → Blazor + Aspire

Nish Anil がデモしたレガシー .NET モダナイゼーション:

1. ASP.NET Web Forms（default.aspx、run at server コンポーネント）を .NET 10 + Blazor Server に変換
2. フローモード「Automatic」で最小プロンプトでの自動変換
3. 変換後アプリに .NET Aspire を追加（コードファーストオーケストレーション・可観測性）
4. Aspire ダッシュボードでサービスディスカバリ・URL・ポリグロット構成を管理
5. Aspire CLI の deploy コマンドで Azure Container Apps にデプロイ
