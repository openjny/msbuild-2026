## 概要

GitHub Copilot CLI を中心に、AI エージェントがコードを「マージされるまで」導くワークフロー全体を実演したセッション。週 2.75 億コミットが GitHub に着地する現在、課題は「コードを書けるか」から「マージまで自動化できるか」に移行している。前半は CLI の核心機能（マルチモデル選択、Skills、Custom Agents、Plan/Autopilot モード、スラッシュコマンド群）と Build での新機能（Voice モード、新 TUI、JetBrains 統合、/rubber-duck、/every）を紹介。後半は GitHub Copilot SDK のマルチプラットフォーム展開デモ（ブラウザ操作、PowerPoint 生成、ゲーム UI）と、聴衆参加型のライブコーディングで Issue → Plan → Fleet → PR の完全自動化ループを実践した。

## キーポイント

1. **コードランディングが真の価値**: AI ツールの成否は「コミットを生成できるか」ではなく「PR がマージされるか」で測られる。Issue 選択 → AI スキャフォールド → ブランチ → PR → CI → レビュー → マージの全ステップを CLI が支援
2. **マルチモデルハーネス**: CLI 引数・セッションコマンド・環境変数・config ファイルでモデルを即時切替。Claude、GPT、Gemini の最新モデルに対応し、タスクに応じて使い分け可能
3. **Plan モードで仕様駆動開発**: Plan モードで 1 時間以上かけて仕様を詰め、SQLite に todo と依存関係を記録。その後 Autopilot に切り替えると 95% の確率でワンショット実装が完了
4. **Fleet による並列実行**: 複数サブエージェントを並列起動し、異なるモデルでレビュー・テスト計画・実装を同時実行。todo 間の依存関係を認識し安全に並列化
5. **/rubber-duck でクロスモデル批評**: 異なるモデルファミリーに計画を批評させることで統計的に有意な品質改善を達成。Claude で計画 → GPT-5.5 で批評のパターンが紹介された
6. **Voice モード（GA）**: Foundry Local の Nemotron Speech Streaming モデルによるオンデバイス音声認識。発話中にリアルタイムでテキストストリーミングされ、自然な言い直しも解釈可能
7. **GitHub Copilot SDK**: TypeScript、Python、.NET、Go、Java、Rust で利用可能。ブラウザ操作プラグイン、Office プラグイン、カスタムチャット UI など、CLI の能力を任意のインターフェースに組み込み可能
8. **GitHub Copilot App は Rust SDK で構築**: アプリのカラースキーム変更、デフォルト MCP サーバー設定、スキル・カスタムエージェント管理がすべて CLI ベースで動作

## 詳細

### コードがマージされるまでのワークフロー

セッション冒頭で「エージェントはコードがランディングして初めて価値がある」と宣言。GitHub では週 2.75 億コミットが着地し、AI 関連リポジトリは 430 万を突破（2 年で約 2 倍）。開発者のワークフローは以下のステップで構成される:

1. Issue を選択
2. AI ツールで探索・スキャフォールド・リファクタ・修正ドラフト
3. ブランチにプッシュ
4. Pull Request を作成
5. CI で基本検証
6. レビュアーがエッジケースと設計を検証
7. マシンと人間の両方の信頼を得てマージ

### CLI のコア機能

#### モデル選択

| 切替方法 | 使い方 |
|----------|--------|
| CLI 引数 | `copilot --model claude-sonnet-4.6` |
| セッションコマンド | `/model gpt-5.4` |
| 環境変数 | `COPILOT_MODEL=gpt-5.3-codex` |
| Config ファイル | `~/.copilot/config.json` → `{ "model": "claude-sonnet-4.5" }` |

#### Skills

`.github/skills/{skill-name}/` に `SKILL.md` を配置し、カスタムスキルを定義。読み込み優先順位はプロジェクト → 親ディレクトリ → パーソナル → プラグイン → `COPILOT_SKILLS_DIRS` 環境変数。Claude Code の `.claude/skills/` フォルダとも互換。

#### ビルトインエージェント

| エージェント | 役割 |
|-------------|------|
| EXPLORE | grep/glob/view によるコードベース探索。並列実行セーフ |
| TASK | テスト・ビルド・リント・フォーマッタの実行 |
| GENERAL | 全ツールを使う汎用マルチステップエージェント |
| REVIEW | コード変更のレビュー。バグ・セキュリティ脆弱性・ロジックエラーのみ報告 |

Copilot Code Review エージェントは GitHub 自身のコードベースへの第 3 位のコントリビューターであると紹介された。

#### Custom Agents

`.github/agents/` または `~/.github/agents/` に `.agent.md` ファイルを配置。frontmatter で `name`、`description`（必須）、`tools`、`mcp-servers`、`infer` を設定。

### Plan モードと Autopilot

Plan モードでは仕様を対話的に詰め、リサーチ（Web 検索、OSS リポジトリ調査、パッケージ探索）を行い、todo リストを SQLite に記録。依存関係グラフにより直列・並列の判断が可能。Evan Boyle は「非自明な機能は必ず Plan モードで 1 時間以上費やし、Autopilot に切り替えると 95% ワンショットで実装完了する」と述べた。

### 主要スラッシュコマンド

| コマンド | 機能 |
|----------|------|
| `/review` | マルチモデル並列コードレビュー。3 モデルのサブエージェントが独立レビューし外側モデルが統合 |
| `/new` | 新規 CLI セッション作成。セッション間切替が可能 |
| `/pr` | PR の作成・管理。`auto-fix` サブコマンドで CI 失敗やレビュー指摘を自動修正 |
| `/fleet` | 複数サブエージェントの並列実行。モデル混合指定可能 |
| `/research` | Blackbird（GitHub コード検索エンジン）を使い全 OSS・プライベートリポジトリから詳細レポートを生成。引用付き、Gist エクスポート対応 |
| `/experimental` | 実験的機能の有効化トグル |
| `/every` | セッション内で定期自動化を設定（5 分・1 時間・1 日など）。MCP 連携で Slack/Teams/メールの要対応メッセージ Top 10 レポート等 |
| `/rubber-duck` | クロスモデルファミリー批評。小モデルから大モデルまで機能し統計的に有意な改善を確認 |
| `/share` | セッションを Gist に共有 |
| `/clear` | コンテキストウィンドウをクリア |
| `/sessions` | セッション一覧表示・切替 |

### Build での新機能

#### Voice モード（GA）

Foundry Local の Nemotron Speech Streaming モデル（NVIDIA 開発）をオンデバイスで実行。従来の音声モデルと異なり発話中にリアルタイムでテキストがストリーミングされる。スペースバー長押しで起動。

#### 新 TUI（Terminal User Interface）

セッション・Issues・Pull Requests・Gists のタブ切替が可能な新しいターミナル UI。Build で正式リリース。

#### JetBrains 統合

JetBrains IDE 内で Copilot CLI を利用可能に。エージェント選択、スラッシュコマンド（`/remote`、`/chronicle` 等）、エージェントデバッグパネル（Preview）、エージェントカスタマイズパネル、クラウドエージェント統合、モデル thinking effort 設定に対応。

### GitHub Copilot SDK

CLI の能力を任意のプラットフォームに組み込むための SDK。対応言語は TypeScript、Python、.NET、Go、Java、Rust（+ コミュニティによる Erlang、C++ 等）。

デモされたユースケース:
- **ブラウザ操作プラグイン**: Amazon で商品検索・カート追加を自律実行
- **PowerPoint 生成**: CLI セッション履歴から週次プレゼンを自動生成し、Outlook で練習会議を設定
- **ゲーム UI**: トークン消費を可視化するゲーム風インターフェース

GitHub Copilot App 自体が Rust SDK で構築されており、同じ OSS ツールで誰でも同等のアプリを作れることが強調された。

### ライブコーディングデモ

聴衆が公開リポジトリに Issue を投稿し、CLI がそれを処理する参加型デモ:

1. Voice モードで Issue 一覧を確認し「最もスパイシーなデモ」を選択
2. 「Always-on-top Octocat that celebrates git lifecycle hooks」を選択
3. Electron アプリとして Plan を策定
4. Autopilot で実装→動作確認（git 操作時に Octocat がアニメーション）
5. 追加 Issue（Goblin モード等）を Plan に取り込み
6. `/rubber-duck` で GPT-5.5 による計画批評を実施
7. Fleet モードで並列実装を開始

### 開発者向けベストプラクティス

- **インストラクションは軽量に**: メジャーモデルリリースごとに削除し、問題発生時に再追加。トークン節約とコンパクション回避
- **検証ループが生産性の鍵**: エージェントが自律動作し続けるにはテスト（Unit/E2E）が不可欠。CLI コードベースの 60-70% はテストコード
- **大規模コードベース対応**: grep が非効率な場合はコード検索インデックス（ベクター検索）を活用。CLI にローカルインメモリ/ディスクインデックスが近日搭載予定
- **CI 駆動開発**: PR を開いてエージェントに CI を開発ループとして反復させるワークフロー。マルチ OS 対応に有効
