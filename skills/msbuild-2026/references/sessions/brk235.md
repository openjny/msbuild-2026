## 概要

Ollama 共同創設者の Michael Chiang と Parth Sareen が、オープンモデルによるエージェント構築とローカル・クラウドハイブリッド推論の現状と将来を解説。`ollama launch` コマンドによる GitHub Copilot CLI、VS Code、Open Claw 等のエージェントハーネスとのワンコマンド統合を実演し、プライバシーが要求されるタスクにはローカル推論、高難度タスクにはクラウド推論を使い分けるハイブリッドワークフローの実用性を示した。オープンモデルの急速な能力向上（推論・ツール呼び出し・長期タスク対応）と統合メモリ搭載ハードウェアの進展により、ローカル AI 推論がエンタープライズレベルの実用段階に入ったことを強調した。

## キーポイント

1. **ollama launch によるツール統合**: 1 コマンドで GitHub Copilot CLI、VS Code、Open Claw、Hermes、Codex 等のエージェントハーネスにオープンモデルを注入可能
2. **ハイブリッド推論**: プライバシー重視タスクはローカル、高難度タスクはクラウド（ゼロデータ保持）と使い分けるワークフローが実用化
3. **オープンモデルの急速進化**: 汎用チャット → 推論 → ツール呼び出し → 長期タスク対応へと能力が段階的に拡大
4. **800 万以上のアクティブ開発者**: Ollama プラットフォーム上で研究機関・製造業・メディア企業がミッションクリティカルなタスクに活用中
5. **統合メモリの重要性**: Apple Silicon、RTX Spark（128GB）、AMD Strix Halo 等の統合メモリにより大規模コンテキスト長をローカルで実現
6. **API 互換性**: OpenAI・Anthropic 互換 API と Python/JavaScript SDK でアプリケーション構築が容易

## 詳細

### Ollama プラットフォーム概要

Ollama は開発者がオープンモデルに最も簡単にアクセスし、自分のツールと連携して使用するためのプラットフォームである。主な機能:

- `ollama run` でモデルを即座に起動しチャット
- `ollama launch` でお気に入りのツールにオープンモデルを統合
- OpenAI・Anthropic API 互換でカスタムアプリケーション構築
- Python・JavaScript SDK
- ツール呼び出し・構造化出力・ワンラインエージェントセットアップに最適化

800 万以上のアクティブ開発者を擁し、主要モデルラボ・ハードウェア企業とパートナーシップを結んでいる。セッション当日にも Google DeepMind の Gemma 412B Unified モデルが即日提供開始された。

### Ollama Cloud

2026 年初頭に導入されたクラウドサービスで、ローカル推論を補完する:

- データセンターグレードのハードウェアでフロンティアモデルを実行
- **ゼロデータ保持** — データは一切学習に使用されない
- ローカルに十分なコンピュートがない場合のスケールアウト手段
- 並列エージェント実行による高難度タスクの対処
- モデルプロバイダーが提供する最大コンテキストウィンドウをそのまま利用可能

### ollama launch によるエージェントハーネス統合

`ollama launch` コマンドにより、以下のツールにオープンモデルをワンコマンドで接続:

- GitHub Copilot CLI
- VS Code
- Plot Code / Codex
- Open Claw / Hermes（パーソナルエージェント）
- Droid Pie
- その他多数

デモでは `ollama launch copilot` で Kimi K 2.6（クラウド）を GitHub Copilot CLI に接続し、GitHub Issue の取得・分析・サブエージェントによるコードベース探索・修正プラン生成までを実演。ローカルモデルも同様のワークフローで利用可能。

### ハイブリッド推論デモ

プライバシーが最優先されるユースケースとしてクレジットカード明細の分析を実演:

- **ツール**: π（Pie）— システムプロンプトを肥大化させないミニマルなハーネス
- **モデル**: Qwen 3.6（ローカル実行、MacBook 上）
- **タスク**: PDF 明細の処理・カテゴリ別支出分析
- **特徴**: データが一切端末外に送信されず、完全にローカルで処理完結

ローカルモデルが「十分に実用的」なレベルに達しており、機密データを扱う日常タスクに即座に活用可能であることを実証した。

### オープンモデルの能力進化

オープンモデルの能力は段階的に進化している:

1. **汎用チャット** — 初期のモデル
2. **推論能力** — 思考プロセスの獲得
3. **ツール呼び出し** — 単一・複数ツールの利用
4. **長期タスク** — 数時間〜数日に及ぶ自律タスク（出現段階）

フロンティアリサーチ（DNA 合成等）には依然としてクラウドモデルが必要だが、主流のユースケースではオープンモデルが大幅に低コストで実用に耐えるレベルに到達している。

### 実世界での活用事例

セッションで紹介された公開事例:

- **Lawrence Berkeley National Laboratory**: X 線物理研究の自律的実行。クラウドモデルと Ollama のルーティングレイヤーで推論を分散
- **NASA Glenn Research Center**: 火星ミッションタスクの分類。18 の人間系タスクカテゴリへの分類に使用
- **US DOE Brookhaven National Laboratory**: ログデータ分析の自動化。LangChain と GUI を組み合わせた自然言語による問題解決
- **多国籍メディア企業**: SEC 提出書類・四半期決算の財務文書処理（機密性によりクラウド AI 不可）
- **大手製造業**: 機械エンジニア向け CAD 設計アシスタント
- **自動車メーカー**: 工場フロアでの RAG ベース検査システム

### ハードウェアと統合メモリの展望

ローカル AI 推論におけるハードウェアの進展:

- **統合メモリの利点**: VRAM 制約なしに大規模コンテキスト長をサポート。速度のトレードオフはあるがタスク対応範囲が大幅に拡大
- **Apple Silicon**: 統合メモリの先駆け。MLX 推論エンジンで高速化
- **NVIDIA RTX Spark**: 統合メモリ最大 128GB
- **AMD Strix Halo**: 統合メモリ最大 128GB
- **Qualcomm Snapdragon**: モバイルデバイスでも実験的に Ollama が動作

Ollama は MLX 推論エンジン（Apple Silicon 向け）を独自に持ち、NF4 量子化でローカル推論を高速化している。

### Q&A ハイライト

- **モバイル対応**: 現時点ではモバイルを直接ターゲットにしていないが、Snapdragon チップセットでの動作は確認済み。業務用途の主流ユースケースに注力
- **推奨コーディングモデル（128GB Mac）**: DeepSeek 系 70B、Gemma 4 Dense モデル、QwQ 2SS 120B。複数モデルを試してベンチマークすることを推奨
- **モデル選択のアプローチ**: タスクに依存。ollama launch で使用ハーネスに応じた最新モデル推奨機能を今後追加予定
