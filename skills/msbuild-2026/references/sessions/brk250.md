# Observe and control agents across any framework with open source tools

AI エージェントのリスク特定・評価・制御・監視の 4 ステップを、新オープンソースツール ASSERT（ポリシー駆動評価フレームワーク）と Agent Control Specification（ACS、ランタイム制御標準）で実演。銀行エージェントのデモを通じ、ポリシー違反率を 57% から 10% に低減する過程と Microsoft Foundry への本番展開パスを示した。

## 概要

AI エージェントが本番環境で引き起こす意図しない有害行動（プロンプトインジェクション、データ漏洩、不正ツール実行、過剰拒否）に対処するため、Microsoft が新たにオープンソースで公開した 2 つのツールを中心にデモを展開。ASSERT（Adaptive Spec-Driven Scoring for Evaluations and Regression Testing）で自然言語からポリシー評価を自動生成し、Agent Control Specification（ACS）で決定論的ガードレールを適用する一連のワークフローを、LangGraph で構築した銀行エージェントを題材にライブで実演した。プロンプト修正のみでは複雑なマルチターンシナリオの違反を解消できないことを示し、ACS の決定論的制御が過剰拒否を増やさずに違反率を大幅低減することを証明した。

## キーポイント

1. **エージェントの 4 つの失敗モード**: 命令・制御失敗、情報完全性・漏洩、ツール/アクション誤用、システムレベル・創発的リスク
2. **Lethal Trifecta**: コンテキスト汚染 → エージェント混乱 → 機密データアクセス → 外部へのデータ流出という連鎖パターン
3. **ASSERT のオープンソース公開**: 自然言語ポリシーから体系化・分類・テスト生成・判定を自動化。従来 21〜28 日かかった評価セット作成を数分に短縮
4. **Agent Control Specification（ACS）のオープンソース公開**: 18 フレームワーク対応、入力・モデル・ツールコール・出力の各チェックポイントで決定論的/確率的制御を適用
5. **プロンプト修正の限界**: シングルターンでは 28% → 15% に改善するが、マルチターンシナリオでは 57% のまま改善せず
6. **ACS ガードレールの効果**: マルチターンシナリオの違反率を 57% → 10% に低減。過剰拒否を増やさない決定論的制御
7. **Agent Governance Toolkit（AGT）**: MCP セキュリティゲートウェイ、サンドボックス、アイデンティティ、ACS を統合。3.7K GitHub Stars、102+ Contributors
8. **Microsoft Foundry との連携**: ASSERT → ACS → Foundry Evaluations → Agent Optimizer のプロダクションパイプライン

## 詳細

### エージェントの失敗モードとリスク

Sarah Bird がエージェントの失敗を 4 カテゴリに整理:

| カテゴリ | 説明 | 銀行エージェントでの例 |
|---------|------|----------------------|
| 命令・制御失敗 | エージェントが制約に従わない、操作される | プロンプトインジェクションで管理者モード解放 |
| 情報完全性・漏洩 | ハルシネーション、機密データ露出 | 架空の国際送金手数料を回答 |
| ツール/アクション誤用 | 不正なツール呼び出し、承認なし実行 | 承認なしで $25,000 送金実行 |
| システムレベル・創発的リスク | スケールや相互作用から生じるリスク | 正当なリクエストの過剰拒否 |

SailPoint の調査（5 大陸の開発者・IT 管理者・セキュリティ専門家対象）では、60% のエージェントが特権データにアクセス、57% が無許可で機密データ共有、54% が不適切な情報を配布しているとの結果が示された。

### ASSERT: ポリシー駆動評価フレームワーク

Microsoft Research との共同開発によるオープンソースツール。4 段階のパイプラインで動作:

1. **体系化・分類（Systematize & Taxonomize）**: 曖昧なポリシー記述を、文献調査→多角的視点シミュレーション→コンセプト仕様→行動タクソノミーに展開。許容/非許容の行動を明確に定義
2. **テストセット生成（Generate Test Sets）**: タクソノミーからシングルターン・マルチターンのテストケースを自動生成。タスクタイプ・ペルソナ・ツール・リクエストクラス等の次元で層別化
3. **推論実行（Run Inference）**: エージェントに対してテストケースを実行。ツールコール・取得コンテキスト・中間アクションを含む完全トレースを記録
4. **ポリシー判定（Score Against Policy）**: 各トレースを行動スタンスに対して判定。pass/flag だけでなく根拠・ポリシー引用・判定理由を出力

デモでは銀行エージェントに対し「財務情報の歪曲禁止」「無許可取引の実行禁止」「機密データ漏洩禁止」「ソーシャルエンジニアリング耐性」を自然言語で定義し、YAML ファイルに記述するだけで評価を実行。結果はシングルターンで 28%、マルチターンシナリオで 58% のポリシー違反を検出した。

パートナー: CrewAI、Ariza、Pipecat、Pydantic 他

### Agent Control Specification（ACS）

エージェントのランタイム制御をフレームワーク非依存で標準化するオープン仕様:

- **8 つの介入ポイント**: 入力、モデル前後、ツールコール前後、出力等の各段階で制御を適用
- **制御タイプの混合**: Rego による決定論的ルール、分類器によるニュアンス判定、LLM ジャッジによる複雑推論を任意のチェックポイントで組み合わせ可能
- **18 フレームワーク対応**: Agent Framework、CrewAI、LangChain 等。オープン標準による移植性
- **Fail-closed 設計**: ポリシー・アノテータ・アダプタのいずれかが失敗した場合はデフォルトで拒否

ACS のアーキテクチャ:
- **Host Runtime**: 介入ポイントで ACS を呼び出し、最終判定を適用
- **Agent Control Specification**: 標準的なポリシーリクエストを構築し、何に対してポリシーが適用されるかを特定
- **Policy Engine**: 純粋なポリシーロジックを評価し、決定と効果を返却

デモでは SSN（社会保障番号）検出時に機密情報をモデルに渡さないガードレールを Rego で実装。ACS 適用後、シングルターン違反率は 28% → 0%、マルチターンは 57% → 10% に低減した。

### Microsoft Foundry でのプロダクション展開

ローカル開発からプロダクションへの 5 ステップ:

1. **Define & Evaluate locally**: ASSERT でポリシーから評価を生成し欠陥率を把握
2. **Set intervention points**: ACS でチェックポイントと制御を定義
3. **Re-evaluate & validate**: ACS 適用後に ASSERT を再実行し改善を検証
4. **Evaluate in Cloud**: Foundry Evaluations でプロダクショントラフィックを継続評価
5. **Optimize**: Agent Optimizer で品質・コスト・レイテンシを最適化

Foundry の統合機能:
- **Microsoft Defender**: 脅威検出とセキュリティオペレーションチームへのアラート
- **Microsoft Purview**: データ保護とガバナンス
- **Microsoft Entra Agent ID**: エージェントの安全なアクセス制御
- **OpenTelemetry + Application Insights**: 分散トレーシングとパフォーマンス可視化

### CI/CD 統合

ASSERT は CI/CD パイプラインに統合可能。デモではプロンプト修正のみのバージョンが AI Safety Regression テストに不合格となり、ACS ガードレール適用後に初めてパイプラインを通過する様子を示した。

### 今後の研究方向

#### Continuous Evaluation（継続的評価）

強化学習（RL）ベースの適応型攻撃者がエージェントを継続的にプローブ:
- 攻撃提案 → エージェント実行 → 失敗スコアリング → ポリシー更新の閉ループ
- 顧客が発見する前に障害モードを検出
- 静的ファジングではなく、エージェントの弱点を学習する知的攻撃

#### Continuous Learning（継続的学習）

適応型攻撃者の発見を自動修正に直結:
- **プロンプト最適化**（低コスト）: GEPA、DSPy 等で指示を自動書き換え
- **ハーネス・ツール改善**（中コスト）: スキャフォールディング、ツール仕様、制御フロー、ガードレールの修正
- **モデル重み更新**（高コスト）: SFT/RL によるファインチューニング

#### Content Provenance（コンテンツ出自）

AI 生成画像への不可視ウォーターマーク埋め込みと C2PA 標準による署名。マニフェストが失われても AI 生成コンテンツを特定可能。

#### Information Flow Tracking（情報フロー追跡）

ツール結果に完全性・機密性ラベルを付与し、ツールコールへ伝播。ポリシーエンジンが安全性を検証し、プロンプトインジェクションによるデータ流出を決定論的にブロック。GitHub CLI と Fabric で利用可能。

#### Multi-Agent セキュリティ

Microsoft Research との共同研究:
- **SocialReasoning-Bench**: マルチエージェント相互作用の社会的推論ベンチマーク
- **100+ エージェントへの Red Teaming**: 1 メッセージでネットワーク全体に波及する脆弱性の調査
