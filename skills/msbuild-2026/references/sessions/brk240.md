# Build context-aware agents: From data to decisions

Microsoft IQ プラットフォーム（Work IQ・Fabric IQ・Foundry IQ・Web IQ）を活用し、コンテキスト認識型エージェントを構築する方法を解説。統一インテリジェンスレイヤーによるエージェントの精度向上・コスト削減、コンテキスト委任パターン、エージェントの組織内アイデンティティ付与までを実演した。

## 概要

エージェント AI プロジェクトの 40% 以上が失敗すると予測されるなか、その主因はモデルの能力不足ではなくコンテキストの欠如にある。本セッションでは Microsoft IQ プラットフォームを「エージェンティックウェブのコンテキストレイヤー」として位置づけ、4 つの IQ（Work IQ・Fabric IQ・Foundry IQ・Web IQ）がエージェントに組織・ビジネス・ナレッジ・ウェブの共有理解を与える仕組みを解説した。

Amanda Silver が各 IQ の設計思想とユースケースを説明し、Marco Casalaina が返金処理エージェントのライブデモを通じて、4 つの IQ すべてを統合したコンテキスト認識型エージェントの構築と動作を実演した。

## キーポイント

1. **統一コンテキストレイヤーの重要性**: Gartner によれば統一コンテキストレイヤーの優先はエージェント精度を最大 80% 向上、運用コストを最大 60% 削減する
2. **Microsoft IQ の 4 層構成**: Work IQ（組織・人・コラボレーション）、Fabric IQ（ビジネスデータ・オントロジー）、Foundry IQ（ナレッジ・ポリシー）、Web IQ（ウェブ・ニュース・画像・動画）
3. **Web IQ の 3 つの強み**: 品質（構造化・引用付き結果）、速度（P95 200ms 未満）、効率（パッセージレベルランキングによるトークン消費削減）
4. **Foundry IQ のサーバーレス開発者ティア**: 開発者が容易に開始でき、プロダクションにスケール可能な新ティアを導入
5. **Fabric IQ のオントロジー**: セマンティックモデルからオントロジーを自動生成し、エージェントにビジネスエンティティと関係性の理解を付与。MCP サーバーとしてエージェントに接続可能
6. **コンテキスト委任パターン**: Fabric Data Agent に業務固有のコンテキスト（データ辞書、例示クエリ、略語定義）を集約し、下流の全エージェントからの再定義を不要にする
7. **Work IQ の 4 コンポーネント**: Chat（会話）、Context（組織コンテキスト取得）、Tools（M365 アクション実行）、Workspaces（中間成果物の永続化）。2026年6月16日 GA
8. **エージェントの組織内アイデンティティ**: Agent 365 によりエージェントが組織図上にアイデンティティを持ち、専用メールボックス・Teams チャット・独立した権限スコープで動作

## 詳細

### Microsoft IQ プラットフォーム概要

エージェントが組織で有効に機能するには、データへのアクセスだけでなく人・データ・ポリシー・ビジネスコンテキストの共有理解が必要である。Microsoft IQ はこの課題に対し、4 つの専門レイヤーを統一プラットフォームとして提供する。

- **Work IQ**: 組織のコラボレーション文脈（メール・Teams・会議・ドキュメント）をエージェントに提供
- **Fabric IQ**: ビジネスの運用データ・セマンティックモデル・オントロジーを通じてリアルタイムのビジネス状態を把握
- **Foundry IQ**: ナレッジベース・ポリシードキュメント・エンタープライズ知識を統合検索
- **Web IQ**: ウェブ・ニュース・画像・動画からのリアルタイムグラウンディング

### Web IQ — エージェント向けウェブグラウンディング

Web IQ は従来の検索 API のラッパーではなく、エージェンティックシステム専用のグラウンディングプラットフォーム。NASDAQ Boardvantage での採用事例が紹介された。

- **品質**: 構造化された引用付き結果をウェブ・ニュース・画像・動画横断で提供
- **速度**: P95 レイテンシ 164ms（約 2.5 倍高速）でマルチステップエージェントワークフローに対応
- **効率**: パッセージレベルランキングにより情報密度を最大化し、トークン消費を削減
- 一部 Azure 顧客に提供中

### Foundry IQ — エンタープライズナレッジの統合

Foundry IQ はカスタム RAG パイプライン構築の負担を軽減する統合ナレッジプレーン。知識ソース → コンテキストレイヤー（メモリ・エンベディング・セマンティックモデル）→ エージェント/アプリの 3 層構成。

- **知識ソースの統合**: エンタープライズデータ・ドキュメント・アプリ・ウェブを一元的に取り込み
- **サーバーレス開発者ティア**: インフラ管理不要で即座に開始可能（Public Preview）
- **エージェンティック検索**: リランキング・リトライを含むサブエージェントとして動作
- **再利用性**: 1 つのナレッジベースを複数エージェントで共有可能
- Sitecore がマーケティング・ブランドナレッジのエージェント横断アクセスに活用

### Fabric IQ — ビジネスコンテキストの基盤

Fabric IQ は OneLake・セマンティックモデル・オントロジーの 3 レイヤーでビジネスの運用状態をエージェントに供給する。

- **OneLake**: 分析・運用データをクラウド横断で統一するデータレイヤー。サイロを排除し全エージェントが同一ソースで稼働
- **セマンティックモデル**: Power BI で定義済みのビジネスメトリクスを再利用。35M+ ユーザーが日常的に利用
- **オントロジー**: ビジネスエンティティ・関係性・ルールを定義。セマンティックモデルから自動生成可能。エージェントがテーブルではなくビジネス概念で推論可能に
- **MCP サーバーとしてのオントロジー**: 定義したオントロジーを MCP サーバー経由でエージェントに接続
- Hanwha Q Cells が AI データセンターのエネルギー管理にオントロジーを活用

### Work IQ — エージェントのための組織インテリジェンス

Work IQ は M365 のデータを直接エージェントに提供する「組織の人レイヤー」。従来の Graph API とは異なり、エージェントのスケールと速度に最適化されている。

- **Chat**: エージェントと人・エージェント間の会話インタラクション
- **Context**: メール・会議・チャット・ドキュメントから組織コンテキストを取得。独自の検索パイプライン構築が不要
- **Tools**: M365 内でのアクション実行（メール送信、メッセージ送信等）をガバナンス付きで提供
- **Workspaces**: エージェントの中間成果物を永続化し、長時間タスクや他エージェント・人間との共有を実現
- 2026年6月16日 GA 予定。Miro が統合事例として紹介

### ライブデモ: 返金処理エージェント

Marco Casalaina が 4 つの IQ すべてを統合した返金処理エージェントをデモ。

1. **Work IQ + Copilot CLI**: Amanda から届いたエージェント構想のメールを Work IQ で取得し、エージェントの仕様を策定
2. **Web IQ**: Sandy Fire（実際の山火事）のリアルタイム情報を取得し配送影響を判断
3. **Foundry IQ**: 返金ポリシードキュメントをナレッジベースとして統合
4. **Fabric IQ**: Fabric Data Agent 経由で注文・配送データのオントロジーにアクセス。業務固有の略語（SRC = Shipping Refund Concession）等のコンテキストを Data Agent に集約
5. **Agent 365**: エージェントを組織図に配置し、専用メールボックスで顧客対応を自動化。権限スコープを個別に設定可能

### エンタープライズガバナンス基盤

Microsoft IQ の全レイヤーはアイデンティティ・ガバナンス・セキュリティの共通基盤上に構築されている。

- **権限認識**: エージェントはデフォルトで権限認識。Entra ID による Agentic User Authorization（OBO トークン）
- **トレーサビリティ**: すべての推奨・アクションが共有ビジネスコンテキストに紐付けて追跡可能
- **一元的ガバナンス**: ポリシー更新は全エージェントに自動反映
