## 概要

エージェンティック AI ワークロードを本番環境で安定稼働させるためのランタイム要件と、Azure Container Apps が提供するソリューションを解説するセッション。Gartner が「2027年までに40%以上のエージェンティック AI プロジェクトがキャンセルされる」と予測する背景には、モデルの問題ではなくランタイムの問題（コールドスタート、暴走ループによるコスト浪費、分離不足によるデータリーク、コンテキスト喪失）があるとし、Azure Container Apps Sandboxes を **Public Preview** として発表した。

デモでは Container Apps 環境上にサーバーレス GPU で Whisper/Kokoro を動作させ、マルチエージェントブローカーを介して音声対話を実現。Copilot CLI を動かすサンドボックスエージェント（Aria）、個人アシスタント（Nova）、Azure SRE Agent の 3 エージェントをブラウザ UI と電話の両方から操作するマルチエージェントシステムを実演した。

顧客事例として Auger が Azure Container Apps 上で数千のエージェントによるサプライチェーン自律化を実現し、2025年3月のコード初行から1年余りで本番2顧客・40以上のサービスを稼働させている事例を紹介した。

## キーポイント

1. **Azure Container Apps Sandboxes**: エージェント向けの高速・分離・ステートフルなコンピュート基盤。サブ秒コールドスタート、メモリ+ディスクのスナップショット・レジューム、レイヤー7ファイアウォールによるネットワーク分離、ゼロへのスケールを提供（**Public Preview**）
2. **Container Apps Express**: 環境レス（物理クラスター不要）の高速 Container Apps。Web アプリ向けに Sandboxes 基盤上で構築された軽量デプロイ体験（**GA**）
3. **サーバーレス GPU**: T4 および A100 シリーズを15リージョンで提供。スケールトゥゼロ対応で、Whisper・Kokoro 等のカスタムモデルを容易にホスト
4. **Sandbox テンプレートとコネクタ**: 事前構成テンプレートからワンクリックでサンドボックス作成。コネクタ（カレンダー、Teams、メール等）はサンドボックスグループから継承
5. **Microsoft 自身が利用**: GitHub Copilot Cloud Sandboxes、Foundry Hosted Agent Service、Container Apps Express が同じランタイム基盤上で稼働
6. **エージェントランタイムの5要件**: 高速起動とレジューム、ツール呼び出し実行、状態の永続化、タスク単位の強力な分離、デフォルトでのセキュリティ

## 詳細

### エージェントランタイムが壊れる5つのポイント（Devanshi Joshi）

Devanshi はエージェントが本番で失敗する5つのランタイム課題を提示した:

1. **予算の暴走（Budgets that burn unattended）**: 無人のエージェントループが一晩でトークン予算を消費。「エージェントは予算を燃やすのが極めて得意で、無人だとさらに上手くなる」
2. **信頼できないコードの実行（Untrusted code on a developer's laptop）**: 開発マシン上での実行は SSH キー・本番クッキー・認証情報へのリスク。サンドボックスではなく脆弱性
3. **コールドスタート（Cold starts that throttle the loop）**: 10秒の起動は過去の技術。100ミリ秒未満が現在の基準。コールドスタートだけを解決するスタートアップが存在するほどの課題
4. **再起動によるコンテキスト喪失（Workspaces that die on every restart）**: キャッシュ・中間状態がリセットされ、長時間実行エージェントが機能しない。毎回「環境セットアップ税」を支払う状態
5. **手作業によるツール接続（Tooling stitched together by hand）**: ランタイムごとに異なる設定・パッケージング・ルール

これらはモデルの問題ではなくランタイムの問題であり、解決には以下の5つのランタイム仕様が必要とした:

| 要件 | 解決する課題 |
|------|-------------|
| 高速起動とレジューム | エージェントはイベント駆動。サブ秒のコールドスタートが必須 |
| ツール呼び出し実行 | API 呼び出し・コード実行・クラウドサービス連携をランタイム内部で |
| 状態の永続化 | 時間単位の推論を行う長時間エージェントのコンテキスト維持 |
| タスク単位の強力な分離 | 各実行が独立したサンドボックスでデータリーク防止 |
| デフォルトでのセキュリティ | ID・データ・アクセス制御をランタイムレベルで統制 |

### マルチエージェント音声デモ（Simon Jakesch）

Simon は Container Apps 環境上で以下のアーキテクチャを構築しライブデモを実施した:

**上段（共通インフラ — Container Apps 環境）:**
- サーバーレス GPU 上の **Whisper**（Speech-to-Text）と **Kokoro**（Text-to-Speech）
- **マルチエージェントブローカー**: 全エージェントのプロンプト入出力をテキスト↔音声に変換

**下段（エージェント群 — ACA Sandboxes）:**
- **Aria**: Copilot CLI を実行するサンドボックス。tic-tac-toe ゲームを構築しポート80で公開するタスクを実演
- **Nova**: 個人アシスタント。カレンダー・Teams・メールコネクタ経由で会議情報とメッセージを取得
- **Azure SRE Agent**: アプリケーション監視。インフラの健全性と直近のコミット情報を報告

ブラウザ UI で各エージェントと音声で自然に対話。さらに Twilio の通話ゲートウェイ（Container Apps 上で稼働）経由で電話からのエージェント呼び出しも実演し、オムニチャネル体験を示した。

### ACA Sandboxes の機能デモ（Simon Jakesch）

#### レジューム（メモリ状態の保持）

- サンドボックスはスリープ後もメモリ状態を保持
- 20〜25分スリープしたサンドボックスをウェイクアップ→スクリプトがスリープ前の正確な状態から継続実行
- ディスクのみ、またはディスク+メモリのキャプチャを設定可能

#### 高速作成（サブ秒）

- テンプレートから「Create」ボタン一押しで即座にサンドボックス作成
- 接続後すぐに Copilot CLI が利用可能
- GitHub トークン等の認証情報はサンドボックスグループから自動継承

#### ネットワーク分離

- レイヤー7ファイアウォールによるホスト単位のアクセス制御
- 許可ホスト・拒否ルートの設定が可能
- プロセス分離だけでなくネットワーク分離まで提供

#### コネクタ

- サンドボックスグループに設定されたコネクタ（カレンダー、Teams、メール等）が個別サンドボックスに自動継承
- エンドポイントへの接続数に制限なし

#### スナップショット

- ディスク全体をスナップショットし、新しいサンドボックスのベースとして利用
- エージェントの作業を巻き戻し・やり直しする際のチェックポイントとして機能
- デモではスナップショットから Aria の完全コピーを即座に作成

#### ポート公開

- サンドボックス内で動作するアプリのポートを動的に公開可能（デモでは Aria が構築した tic-tac-toe をポート80で公開）

### Container Apps Express と Sandboxes の使い分け

Simon が Q&A で説明した位置付け:

- **Sandboxes**: エージェントのための高速イテレーション環境。エージェントが使う環境であり、エージェントを動かす環境。トランジェントな性質を持つ（「空港を走る時の開いたままのラップトップ」に例えた）
- **Express**: デプロイ先。環境レス（物理クラスターなし）で、従来の Container Apps のベストを凝縮した Web アプリ向け。約1ヶ月前にローンチ済み

### Azure Container Apps Sandboxes のユースケース

スライドで提示された6つのユースケース:

| ユースケース | 説明 |
|-------------|------|
| Agent Workflows | タスク境界を超えて永続する分離ワークスペースを AI エージェントに提供 |
| AI Code Execution | LLM 生成コードを即時起動の分離環境で安全に実行 |
| Platform Building | Microsoft サービスと同じプリミティブの上に独自プラットフォームを構築 |
| Burst Workloads | 需要に応じてゼロから数千のサンドボックスにスケールアウト |
| Secure Multi-Tenant Compute | 複数テナントの信頼できないワークロードに強力な分離を保証 |
| Interactive User Sessions | ユーザーごとに独立したコンピュート環境を提供 |

### 顧客事例: Auger の自律サプライチェーン（Gopi Prashanth）

Auger の Chief AI & Agents Office である Gopi Prashanth が、Azure Container Apps 上で構築した自律サプライチェーンシステムを紹介した。

#### サプライチェーンの複雑性

- 1本のペンでさえ50のアクター、8階層、17カ国にまたがるサプライチェーン
- 大半の ERP システムは末端3層しか可視化できない
- ベトナムのタングステン鉱山が洪水で停止しても ERP は知らないが、Auger は既に影響を緩和している

#### Auger の3つの柱

1. **AUSCO（Auger Universal Supply Chain Ontology）**: データスキーマを超えた世界モデル。エンティティ・アクション・関数・モデル・ワークフロー・ナレッジを統一的に定義。エイリアシングレイヤーで「DC」=「Fulfillment Center」等の用語統一を実現
2. **AI ネイティブ**: スタック全層が同一用語体系を共有し、エージェントが快適に動作
3. **コンテキスト駆動**: 制度的知識（文書化されていない運用ノウハウ）を構造化して活用

#### Agentic Data Integration（ADI）

数千のエージェントが ACA 上で並行稼働し、顧客の Snowflake アカウント（900テーブル × 1000+カラム × 数百万行）を自動的に理解・マッピング:

1. **Understand**: スキーマ・カーディナリティのプロファイリング、AI によるビジネス意味の付与
2. **Map to Ontology**: エンティティ・関数のマッピング、質量バランス整合性ゲート
3. **Design & Build**: ETL DAG の設計、PySpark on Fabric の自動生成
4. **Deploy**: 顧客承認（HITL）、Microsoft Fabric へのプッシュ、パイプライン稼働

従来12ヶ月かかるデータ統合を数日に短縮。

#### リアルタイムコンテキストレイヤー

- What-if シナリオ（「この場所に倉庫を設置したら？」）にリアルタイムで回答
- モデリング→シミュレーション→結果取得を10秒以内に完了
- ACA エージェントが高速にサンドボックスを開き、ローカル判断を行い、データを取得

#### 実績

- 2025年3月に最初のコードを記述開始
- 現在: 本番2顧客、40以上のサービスが Azure Container Apps 上で稼働、日常的に数千のエージェントが動作
- 50% の実行高速化、85% の自律判断率を達成

### 顧客事例: SitecoreAI・EdChat

- **SitecoreAI**: 長時間稼働の自律エージェントがコード実行・ワークフロー管理・エンタープライズシステム連携を安全に実行。マルチテナント分離、高速スケールアウト、完全自動化を活用
- **EdChat（南オーストラリア教育省）**: 学生向け AI 学習プラットフォーム。マルチテナント分離とスケールトゥゼロにより、各学生がコスト効率の良い分離環境でコーディング・データ分析・探索を実行
