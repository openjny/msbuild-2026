## 概要

AI エージェントの構築は容易になったが、本番環境でのスケーラブルな運用には依然として多くの課題がある。本セッションでは Microsoft Foundry の Agent Service と Microsoft Agent Framework を使い、ローカルでのプロトタイピングからエンタープライズグレードのホスティング、継続的な最適化までのエンドツーエンドライフサイクルを 45 分間のライブデモで実演した。

デモシナリオとして「自律型光ファイバー障害対応エージェント」を構築。Azure ネットワーキングオペレーションチームが実際に直面する障害対応フロー（IoT シグナルによる障害検知 → 情報収集 → フィールド担当者のディスパッチ → D365 チケット起票 → Teams ステータス更新）をエージェントシステムとして再現した。

セッションは Build（構築）→ Deploy（デプロイ）→ Operate（運用）の 3 フェーズで構成され、各フェーズの開発者課題と Foundry による解決策が示された。

## キーポイント

1. **構築は容易になったが、問いは「作れるか」から「本番でスケールして信頼性高く運用できるか」に移行**
2. **エージェントはもはや固定ツール間のルーターではなく、動的にスキル・メモリ・サブエージェントを生成する汎用システム**
3. **Microsoft Agent Framework v1.0 のハーネス機能で、エージェントがコード実行・ファイル操作・シェルコマンドを安全に実行可能に**
4. **Foundry Toolbox は MCP 互換の単一エンドポイントで認証・ガードレール・ツール検索を一元管理**
5. **Hosted Agents はセッション単位のサンドボックス分離、サブ秒コールドスタート、アイドル時ゼロコスト**
6. **Routines によりエージェントがリアクティブからプロアクティブに転換**
7. **Agent Optimizer がプロダクショントレースとカスタム Rubric から最適化候補を自動生成し、11% のスコア改善を実演**
8. **Procedural Memory でセッション横断の学習を蓄積し、エージェントが運用するほど改善**

## 詳細

### エージェント時代の 3 つのシフト

Tina Schuchman がオープニングで整理した AI エージェントの現在地:

1. **構築が難しくなくなった** — GitHub Copilot・CLI・Cloud Code で数分以内にエージェントを立ち上げ可能。問いは「作れるか」から「信頼性高くスケールできるか」へ
2. **エージェントの能力が爆発的に向上** — 新しいエージェントを生成し、スキルを作り、メモリを獲得する汎用システムへ
3. **エージェントはツールではなくチームメイト** — チャットボットではなく、重要なビジネス成果を自律的に達成するオペレーター

企業が勝つには「1 つの強力なエージェント」ではなく、全機能（コンプライアンス・ファイナンス・サプライチェーン・サポート）に AI を組み込んだ協調エージェントシステムが必要であり、そのためのオペレーティングシステムが Foundry であると位置付けた。

### Microsoft Agent Platform の全体像

GitHub でコード・プロンプト・スキル・Eval 定義を構築し、Foundry でホスティング・最適化（ルーティング + RL）し、M365・Teams・Apps + APIs を通じてユーザーに届けるガバナンス付きライフサイクルを提示:

- **AI Models** — 11,000 以上のモデル
- **Context with IQ** — エンタープライズナレッジ
- **Powered by Azure** — グローバルスケール、データゾーン、ソブリン制御
- **Manage with Agent 365** — 監視・セキュリティ・ガバナンス、Agent ID

### Build: ローカルでのエージェント構築

Microsoft Agent Framework（Python）を使用し、VS Code 上でエージェントを構築するフローを実演。

#### Foundry Toolkit for VS Code

Foundry Toolkit 拡張機能（**GA**）の「Generate with Copilot」で光ファイバー障害対応エージェントをスキャフォールド。拡張機能は Foundry ベストプラクティスのスキルを自動統合し、エージェント生成時に適切なパターンを適用する。トレーシング・評価・モデル管理も VS Code 内で完結する。

#### Agent Framework のハーネス

Agent Framework v1.0（**Stable Release**）の新機能であるハーネスにより、エージェントは事前定義ツール実行だけでなく、動的なコード生成・調査・シェルコマンド実行が可能に。GitHub Copilot SDK（**GA soon**）を追加ハーネスとして接続し、Cloud Agent SDK との統合も可能。

#### Foundry Toolbox

Foundry Toolbox（**GA soon**）をツール管理基盤として使用し、MCP 互換の単一エンドポイントで以下を統合:

- **Foundry IQ**: サプライヤー契約・合意書のインデックス検索
- **Fabric IQ**: サイト稼働率・信頼性データ
- **Work IQ**: Teams・Outlook 連携（Microsoft Graph）
- **Web IQ**: Web 検索
- **Content Understanding**: PDF ドキュメントをエージェント可読フォーマット（テーブル・Markdown・図表・JSON）に変換

Toolbox の主要機能:
- **認証管理**: 各ツールへの認証を一括管理
- **ガードレール**: PII 漏洩防止等のポリシー設定
- **ツール検索**: タスクに関連するツールのみを返却し、コンテキストウィンドウの利用効率を最適化

#### ローカルデバッグ

F5 キーでエージェントをローカル起動し、ブレークポイントでリクエスト/レスポンスを検査可能。Toolbox への接続は単一の MCP 互換エンドポイント指定のみ。

### Deploy: Hosted Agents と Teams 公開

#### Voice Live 統合

デプロイ済みエージェントにワンクリックで音声モードを有効化（**GA** for prompt agents / **Public Preview** for hosted agents）。業界最先端の音声モデルを自動ラッピングし、WebSocket 経由でリアルタイムストリーミング応答を実現。デモでは光ファイバー端末仕様の問い合わせを音声で実行し、コネクタタイプの回答を得た。

#### Hosted Agents

Hosted Agents in Foundry Agent Service（**GA soon**）により、各会話・各 Routine が専用のサンドボックスセッションで実行される:

- **セッション分離**: 会話ごとに独立したワークスペース。サブコントラクター A の機密データがサブコントラクター B のセッションに漏洩しない
- **永続状態**: アイドルセッションでもファイルシステムの状態を保持。再開時に調査ファイル・ログ等を完全復元
- **ゼロコストアイドル**: セッションがアイドル中は課金なし。サブ秒コールドスタートで再開
- **フレームワーク非依存**: Open Claw・Hermes 等の長時間自律エージェントにも対応
- **Durable Task Scheduler 統合**: 全セッションがアイドル中でも人間承認待ち等の状態をサーバーレスで監視。承認イベントでセッションを自動再開

#### Routines

Routines（**Public Preview**）によりエージェントをプロアクティブに起動:

- ハートビート（時間ベース）やイベントベースのトリガーを設定
- デモでは 1 時間ごとに調査ログ確認 → 異常検知 → アラート → サブコントラクター手配のフローを自動実行
- Foundry がキュー管理・実行・トラッキングを一括管理

#### Teams 公開と Autopilot Agent

Foundry エージェントを Microsoft Teams に公開（**GA soon**）:

- 独自のアイデンティティ・メールアドレス・Teams プレゼンスを付与
- 会話を自ら開始し、アクションアイテムのフォローアップも可能
- Agent 365 によるエンドツーエンドのガバナンス（ID・ポリシー・権限が自動フロー）

Autopilot Agent としての公開は **Public Preview**。

### Operate: 監視・評価・最適化

#### トレースと可視性

Tracing and Evaluation for Hosted Agents（**GA soon**）:

- 全 LLM コール・ツール呼び出し・サブエージェントハンドオフを OpenTelemetry パイプラインで記録
- **トレースリプレイビュー**: 各リクエストの軌跡を時系列で再生。推論時間・ツールコール順序を可視化
- **トークンビュー**: リクエスト内のトークン消費分布を表示
- **8 倍速再生**: 会話全体をアニメーションで確認し、ユーザー体験を追体験

#### カスタム評価（Rubric）

Rubric for Custom Evaluation（**Public Preview**）:

- `azd ai agent eval init` コマンドでエージェントプロジェクトを評価用に初期化
- 評価データセットは既存トレース履歴から LLM が自動生成可能
- カスタム Rubric は複数の評価次元を重み付きで定義（例: ツール正確性、安全性警告、音声最適化簡潔性）
- LLM がエージェント固有の評価基準を自動提案し、開発者がカスタマイズ

デモでは音声エージェント向けに「voice optimized conciseness」の重みを 3 → 10 に変更し、音声応答の簡潔さを重視する評価基準を構築。

#### Agent Optimizer

Agent Optimizer in Foundry Agent Service（**Private Preview**）:

- `azd ai agent optimize` コマンドで最適化ループを起動
- システムプロンプト・スキル構成・ツール設定・ターゲットモデルを変数として最適化
- データサイエンス技法により複数の改善候補を自動生成
- デモでは音声簡潔性を重視した最適化で **11% のスコア改善**を達成
- 候補ごとに品質・コスト・レイテンシを比較表示
- 既存プロンプトと新プロンプトの diff を確認可能
- フォールドトレースバックとリネージュロールバックでいつでも前バージョンに復帰

#### Procedural Memory

Procedural Memory in Foundry Agent Service（**Public Preview**）:

- セッション横断でエージェントがプレイブックを学習
- 手動のプロンプト・スキル書き換え不要で、運用するほど賢く・安全に・コスト効率良く
- 開発者がすべての変更を制御可能

### 顧客事例

- **Ecopetrol**: 14 か国のミッションクリティカルなエネルギーオペレーションを Foundry Agent Service で運用。ID・メモリ・ガバナンス・可観測性を活用
- **Twilio**: Twilio Agent Connect（OSS フレームワーク）を Foundry の Hosted Agents 上にデプロイ
- **KPMG**: グローバル KPMG Workbench プラットフォームを Hosted Agents 上に構築。Foundry Toolbox のツール・スキルを活用
